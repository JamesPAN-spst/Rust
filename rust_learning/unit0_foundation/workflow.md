## 第二步：章节规划
### Ch 0.1 工具链
- 目标：帮助初学者建立 Rust 工具链心智模型，减少“命令会用但不知道为什么”的状态。
- 知识点规划：
  1. rustup / cargo / rustc 的职责分层
  2. toolchain、channel 与 edition 的区别
  3. debug / release 的使用场景
  4. rustfmt / clippy 的反馈价值
  5. 编译错误阅读顺序与实战理解
- 快速测试方向：围绕职责区分、edition 概念、构建模式、工具用途、错误定位展开。
- 练习梯度：基础解释题 → 运行观察题 → lint 改写题 → 错误复盘题。

### Ch 0.2 心态建设
- 目标：帮助初学者建立面对 Rust 学习曲线、borrow checker 和迭代反馈时的正确预期。
- 知识点规划：
  1. Rust 前期学习曲线为何更陡
  2. 编译器作为“规则老师”的阅读方式
  3. ownership / borrowing 阶段的正常挫败感
  4. cargo check / test / clippy 的迭代工作流
  5. 面对 borrow checker 时如何避免把 clone 当默认修复
- 快速测试方向：围绕错误阅读顺序、借用冲突心态、工作流选择、clone 使用边界展开。
- 练习梯度：心态解释题 → borrow checker 改写题 → clone 重构题 → 迭代节奏设计题。
