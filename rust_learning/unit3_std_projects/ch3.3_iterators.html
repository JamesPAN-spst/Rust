Ch 3.3 迭代器

这一章把 Rust 迭代器讲成“带所有权语义的数据管道”：为什么 map/filter 是惰性的、iter/iter_mut/into_iter 到底在表达什么、collect 为什么经常需要类型注解，以及链式处理中 ownership 该怎么读。

🎯 学习目标

  • 理解迭代器与 `for` 循环的分工，以及惰性执行的核心含义
  • 区分 `iter`、`iter_mut`、`into_iter` 的借用与消费语义
  • 掌握 `map` / `filter` / `collect` 的常见组合方式
  • 能读懂迭代器链中引用、复制、clone 和 move 的变化

📋 前置知识

建议先掌握集合与闭包基础。若还没建立“借用遍历”和“消费遍历”的区别，迭代器报错会显得非常随机。

3.3.1 迭代器不是“更花哨的循环”，而是惰性数据管道

🔁

为什么 Rust 既保留 `for` 循环，又大力推广迭代器适配器？

for 适合直接表达遍历动作；迭代器则把“如何一步步产生元素”抽象成组合式管道。只有在遇到消费器（如 collect、sum、for_each）时，惰性链条才会真正执行。

正式规则：大多数迭代器适配器都是惰性的：调用 map、filter 不会立即执行，直到后面有消费者触发。

Rustiterator_lazy.rs
fn main() {
    let values = vec![1, 2, 3, 4];
    let sum: i32 = values.iter()
        .filter(|&&n| n % 2 == 0)
        .map(|&n| n * 10)
        .sum();

    println!("sum = {sum}");
}
输出 
sum = 60

3.3.2 `iter`、`iter_mut`、`into_iter`：你到底想借用、修改还是消费容器？

🧠

同样是遍历一个 Vec，为什么 Rust 要给你三个不同入口？

因为遍历不仅是“拿到元素”，还隐含 ownership 决策：iter() 产生不可变引用，iter_mut() 产生可变引用，into_iter() 则消费容器并交出元素所有权。

正式规则：选择哪一种迭代方式，取决于你是否还要继续使用原容器、是否要原地修改元素，以及是否需要拿走元素本身。

Rustiter_kinds.rs
fn main() {
    let mut names = vec![String::from("rust"), String::from("cargo")];

    for name in names.iter_mut() {
        name.make_ascii_uppercase();
    }

    println!("{:?}", names);
}
输出 
["RUST", "CARGO"]
因为 `into_iter()` 会消费容器，把元素所有权逐个交出去；容器本身也就被 move 走了。

3.3.3 `map`、`filter`、`collect`：把变换意图写进链条里

🛠️

遍历集合并构造新结果时，为什么迭代器链通常比手写临时变量更紧凑？

迭代器链把“筛选、映射、收集”的意图串成声明式流程。读代码的人先看到处理步骤，再去看闭包细节，通常更容易把握数据流。

正式规则：collect() 需要目标类型上下文，常通过变量类型注解或 turbofish 明确收集到哪种集合。

Rustcollect_vec.rs
fn main() {
    let raw = vec!["1", "20", "300"];
    let numbers: Vec<usize> = raw.iter()
        .map(|text| text.parse::<usize>().unwrap())
        .collect();

    println!("{:?}", numbers);
}
输出 
[1, 20, 300]

3.3.4 迭代器链中的所有权：何时借用，何时复制，何时 move

📌

为什么有些链式代码总要补 `copied()`、`cloned()`，而另一些却不能随便加？

迭代器会把元素以“引用”还是“拥有值”的形式往后传递，这直接影响闭包参数类型和后续容器是否还能用。copied() 适合 `Copy` 类型，cloned() 适合需要显式复制拥有型值的场景。

正式规则：先判断链条里流动的是 &T 还是 T，再决定是否需要 copied / cloned / into_iter。

Rustiterator_ownership.rs
fn main() {
    let values = vec![1, 2, 3];
    let doubled: Vec<i32> = values.iter()
        .copied()
        .map(|n| n * 2)
        .collect();

    println!("doubled = {:?}", doubled);
    println!("original = {:?}", values);
}
输出 
doubled = [2, 4, 6]
original = [1, 2, 3]

📌 章节总结

  • 迭代器的价值不只是省代码，而是把数据流和 ownership 决策写得更清楚。
  • `iter` 借用、`iter_mut` 可变借用、`into_iter` 消费容器，这三者不能混着理解。
  • `map`、`filter` 等适配器默认惰性，真正执行依赖 `collect`、`sum` 等消费者。
  • 当链条里流动的是引用时，需要明确是否应复制、clone，还是改为消费原容器。
