Ch 3.1 字符串

这一章把 Rust 字符串放回真实语境：为什么同时存在 String 和 &str、UTF-8 会带来哪些设计取舍、常用拼接与遍历 API 应该怎么选，以及为什么 Rust 刻意不提供“看起来很方便”的字符索引。

🎯 学习目标

  • 区分 `String` 与 `&str` 的所有权语义与参数设计取舍
  • 理解 UTF-8、字节长度、字符数量和切片边界之间的关系
  • 掌握常见字符串构造、拼接、遍历与切分 API 的使用方式
  • 避免字符串索引、非法切片和“把 len 当字符数”这类常见误区

📋 前置知识

建议先掌握 Ownership、Borrowing、基本类型和迭代器的入门直觉。若还把字符串理解成“固定长度字符数组”，会很容易误判 Rust 字符串 API 的设计原因。

3.1.1 `String` 和 `&str`：一个拥有数据，一个借用视图

🤔

为什么有时候函数签名写 `String`，有时候却更推荐 `&str`？它们看起来都像“字符串”。

`String` 是可增长的拥有型 UTF-8 缓冲区，负责管理堆内存；`&str` 是对 UTF-8 文本的借用切片，只读且不拥有底层数据。

正式规则：如果函数只需要读取文本，优先接收 &str；只有确实要接管或构造新字符串时才接收 String。

Ruststring_vs_str.rs
fn print_label(text: &str) {
    println!("label = {text}");
}

fn main() {
    let owned = String::from("rust");
    let borrowed = "book";

    print_label(&owned);
    print_label(borrowed);
}
输出 
label = rust
label = book

3.1.2 UTF-8 不是“一个字符等于一个字节”

🌐

为什么 `"你好".len()` 返回 6，而不是 2？如果按字节切错位置，又为什么会 panic？

Rust 字符串以 UTF-8 存储，因此 len() 返回的是字节数，不是人眼看到的“字符数”。Unicode 标量值、字节和字形簇是三层不同概念。

正式规则：String / &str 必须始终保持合法 UTF-8，因此切片边界只能落在有效字符边界上。

Rustutf8_len.rs
fn main() {
    let text = "你好";
    println!("bytes = {}", text.len());
    println!("chars = {}", text.chars().count());
}
输出 
bytes = 6
chars = 2
因为第 n 个“字符”在 UTF-8 中并没有 O(1) 的字节偏移量。Rust 不想用一个看似简单的索引语法掩盖潜在的线性扫描和非法切片风险。

3.1.3 构造和拼接：`push_str`、`push`、`format!` 各解决什么问题？

🧱

拼接字符串时，什么时候应该原地修改，什么时候应该新建结果？

如果你已经拥有可变 String，用 push_str / push 原地追加最直接；如果需要组合多个来源并生成新值，format! 可读性通常更高。

正式规则：push_str 借用 &str，不会拿走追加文本的所有权；format! 返回新的 String。

Rustbuild_string.rs
fn main() {
    let mut path = String::from("logs");
    path.push('/');
    path.push_str("app.log");

    let display = format!("reading {path}");
    println!("{display}");
}
输出 
reading logs/app.log

3.1.4 遍历与切片：按字节、按字符、按模式要分清

🔎

如果你要统计单词、枚举字符或截取前缀，为什么 Rust 鼓励你显式选择遍历方式？

字符串处理并没有唯一“默认粒度”。你可以按 bytes() 处理底层协议，按 chars() 处理 Unicode 标量值，按 split_whitespace() 等模式处理更高层文本结构。

正式规则：面对字符串处理问题时，先确定你真正操作的是字节、字符，还是文本片段；API 选择取决于这个粒度。

Rustiterate_str.rs
fn main() {
    let line = "rust makes systems safer";
    let words: Vec<&str> = line.split_whitespace().collect();

    println!("word count = {}", words.len());
    println!("first word = {}", words[0]);
}
输出 
word count = 4
first word = rust

📌 章节总结

  • `String` 负责拥有和扩展 UTF-8 文本，`&str` 负责借用和只读视图。
  • `len()` 返回字节数，不等于“字符数量”；切片必须落在合法 UTF-8 边界。
  • 字符串处理要先确定粒度：字节、字符还是文本片段。
  • 只读文本优先用 `&str`，原地修改再使用可变 `String`。
