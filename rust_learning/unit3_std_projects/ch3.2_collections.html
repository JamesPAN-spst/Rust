Ch 3.2 集合

这一章把 Unit 3 的“数据容器”心智模型补齐：什么时候先用 Vec，什么时候换成 HashMap，集合为什么会和借用检查强相关，以及项目里最常见的收集、计数、排序、去重模式应该怎么表达。

🎯 学习目标

  • 理解 `Vec` 与 `HashMap` 分别适合什么访问模式
  • 掌握集合增长、借用和重分配之间的关键关系
  • 学会用 `entry`、排序、去重等常见集合模式组织数据
  • 避免“拿着元素引用还继续修改容器”这类常见错误

📋 前置知识

建议先理解所有权、借用和字符串基础。很多集合报错本质上不是“容器 API 难”，而是集合内部元素地址与借用关系会受修改操作影响。

3.2.1 `Vec<T>`：默认的可增长顺序集合

📦

为什么 Rust 教程一讲集合，几乎都会先讲 `Vec`？

`Vec<T>` 以连续内存存放同类型元素，支持按顺序追加、索引和遍历，是绝大多数“先收集一批同类数据再处理”的默认选择。

正式规则：Vec<T> 拥有其中所有元素，长度可变；push 可能触发重新分配，因此集合增长与借用规则紧密相关。

Rustvec_basics.rs
fn main() {
    let mut scores = vec![10, 20];
    scores.push(30);

    println!("len = {}", scores.len());
    println!("last = {}", scores[scores.len() - 1]);
}
输出 
len = 3
last = 30

3.2.2 借用与重分配：为什么拿着元素引用时不能继续 `push`？

⚠️

如果 `Vec` 扩容后搬家了，之前指向旧地址的引用还安全吗？

`Vec` 的元素存储在堆上，扩容时可能整体搬迁到新内存。因此 Rust 不允许你在持有元素引用时继续做可能重分配的修改操作。

正式规则：当对集合元素的借用仍然有效时，不能执行会与该借用冲突的可变操作；这能阻止悬垂引用。

Rustvec_borrow.rs
fn main() {
    let mut values = vec![1, 2, 3];
    let first = values.first().copied().unwrap();

    values.push(4);
    println!("first = {first}, len = {}", values.len());
}
输出 
first = 1, len = 4
因为 `i32` 实现了 Copy，把第一个元素复制出来后，就不再持有对 Vec 内部存储的借用，后续 push 不会与旧引用冲突。

3.2.3 `HashMap<K, V>`：键值查找与所有权进入表中

🗂️

当你需要“通过键查值”时，为什么 Vec 再怎么遍历也不如 HashMap 合适？

`HashMap` 适合根据键快速定位值。把拥有型键值插入表时，所有权会移动进 map；若只借用外部数据，就要明确使用引用型键值并保证生命周期。

正式规则：insert 对拥有型键值通常会接管所有权；更新已有键时，entry API 比“先查再插”更自然。

Rusthashmap_entry.rs
use std::collections::HashMap;

fn main() {
    let text = "rust makes rust safer";
    let mut counts = HashMap::new();

    for word in text.split_whitespace() {
        *counts.entry(word).or_insert(0) += 1;
    }

    println!("rust => {}", counts["rust"]);
}
输出 
rust => 2

3.2.4 常见集合模式：收集、去重、计数与分组

🧭

面对业务数据时，集合往往不是孤立使用，而是作为“中间状态容器”。哪些模式最常见？

日常项目里常见的模式包括：先把输入收集到 `Vec` 中排序处理、用 `HashMap` 计数或索引、用 `retain` / `dedup` 过滤重复项、以及配合迭代器把集合当作数据管道的承接层。

正式规则：先根据访问模式选集合：顺序遍历优先 `Vec`，键值查询优先 `HashMap`，之后再考虑是否需要排序、去重或额外索引。

Rustcollection_pattern.rs
fn main() {
    let mut ids = vec![3, 1, 3, 2, 2];
    ids.sort();
    ids.dedup();

    println!("unique ids = {:?}", ids);
}
输出 
unique ids = [1, 2, 3]

📌 章节总结

  • `Vec` 适合顺序收集与遍历，`HashMap` 适合通过键查值。
  • 集合修改可能影响内部存储位置，因此借用规则不能和容器操作分开理解。
  • `entry` 模式能优雅表达“查找或初始化再更新”的逻辑。
  • 排序、去重、计数和索引是项目中最常见的集合组织方式。
