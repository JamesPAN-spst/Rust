Unit 3 · 标准库与项目实践

集合类型

集合是程序管理多个值的核心工具。Rust 标准库提供了 Vec、HashMap、HashSet 等高效集合，它们与所有权系统深度集成，确保内存安全的同时保持零成本抽象。

01Vec<T>

🎯 为什么需要了解这个

数组 [T; N] 大小固定，无法在运行时增长。当你需要一个可增长的同类型元素序列时，Vec<T> 是 Rust 中最基础、使用最频繁的集合类型。理解它的内存模型对写出高效代码至关重要。

🔍 核心思路

Vec<T> 的内部结构和 String 几乎一致（String 就是 Vec<u8> 的包装）：

  • ptr：指向堆上连续内存的指针
  • len：当前元素数量
  • cap：已分配的容量（可容纳的元素数）

扩容策略：当 len == cap 时 push 会触发重新分配。新容量通常为旧容量的 2 倍（实现细节可能随版本变化）。扩容涉及分配新内存 + 复制旧数据 + 释放旧内存，代价不可忽视。

📐 规则与语法
RUSTvec_basics.rs
// === 创建 ===
let v1: Vec<i32> = Vec::new();          // 空 Vec
let v2 = vec![1, 2, 3];                  // 宏创建
let v3 = vec![0; 10];                    // 10 个 0
let v4 = Vec::with_capacity(100);        // 预分配

// === 增 ===
let mut v = vec![1, 2, 3];
v.push(4);                               // 末尾添加
v.insert(1, 10);                         // 在索引 1 处插入 → [1, 10, 2, 3, 4]
v.extend([5, 6, 7]);                     // 批量追加

// === 删 ===
let last = v.pop();                      // 移除末尾 → Some(7)
let removed = v.remove(1);              // 移除索引 1 → 10（后面元素前移）
v.retain(|&x| x % 2 != 0);             // 保留奇数

// === 查 ===
let first = v[0];                        // 直接索引（越界 panic！）
let maybe = v.get(99);                   // 安全访问 → None
let has = v.contains(&3);               // 存在判断

// === 改 ===
v[0] = 100;                             // 直接修改
if let Some(elem) = v.get_mut(1) {
    *elem *= 2;                          // 通过可变引用修改
}
RUSTvec_slice.rs
let v = vec![10, 20, 30, 40, 50];

// 切片：Vec 自动 Deref 为 &[T]
let slice: &[i32] = &v;           // 整个 Vec 的切片
let mid: &[i32] = &v[1..4];       // [20, 30, 40]

// 函数参数推荐用 &[T] 而不是 &Vec<T>
fn sum(nums: &[i32]) -> i32 {
    nums.iter().sum()
}

let total = sum(&v);               // Vec 自动解引用为 &[i32]
let partial = sum(&v[1..3]);       // 也可以传切片
🌐 工程实践
  • 预分配：已知大致元素数时用 Vec::with_capacity(n)，减少扩容次数
  • 参数类型：函数参数用 &[T] 而非 &Vec<T>，更通用
  • 批量操作：extend 优于多次 push，编译器可一次性计算所需容量
  • 注意 remove(i)：需要 O(n) 移动元素。如果不关心顺序，用 swap_remove(i)（O(1)）
✏️ 示例分析与易错点
RUSTvec_borrow_trap.rs
let mut v = vec![1, 2, 3, 4, 5];

// ❌ 同时持有不可变引用和可变操作
let first = &v[0];      // 不可变借用
v.push(6);              // 可变借用——编译错误！
// push 可能触发重分配，使 first 悬垂
// println!("{}", first);

// ✅ 确保不可变引用的作用域结束后再修改
let first = v[0];       // 拷贝值（i32 实现了 Copy）
v.push(6);              // OK
println!("{}", first);   // OK，first 是独立的值

⚠ Vec 的 push 可能触发重分配，使所有现存的引用失效。这就是为什么不能在持有 Vec 元素引用的同时修改 Vec。

02HashMap<K,V>

🎯 为什么需要了解这个

当你需要通过"键"快速查找"值"时，HashMap 是首选。它提供平均 O(1) 的查找、插入和删除操作，是计数、缓存、映射等场景的核心数据结构。

🔍 核心思路

HashMap 的两个关键约束：

  • Key 必须实现 Eq + Hash：基本类型（i32, String, bool 等）默认满足；浮点数 f64 不行（NaN != NaN 违反 Eq）
  • 所有权转移：插入时，实现 Copy 的 key/value 会被拷贝；不实现 Copy 的（如 String）会被移动

标准库使用 SipHash 作为默认哈希算法，牺牲了少量速度换取抗 HashDoS 攻击的安全性。

📐 规则与语法
RUSThashmap_basics.rs
use std::collections::HashMap;

// === 创建 ===
let mut scores: HashMap<String, i32> = HashMap::new();

// === 插入 ===
scores.insert(String::from("Alice"), 95);
scores.insert(String::from("Bob"), 87);
// 注意：String key 的所有权转移给了 HashMap

// 重复插入会覆盖
scores.insert(String::from("Alice"), 100);  // Alice → 100

// === 查询 ===
let alice_score = scores.get("Alice");       // Some(&100)
let unknown = scores.get("Charlie");          // None

// 用 if let 解包
if let Some(score) = scores.get("Bob") {
    println!("Bob 得了 {} 分", score);
}

// === 遍历 ===
for (name, score) in &scores {
    println!("{}: {}", name, score);         // 顺序不确定！
}

// === 删除 ===
let removed = scores.remove("Bob");          // Some(87)

// === 从迭代器构建 ===
let names = vec!["Alice", "Bob"];
let vals = vec![95, 87];
let map: HashMap<&str, i32> = names.into_iter().zip(vals).collect();

entry API——避免重复查找：

RUSTentry_api.rs
use std::collections::HashMap;

let text = "hello world hello rust hello";
let mut word_count: HashMap<&str, i32> = HashMap::new();

for word in text.split_whitespace() {
    // 如果 key 不存在，插入 0；然后对值 +1
    let count = word_count.entry(word).or_insert(0);
    *count += 1;
}
// {"hello": 3, "world": 1, "rust": 1}

// entry 的其他变体
let mut cache: HashMap<String, Vec<u8>> = HashMap::new();

// or_insert_with：用闭包延迟创建默认值
cache.entry("key".to_string())
     .or_insert_with(|| {
         println!("计算默认值...");
         vec![1, 2, 3]
     });

// or_default：使用 Default trait
let mut counts: HashMap<&str, Vec<i32>> = HashMap::new();
counts.entry("a").or_default().push(1);
counts.entry("a").or_default().push(2);
// {"a": [1, 2]}
🌐 工程实践
  • 预分配：HashMap::with_capacity(n) 避免频繁 rehash
  • 借用查询：.get("str") 接受 &str 查询 String key（通过 Borrow trait）
  • 性能敏感：如需更快的哈希，可换用 FxHashMap（rustc-hash crate）
  • 复合 key：可用元组 (i32, i32) 作为 key，只要实现了 Eq + Hash
✏️ 示例分析与易错点
RUSThashmap_ownership.rs
use std::collections::HashMap;

let key = String::from("name");
let value = String::from("Alice");

let mut map = HashMap::new();
map.insert(key, value);

// ❌ key 和 value 的所有权已转移
// println!("{}", key);    // 编译错误
// println!("{}", value);  // 编译错误

// ✅ 如果需要保留原值，插入引用或 clone
let key = String::from("name");
let value = String::from("Alice");
map.insert(key.clone(), value.clone());
println!("{} = {}", key, value);  // OK

// ✅ 或者用 &str 作为 key 类型
let mut map2: HashMap<&str, &str> = HashMap::new();
map2.insert("name", "Bob");   // &str 实现了 Copy

⚠ HashMap 的遍历顺序是不确定的（每次运行可能不同）。如果需要有序遍历，使用 BTreeMap。

⚡ 快速检测
entry("key").or_insert(default) 在 key 已存在时会怎样？

03HashSet<T>

🎯 为什么需要了解这个

当你需要快速判断"某个值是否出现过"，或者对两组数据做交集、并集运算时，HashSet 是最合适的工具。它保证元素唯一，查找是 O(1)。

🔍 核心思路

HashSet<T> 就是 HashMap<T, ()> 的封装——只存 key，不存 value。它具备 HashMap 对 key 的所有约束（Eq + Hash）和 O(1) 查找性能。

核心能力：去重和集合运算（交集、并集、差集、对称差）。

📐 规则与语法
RUSThashset_basics.rs
use std::collections::HashSet;

// 创建
let mut fruits: HashSet<&str> = HashSet::new();
fruits.insert("apple");
fruits.insert("banana");
fruits.insert("apple");     // 重复插入无效
println!("{}", fruits.len()); // 2

// 从迭代器创建（自动去重）
let nums: HashSet<i32> = vec![1, 2, 2, 3, 3, 3].into_iter().collect();
// {1, 2, 3}

// 查询
let has_apple = fruits.contains("apple");  // true
let has_grape = fruits.contains("grape");  // false

// 删除
fruits.remove("banana");  // true（删除成功）

集合运算：

RUSTset_operations.rs
use std::collections::HashSet;

let a: HashSet<i32> = [1, 2, 3, 4].iter().cloned().collect();
let b: HashSet<i32> = [3, 4, 5, 6].iter().cloned().collect();

// 交集：两个集合都有的元素
let inter: HashSet<&i32> = a.intersection(&b).collect();
// {3, 4}

// 并集：两个集合所有的元素
let union: HashSet<&i32> = a.union(&b).collect();
// {1, 2, 3, 4, 5, 6}

// 差集：在 a 中但不在 b 中
let diff: HashSet<&i32> = a.difference(&b).collect();
// {1, 2}

// 对称差：只在其中一个集合中
let sym: HashSet<&i32> = a.symmetric_difference(&b).collect();
// {1, 2, 5, 6}

// 子集判断
let c: HashSet<i32> = [1, 2].iter().cloned().collect();
println!("{}", c.is_subset(&a));  // true
🌐 工程实践
  • 去重：vec.into_iter().collect::<HashSet<_>>() 是最简洁的去重方式
  • 快速查找：当 Vec::contains 成为性能瓶颈时，换用 HashSet
  • 位运算替代：对少量已知元素，可用 &/| 运算符代替 intersection/union
  • 有序集合：需要排序时用 BTreeSet（基于 B 树，key 要求 Ord）
✏️ 示例分析与易错点
RUSTdedup_example.rs
use std::collections::HashSet;

// 实际场景：找出两个列表的共同用户
let list_a = vec!["alice", "bob", "charlie", "dave"];
let list_b = vec!["charlie", "dave", "eve", "frank"];

let set_a: HashSet<&str> = list_a.into_iter().collect();
let set_b: HashSet<&str> = list_b.into_iter().collect();

let common: Vec<&&str> = set_a.intersection(&set_b).collect();
println!("共同用户: {:?}", common);
// 共同用户: ["charlie", "dave"]  （顺序不确定）

// 保持去重后的顺序？HashSet 做不到
// 方案：用 Vec + seen HashSet
fn dedup_ordered<T: Eq + std::hash::Hash + Clone>(input: Vec<T>) -> Vec<T> {
    let mut seen = HashSet::new();
    input.into_iter()
         .filter(|x| seen.insert(x.clone()))
         .collect()
}

let data = vec![3, 1, 4, 1, 5, 9, 2, 6, 5, 3];
let unique = dedup_ordered(data);
// [3, 1, 4, 5, 9, 2, 6]  —— 保持首次出现顺序

⚠ HashSet 不保证遍历顺序。如果需要"去重但保持插入顺序"，可用 indexmap crate 的 IndexSet。

04VecDeque 与 BTreeMap

🎯 为什么需要了解这个

Vec 和 HashMap 覆盖了大部分场景，但并非万能。当你需要高效的双端队列操作或有序的键值映射时，标准库还提供了 VecDeque 和 BTreeMap。了解何时选用它们，可以避免性能陷阱。

🔍 核心思路
类型底层结构核心优势key 约束
VecDeque<T>环形缓冲区两端 O(1) push/pop无
BTreeMap<K,V>B-Treekey 有序，范围查询Ord
BTreeSet<T>B-Tree有序集合Ord
📐 规则与语法
RUSTvecdeque.rs
use std::collections::VecDeque;

let mut dq: VecDeque<i32> = VecDeque::new();

// 两端操作都是 O(1)
dq.push_back(1);    // [1]
dq.push_back(2);    // [1, 2]
dq.push_front(0);   // [0, 1, 2]

let front = dq.pop_front();  // Some(0) → [1, 2]
let back = dq.pop_back();    // Some(2) → [1]

// 典型应用：BFS 队列
// while let Some(node) = queue.pop_front() {
//     for neighbor in node.neighbors() {
//         queue.push_back(neighbor);
//     }
// }
RUSTbtreemap.rs
use std::collections::BTreeMap;

let mut scores = BTreeMap::new();
scores.insert("Charlie", 78);
scores.insert("Alice", 95);
scores.insert("Bob", 87);

// 遍历时按 key 的自然顺序！
for (name, score) in &scores {
    println!("{}: {}", name, score);
}
// Alice: 95
// Bob: 87
// Charlie: 78

// 范围查询
for (name, score) in scores.range("A".."C") {
    println!("{}: {}", name, score);
}
// Alice: 95
// Bob: 87
输出
Alice: 95
Bob: 87
Charlie: 78
🌐 工程实践

集合类型选择速查：

需求推荐类型
可增长列表Vec<T>
FIFO 队列 / 双端操作VecDeque<T>
快速 key-value 查找HashMap<K,V>
有序 key-value + 范围查询BTreeMap<K,V>
唯一值集合HashSet<T>
有序唯一值集合BTreeSet<T>
✏️ 示例分析与易错点
RUSTchoice.rs
// ❌ 用 Vec 做队列是低效的
let mut queue = vec![1, 2, 3];
let first = queue.remove(0);  // O(n)！所有元素前移

// ✅ 用 VecDeque 做队列
use std::collections::VecDeque;
let mut queue = VecDeque::from([1, 2, 3]);
let first = queue.pop_front();  // O(1)

// ❌ 用 HashMap 然后排序 key
use std::collections::HashMap;
let mut map = HashMap::new();
map.insert(3, "c");
map.insert(1, "a");
let mut pairs: Vec<_> = map.into_iter().collect();
pairs.sort_by_key(|&(k, _)| k);  // 额外排序

// ✅ 直接用 BTreeMap
use std::collections::BTreeMap;
let mut map = BTreeMap::new();
map.insert(3, "c");
map.insert(1, "a");
// 遍历自动有序，无需排序

05迭代器消费与收集

🎯 为什么需要了解这个

Rust 的集合与迭代器是一体两面的。几乎所有集合操作（过滤、映射、累加）都通过迭代器链来表达。iter()、into_iter()、iter_mut() 三兄弟的区别是 Rust 新手最常困惑的点之一。

🔍 核心思路
方法消耗集合？产生的元素类型典型场景
iter()否（不可变借用）&T只读遍历
iter_mut()否（可变借用）&mut T原地修改元素
into_iter()是（获取所有权）T消耗集合

collect() 是迭代器的"汇合点"——把迭代器的元素收集成任意实现了 FromIterator 的类型（Vec、HashMap、String 等）。

📐 规则与语法
RUSTthree_iters.rs
let mut v = vec![1, 2, 3, 4, 5];

// iter() —— 借用，遍历后 v 还能用
let sum: i32 = v.iter().sum();
println!("sum = {}, v = {:?}", sum, v);  // v 还在

// iter_mut() —— 可变借用，原地修改
for x in v.iter_mut() {
    *x *= 2;
}
println!("{:?}", v);  // [2, 4, 6, 8, 10]

// into_iter() —— 消耗 v
let doubled: Vec<i32> = v.into_iter().map(|x| x + 1).collect();
println!("{:?}", doubled);  // [3, 5, 7, 9, 11]
// println!("{:?}", v);     // ❌ v 已被消耗
输出
sum = 15, v = [1, 2, 3, 4, 5]
[2, 4, 6, 8, 10]
[3, 5, 7, 9, 11]

collect 的多种目标类型：

RUSTcollect_targets.rs
use std::collections::{HashMap, HashSet};

let data = vec![1, 2, 3, 2, 1];

// 收集为 Vec
let v: Vec<i32> = data.iter().cloned().collect();

// 收集为 HashSet（自动去重）
let s: HashSet<i32> = data.iter().cloned().collect();
// {1, 2, 3}

// 收集为 HashMap
let map: HashMap<i32, i32> = (0..5).map(|i| (i, i * i)).collect();
// {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

// 收集为 String
let chars = vec!['h', 'e', 'l', 'l', 'o'];
let s: String = chars.into_iter().collect();
// "hello"

// 收集 Result 迭代器 → Result<Vec, E>
let results: Vec<Result<i32, &str>> = vec![Ok(1), Ok(2), Ok(3)];
let all: Result<Vec<i32>, &str> = results.into_iter().collect();
// Ok([1, 2, 3])  — 遇到第一个 Err 就短路
🌐 工程实践
  • for 循环的隐式调用：for x in &v 等价于 for x in v.iter()；for x in v 等价于 for x in v.into_iter()
  • 惰性求值：map、filter 等只是构建链，不真正执行。需要 collect、sum、for_each 等消费者驱动执行
  • turbofish 语法：.collect::<Vec<_>>() 用于编译器无法自动推断目标类型时
  • 链式 vs 循环：简单逻辑用迭代器链更惯用，复杂逻辑用 for 更清晰
✏️ 示例分析与易错点
RUSTiter_chain.rs
let words = vec!["hello", "world", "rust", "is", "great"];

// 实际场景：过滤 + 转换 + 收集
let long_words: Vec<String> = words.iter()
    .filter(|w| w.len() > 3)         // 保留长度 > 3 的
    .map(|w| w.to_uppercase())        // 转大写
    .collect();
// ["HELLO", "WORLD", "RUST", "GREAT"]

// 注意 iter() 产生 &&str
// filter 的闭包参数是 &&&str（双重引用），需要 **w 或模式匹配
let count = words.iter()
    .filter(|&&w| w.starts_with('r'))  // 模式匹配解引用
    .count();
// ❌ 新手常忘记处理多层引用

// ✅ 更清晰的写法
let count = words.iter()
    .filter(|w| w.starts_with('r'))    // &&str 可以直接调 str 的方法
    .count();

⚠ iter() 对 Vec<T> 产生 &T。如果 T 本身是引用（如 Vec<&str>），则 iter() 产生 &&str。注意多层引用的处理。
