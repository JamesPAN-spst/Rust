Unit 3 · 标准库与项目实践

I/O 与文件

Rust 的 I/O 系统建立在 Read 和 Write 两个 trait 之上，通过统一的接口抽象文件、网络、标准输入输出等不同数据源。配合 Path/PathBuf 的跨平台路径抽象，你可以写出既安全又可移植的 I/O 代码。

01Read / Write trait

🎯 为什么需要了解这个

文件、网络套接字、标准输入、内存缓冲区——它们的底层完全不同，但"读数据"和"写数据"的基本操作却是一样的。Rust 用 Read 和 Write 两个 trait 统一了所有 I/O 操作，让你可以写出不依赖具体数据源的通用代码。

🔍 核心思路

核心 trait 层级：

  • Read：fn read(&mut self, buf: &mut [u8]) -> Result<usize> — 基础读取
  • BufRead：扩展 Read，增加 read_line()、lines() 等缓冲读取方法
  • Write：fn write(&mut self, buf: &[u8]) -> Result<usize> + fn flush(&mut self)

实现了这些 trait 的类型：File、TcpStream、Stdin/Stdout、BufReader、BufWriter、Vec<u8> 等。

📐 规则与语法
RUSTread_write_trait.rs
use std::io::{self, Read, Write};

// 通用函数：任何实现了 Read 的类型都行
fn count_bytes(reader: &mut impl Read) -> io::Result<usize> {
    let mut buf = Vec::new();
    let bytes = reader.read_to_end(&mut buf)?;
    Ok(bytes)
}

// 通用函数：任何实现了 Write 的类型都行
fn write_greeting(writer: &mut impl Write) -> io::Result<()> {
    writer.write_all(b"Hello, Rust!\n")?;
    writer.flush()?;
    Ok(())
}

fn main() -> io::Result<()> {
    // 写入文件
    let mut file = std::fs::File::create("output.txt")?;
    write_greeting(&mut file)?;

    // 写入 Vec（内存缓冲区）
    let mut buf: Vec<u8> = Vec::new();
    write_greeting(&mut buf)?;
    println!("缓冲区: {:?}", String::from_utf8(buf));

    Ok(())
}
RUSTread_methods.rs
use std::io::Read;

let mut file = std::fs::File::open("data.txt")?;

// read：读入固定大小缓冲区
let mut buf = [0u8; 1024];
let n = file.read(&mut buf)?;    // 返回实际读取的字节数

// read_to_string：一次性读成 String
let mut content = String::new();
file.read_to_string(&mut content)?;

// read_to_end：一次性读成 Vec<u8>
let mut bytes = Vec::new();
file.read_to_end(&mut bytes)?;

// read_exact：精确读取指定字节数（不够则报错）
let mut header = [0u8; 4];
file.read_exact(&mut header)?;
🌐 工程实践
  • 泛型 I/O：用 impl Read / impl Write 作为参数，让函数可测试——测试时传入 Vec、生产时传入 File
  • read 可能短读：read() 不保证填满缓冲区。需要读完所有数据用 read_to_end 或 read_exact
  • write_all vs write：write() 可能短写，write_all() 保证写完所有数据
  • 别忘了 flush：BufWriter 需要 flush 确保数据真正写入
✏️ 示例分析与易错点
RUSTtestable_io.rs
use std::io::{self, BufRead, Write};

// 通过 trait 抽象 I/O，方便测试
fn process_lines(
    reader: impl BufRead,
    mut writer: impl Write,
) -> io::Result<usize> {
    let mut count = 0;
    for line in reader.lines() {
        let line = line?;
        writeln!(writer, "处理: {}", line.to_uppercase())?;
        count += 1;
    }
    Ok(count)
}

// 生产环境
fn main() -> io::Result<()> {
    let file = std::fs::File::open("input.txt")?;
    let reader = io::BufReader::new(file);
    let stdout = io::stdout();
    process_lines(reader, stdout.lock())?;
    Ok(())
}

// 测试——使用内存缓冲区代替真实文件
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_process_lines() {
        let input = b"hello\nworld\n";
        let mut output = Vec::new();
        let count = process_lines(&input[..], &mut output).unwrap();
        assert_eq!(count, 2);
        let result = String::from_utf8(output).unwrap();
        assert!(result.contains("HELLO"));
    }
}

⚠ read() 返回 0 字节表示 EOF（文件结束），不是错误。循环读取时要检查返回值。

02文件读写

🎯 为什么需要了解这个

文件操作是最常见的 I/O 场景。Rust 提供了从一行搞定到精细控制的多层 API。理解何时用便捷函数（fs::read_to_string）、何时用底层控制（BufReader + OpenOptions），是编写高效 I/O 代码的关键。

🔍 核心思路
需求便捷 API精细 API
读全部文本fs::read_to_stringFile::open + read_to_string
读全部字节fs::readFile::open + read_to_end
逐行读取—BufReader::new(file).lines()
写入/覆盖fs::writeFile::create + write_all
追加写入—OpenOptions::append(true)
📐 规则与语法
RUSTfile_read.rs
use std::fs;
use std::io::{self, BufRead, BufReader};

// === 一行搞定：读取整个文件 ===
let content: String = fs::read_to_string("config.toml")?;
let bytes: Vec<u8> = fs::read("image.png")?;

// === 逐行读取（大文件推荐） ===
let file = fs::File::open("large_log.txt")?;
let reader = BufReader::new(file);

for line_result in reader.lines() {
    let line = line_result?;  // 每行可能产生 io::Error
    if line.contains("ERROR") {
        println!("{}", line);
    }
}

// === 读取固定大小的块 ===
let mut file = fs::File::open("data.bin")?;
let mut buf = [0u8; 4096];
loop {
    let n = file.read(&mut buf)?;
    if n == 0 { break; }  // EOF
    process_chunk(&buf[..n]);
}
RUSTfile_write.rs
use std::fs::{self, File, OpenOptions};
use std::io::{self, Write, BufWriter};

// === 一行搞定：写入文件（创建/覆盖） ===
fs::write("output.txt", "Hello, World!")?;

// === 使用 File::create + BufWriter 高效写入 ===
let file = File::create("output.txt")?;
let mut writer = BufWriter::new(file);
for i in 0..1000 {
    writeln!(writer, "行 {}: 数据", i)?;
}
writer.flush()?;  // 确保所有数据写入

// === 追加写入 ===
let mut file = OpenOptions::new()
    .append(true)
    .create(true)  // 不存在则创建
    .open("log.txt")?;
writeln!(file, "[2026-04-10] 新事件")?;

// === OpenOptions 完全控制 ===
let file = OpenOptions::new()
    .read(true)
    .write(true)
    .create(true)
    .truncate(false)  // 不截断
    .open("data.db")?;
🌐 工程实践
  • 小文件：fs::read_to_string / fs::write 最方便
  • 大文件：BufReader 逐行流式处理，避免内存爆炸
  • 频繁写入：BufWriter 减少系统调用，写完记得 flush()
  • 日志追加：OpenOptions::append(true).create(true)
  • File 的 Drop：离开作用域自动关闭，但 BufWriter 的 Drop 中 flush 的错误会被静默忽略——显式 flush 更安全
✏️ 示例分析与易错点
RUSTbuffered_trap.rs
use std::io::{BufWriter, Write};

// ❌ BufWriter drop 时 flush 失败是静默的
fn bad_write() -> std::io::Result<()> {
    let file = std::fs::File::create("out.txt")?;
    let mut writer = BufWriter::new(file);
    write!(writer, "Important data!")?;
    // writer 在这里 drop → 自动 flush
    // 但 flush 错误被静默忽略了！
    Ok(())
}

// ✅ 显式 flush 以捕获错误
fn good_write() -> std::io::Result<()> {
    let file = std::fs::File::create("out.txt")?;
    let mut writer = BufWriter::new(file);
    write!(writer, "Important data!")?;
    writer.flush()?;  // 显式 flush，错误会被 ? 传播
    Ok(())
}

⚠ BufWriter 在 drop 时会尝试 flush，但 Drop trait 不能返回 Result，所以 flush 失败会被静默忽略。对重要数据，务必显式调用 flush()。

03标准输入输出

🎯 为什么需要了解这个

CLI 程序的数据流动通过 stdin/stdout/stderr 三个标准流完成。掌握它们的 Rust API，是编写命令行工具和管道友好程序的基础。

🔍 核心思路
流方向获取方式用途
stdin输入io::stdin()用户输入、管道输入
stdout输出io::stdout() / println!正常输出（可重定向）
stderr输出io::stderr() / eprintln!错误/诊断信息（不重定向）

注意：println! 每次调用都锁定 stdout。高频输出场景应手动锁定一次。

📐 规则与语法
RUSTstdio.rs
use std::io::{self, BufRead, Write};

fn main() -> io::Result<()> {
    // === 读取一行输入 ===
    print!("请输入名字: ");
    io::stdout().flush()?;  // print! 不自动 flush

    let mut name = String::new();
    io::stdin().read_line(&mut name)?;
    let name = name.trim();  // 去掉末尾的换行符
    println!("你好, {}!", name);

    // === 逐行处理 stdin（管道友好） ===
    let stdin = io::stdin();
    for line in stdin.lock().lines() {
        let line = line?;
        println!(">> {}", line.to_uppercase());
    }

    // === 高频输出：手动锁定 stdout ===
    let stdout = io::stdout();
    let mut handle = stdout.lock();  // 只锁一次
    for i in 0..10000 {
        writeln!(handle, "行 {}", i)?;  // 避免每次 println! 都加锁
    }

    // === 错误输出到 stderr ===
    eprintln!("警告: 配置文件缺失，使用默认值");

    Ok(())
}
🌐 工程实践
  • print! 后要 flush：print! 不像 println! 自动换行和 flush
  • 高频输出锁定 stdout：stdout.lock() 避免每次输出都加锁解锁
  • 管道兼容：正常数据走 stdout，错误信息走 stderr。这样 program | grep 不会被错误信息干扰
  • read_line 追加：read_line 是追加到 String，循环读取前要清空
✏️ 示例分析与易错点
RUSTstdin_trap.rs
use std::io;

// ❌ read_line 是追加的！
let mut input = String::new();
io::stdin().read_line(&mut input)?;  // "hello\n"
io::stdin().read_line(&mut input)?;  // "hello\nworld\n"  ← 两行拼在一起了！

// ✅ 每次读取前清空
let mut input = String::new();
io::stdin().read_line(&mut input)?;
let first = input.trim().to_string();
input.clear();  // 清空！
io::stdin().read_line(&mut input)?;
let second = input.trim().to_string();

// ✅ 或者每次创建新 String
fn read_line() -> io::Result<String> {
    let mut buf = String::new();
    io::stdin().read_line(&mut buf)?;
    Ok(buf.trim().to_string())
}

⚠ read_line 会在 String 末尾追加（包含 \n）。循环读取时务必先 .clear() 或每次用新的 String。

04路径处理

🎯 为什么需要了解这个

文件路径在 Windows 和 Unix 上格式不同（\ vs /），且操作系统路径可能包含非 UTF-8 字符。Rust 用 Path / PathBuf 类型封装了这些跨平台细节，让路径操作既安全又可移植。

🔍 核心思路

路径类型对照：

字符串世界路径世界特点
&str&Path不可变借用，unsized
StringPathBuf拥有所有权，可增长
&OsStr—操作系统原生字符串
OsString—拥有的操作系统字符串

Path 内部用 OsStr（而非 str），因为操作系统路径不保证是有效 UTF-8。

📐 规则与语法
RUSTpath_ops.rs
use std::path::{Path, PathBuf};

// === 创建 ===
let p = Path::new("/home/user/doc.txt");
let mut pb = PathBuf::from("/home/user");

// === 组件操作 ===
println!("{:?}", p.file_name());    // Some("doc.txt")
println!("{:?}", p.extension());    // Some("txt")
println!("{:?}", p.stem());         // Some("doc")
println!("{:?}", p.parent());       // Some("/home/user")

// === 拼接路径 ===
pb.push("documents");              // /home/user/documents
pb.push("report.pdf");             // /home/user/documents/report.pdf

let joined = Path::new("/home").join("user").join("file.txt");
// /home/user/file.txt

// === 检查 ===
let exists = p.exists();
let is_file = p.is_file();
let is_dir = p.is_dir();
let is_abs = p.is_absolute();

// === 转换 ===
let s: Option<&str> = p.to_str();       // 可能失败（非 UTF-8 路径）
let s: String = p.display().to_string();  // 总是成功（有损转换）
let os: &std::ffi::OsStr = p.as_os_str();
RUSTpath_practical.rs
use std::path::Path;
use std::fs;

// 递归遍历目录
fn list_files(dir: &Path) -> std::io::Result<()> {
    if dir.is_dir() {
        for entry in fs::read_dir(dir)? {
            let entry = entry?;
            let path = entry.path();
            if path.is_dir() {
                list_files(&path)?;
            } else {
                println!("{}", path.display());
            }
        }
    }
    Ok(())
}

// 构建输出路径：修改扩展名
fn to_output_path(input: &Path) -> PathBuf {
    let mut output = input.to_path_buf();
    output.set_extension("out");  // foo.txt → foo.out
    output
}

// 安全拼接：join 处理绝对路径的特殊情况
let base = Path::new("/home/user");
let relative = Path::new("docs/file.txt");
let absolute = Path::new("/etc/config");

let p1 = base.join(relative);   // /home/user/docs/file.txt
let p2 = base.join(absolute);   // /etc/config  ← 绝对路径覆盖！
🌐 工程实践
  • 函数参数用 &Path：类似 &str，接受 &PathBuf 和路径字面值
  • 用 join 而非字符串拼接：join 自动处理分隔符和平台差异
  • 注意 join 的覆盖行为：如果参数是绝对路径，会覆盖前面的部分
  • display() 用于输出：path.display() 返回可 format 的对象，处理非 UTF-8 路径时做有损替换
✏️ 示例分析与易错点
RUSTpath_traps.rs
use std::path::Path;

// ❌ 字符串拼接路径——不可移植
let path = "/home/user".to_string() + "/" + "file.txt";
// Windows 上可能出问题

// ✅ 用 Path::join
let path = Path::new("/home/user").join("file.txt");

// ❌ to_str() 可能返回 None
let p = Path::new("/some/path");
let s: &str = p.to_str().unwrap();  // 对非 UTF-8 路径会 panic

// ✅ 安全处理
match p.to_str() {
    Some(s) => println!("路径: {}", s),
    None => println!("路径包含非 UTF-8 字符: {}", p.display()),
}

// ⚠ join 绝对路径的陷阱
let base = Path::new("/home");
let sub = Path::new("/etc/config");  // 注意：这是绝对路径
assert_eq!(base.join(sub), Path::new("/etc/config"));
// join 的参数如果是绝对路径，会直接替换（不是拼接）！

⚠ Path::join 的参数如果是绝对路径，结果就是那个绝对路径（而非拼接）。处理用户输入的路径时要特别注意这一点。
