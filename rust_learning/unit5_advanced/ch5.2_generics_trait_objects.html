Ch 5.2 泛型与 Trait 对象

泛型让你写一次代码、适用多种类型；Trait 对象让你在运行时处理不同类型。两者的核心区别在于"什么时候确定具体类型"——编译期还是运行期，这直接影响性能和灵活性的权衡。

🎯 学习目标

  • 掌握泛型函数和泛型结构体的声明与使用
  • 理解 Trait Bound、where 子句和 impl Trait 语法
  • 理解单态化（monomorphization）的原理与性能影响
  • 掌握 dyn Trait 动态分发的机制与使用场景
  • 明确对象安全（object safety）规则及其设计原因

📋 前置知识

需要理解 Trait 的基本概念（Unit 2.6）。泛型和 Trait 对象是 Trait 系统的延伸，如果对 Trait 定义和实现不熟悉，建议先回顾。

5.2.1 泛型函数与泛型结构体

🤔

你写了一个求最大值的函数，先是 i32 版本，然后发现 f64 也需要——难道要复制粘贴？

泛型用类型参数（通常是 <T>）抽象掉具体类型，让一份代码适用于多种类型。泛型函数在调用时由编译器推断或显式指定具体类型；泛型结构体则在实例化时确定。

正式规则：泛型参数写在函数名或类型名之后的尖括号中。没有约束的 T 几乎什么都做不了——通常需要加上 Trait Bound 来告诉编译器 T 能做什么。

Rustgenerics_basic.rs
// 泛型结构体
struct Pair<T> {
    first: T,
    second: T,
}

// 泛型函数：T 必须实现 PartialOrd 才能比较
fn largest<T: PartialOrd>(a: T, b: T) -> T {
    if a >= b { a } else { b }
}

fn main() {
    let pair = Pair { first: 10, second: 20 };
    println!("最大值: {}", largest(pair.first, pair.second));

    let pair_f = Pair { first: 1.5, second: 2.7 };
    println!("最大值: {}", largest(pair_f.first, pair_f.second));
}
输出 
最大值: 20
最大值: 2.7

5.2.2 Trait Bound、where 子句与 impl Trait

📐

当泛型约束变得又长又复杂时，代码可读性怎么办？

Trait Bound 有三种写法：内联语法 <T: Display + Clone>、where 子句（适合复杂约束）、以及 impl Trait（适合参数和返回值位置的简写）。它们表达的是同一件事：告诉编译器这个类型参数必须满足哪些能力。

使用建议：一两个约束用内联语法；三个以上或多个类型参数时用 where；函数参数/返回值中只出现一次的泛型用 impl Trait 更清晰。

Rusttrait_bounds.rs
use std::fmt::Display;

// 内联语法
fn print_pair<T: Display + Clone>(a: T, b: T) {
    println!("{} and {}", a, b);
}

// where 子句——复杂约束时更清晰
fn complex<T, U>(t: T, u: U) -> String
where
    T: Display + Clone,
    U: Display + Into<String>,
{
    format!("{} / {}", t, u)
}

// impl Trait 语法——参数和返回值位置
fn make_greeting(name: &str) -> impl Display {
    format!("Hello, {name}!")
}

fn main() {
    print_pair(1, 2);
    println!("{}", complex("rust", String::from("lang")));
    println!("{}", make_greeting("world"));
}
输出 
1 and 2
rust / lang
Hello, world!
返回值位置的 impl Trait 要求函数所有返回路径必须返回同一个具体类型——编译器需要在编译期确定大小。如果你需要根据条件返回不同类型，必须使用 Box<dyn Trait>。

5.2.3 单态化：编译器如何处理泛型

⚙️

泛型代码在运行时真的比手写每个类型的版本慢吗？

Rust 的泛型使用单态化（monomorphization）：编译器为每个实际使用的具体类型生成一份专用代码。这意味着泛型在运行时没有额外开销——和手写具体类型版本一样快。代价是编译时间和二进制体积可能增加。

正式规则：单态化 = 零运行时成本的泛型。fn foo<T>(x: T) 如果被 foo(1i32) 和 foo("hi") 调用，编译器会生成 foo_i32 和 foo_str 两个独立函数。

Rustmonomorphization.rs
fn identity<T>(x: T) -> T {
    x
}

fn main() {
    // 编译器生成：identity_i32, identity_f64, identity_string
    let a = identity(42);
    let b = identity(3.14);
    let c = identity(String::from("hello"));

    println!("{a}, {b}, {c}");
    // 运行时性能等同于手写三个函数
}
输出 
42, 3.14, hello

5.2.4 dyn Trait：运行时才知道具体类型

🎭

如果你想把不同类型的值放进同一个 Vec，泛型能做到吗？

dyn Trait 是 Trait 对象——在运行时通过胖指针（数据指针 + vtable 指针）实现动态分发。调用方法时需要查虚函数表，有一定开销，但换来了灵活性：你可以在同一个集合中存放实现了同一 Trait 的不同类型。

大小问题：dyn Trait 本身是一个动态大小类型（DST），不能直接放在栈上。必须通过 Box<dyn Trait>、&dyn Trait 或 Arc<dyn Trait> 使用。

Rustdyn_trait.rs
trait Drawable {
    fn draw(&self) -> String;
}

struct Circle;
struct Square;

impl Drawable for Circle {
    fn draw(&self) -> String { "●".to_string() }
}
impl Drawable for Square {
    fn draw(&self) -> String { "■".to_string() }
}

fn main() {
    // 不同类型放进同一个 Vec
    let shapes: Vec<Box<dyn Drawable>> = vec![
        Box::new(Circle),
        Box::new(Square),
        Box::new(Circle),
    ];

    for shape in &shapes {
        print!("{} ", shape.draw());
    }
    println!();
}
输出 
● ■ ●

静态分发（泛型/impl Trait）：编译期确定类型，零运行时开销，但编译产物更大。适合性能敏感路径。

动态分发（dyn Trait）：运行时查虚函数表，有间接调用开销，但代码更灵活、编译产物更小。适合插件系统、异构集合等场景。

5.2.5 对象安全：不是所有 Trait 都能做 dyn

🚧

为什么有些 Trait（比如 Clone）不能用作 dyn Clone？

要让一个 Trait 成为 Trait 对象（dyn Trait），它必须满足"对象安全"规则。核心约束是：方法不能返回 Self 类型，方法不能有泛型参数——因为动态分发时虚函数表必须有固定的函数签名，而 Self 和泛型都会让签名不确定。

对象安全三条核心规则：① 方法的返回类型不能是 Self；② 方法不能有泛型类型参数；③ Trait 本身不能要求 Self: Sized（或者用 where Self: Sized 排除特定方法）。

Rustobject_safety.rs
// ✅ 对象安全——所有方法签名确定
trait Describe {
    fn name(&self) -> &str;
    fn info(&self) -> String;
}

// ❌ 不对象安全——clone 返回 Self
// trait NotSafe: Clone {}

struct Dog;
impl Describe for Dog {
    fn name(&self) -> &str { "Dog" }
    fn info(&self) -> String { format!("I am a {}", self.name()) }
}

struct Cat;
impl Describe for Cat {
    fn name(&self) -> &str { "Cat" }
    fn info(&self) -> String { format!("I am a {}", self.name()) }
}

fn main() {
    let animals: Vec<Box<dyn Describe>> = vec![
        Box::new(Dog),
        Box::new(Cat),
    ];

    for a in &animals {
        println!("{}: {}", a.name(), a.info());
    }
}
输出 
Dog: I am a Dog
Cat: I am a Cat

📌 章节总结

  • 泛型用类型参数抽象具体类型，通过 Trait Bound 约束能力。
  • where 子句和 impl Trait 是 Trait Bound 的可读性语法糖。
  • 单态化让泛型运行时零开销，但增加编译时间和二进制体积。
  • dyn Trait 提供动态分发，适合异构集合和插件架构。
  • 对象安全规则限制了哪些 Trait 可以用作 Trait 对象——方法不能返回 Self 或有泛型参数。
