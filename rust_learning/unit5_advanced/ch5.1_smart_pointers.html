Ch 5.1 智能指针

智能指针不只是"包了一层的引用"——它们各自回答了一个核心所有权问题：谁来分配？谁来共享？谁来决定可变性？理解每种指针的使用时机，是写出地道 Rust 代码的关键。

01Box：最简单的堆分配

🎯 为什么需要了解这个

Rust 默认把数据放在栈上，但有两类场景必须用堆：递归类型（编译器无法计算无限嵌套的大小）和大型数据（栈空间有限）。Box 是解决这两个问题最直接的工具。

🔍 核心思路

Box<T> 做一件事：把 T 从栈搬到堆，栈上只留一个固定大小的指针（8 字节 on 64-bit）。

  • 独占所有权——一个 Box 只有一个主人，drop 时自动释放堆内存
  • Deref 透明——实现了 Deref<Target=T>，用起来和 &T 几乎一样
  • 零运行时开销——除了堆分配本身，没有引用计数等额外成本
📐 规则与语法
RUSTbox_basics.rs
// 基本用法
let x = Box::new(42);
println!("堆上的值: {x}");

// 递归类型
enum List {
    Cons(i32, Box<List>),
    Nil,
}

let list = List::Cons(1,
    Box::new(List::Cons(2,
        Box::new(List::Cons(3,
            Box::new(List::Nil))))));

// 大型数据的堆分配
let big = Box::new([0u8; 1_000_000]);
println!("堆上数组长度: {}", big.len());
输出
堆上的值: 42
堆上数组长度: 1000000
🌐 工程实践

Trait 对象：Box<dyn Trait> 是存储动态类型最常见的方式。

不要过度使用：普通小型数据放栈上更快，不要为了"安全感"给所有东西套 Box。

✏️ 示例分析与易错点
RUSTbox_trait_object.rs
trait Describe { fn desc(&self) -> String; }
struct Dog;
struct Cat;
impl Describe for Dog { fn desc(&self) -> String { String::from("🐕 Dog") } }
impl Describe for Cat { fn desc(&self) -> String { String::from("🐱 Cat") } }

fn make_pet(is_dog: bool) -> Box<dyn Describe> {
    if is_dog { Box::new(Dog) } else { Box::new(Cat) }
}

fn main() {
    let pet = make_pet(true);
    println!("{}", pet.desc());
}
输出
🐕 Dog

常见错误：

  • 忘了递归类型必须用 Box → 编译报 E0072 "recursive type has infinite size"
  • 混淆 Box 和 Rc：Box 是独占所有权，不能 clone 出多个持有者共享
⚡ 快速检测
为什么 enum List { Cons(i32, List), Nil } 不能编译？

02Rc：单线程共享所有权

🎯 为什么需要了解这个

有时一份数据需要被多个所有者持有——比如图结构中一个节点被多条边引用。Rc 是在单线程场景下打破"一个值只有一个主人"限制的工具。

🔍 核心思路

Rc<T>（Reference Counted）通过引用计数管理共享所有权：

  • 每次 Rc::clone() 引用计数 +1，每次 drop 引用计数 -1
  • 引用计数降到 0 时自动释放数据
  • Rc::clone() 不深拷贝——只是把一个整数加一
  • 不线程安全——引用计数加减是普通操作
📐 规则与语法
RUSTrc_basics.rs
use std::rc::Rc;

let data = Rc::new(vec![1, 2, 3]);
println!("count: {}", Rc::strong_count(&data)); // 1

let a = Rc::clone(&data);
println!("count: {}", Rc::strong_count(&data)); // 2

let b = Rc::clone(&data);
println!("count: {}", Rc::strong_count(&data)); // 3

drop(b);
println!("count: {}", Rc::strong_count(&data)); // 2
输出
count: 1
count: 2
count: 3
count: 2
🌐 工程实践

图/树结构：多个节点共享子节点时，Rc 比手动管理生命周期简单。

Weak 打破循环：用 Weak<T> 打破互相引用导致的内存泄漏。

不要跨线程：编译器会阻止——Rc 没有实现 Send。

✏️ 示例分析与易错点
RUSTrc_shared_tail.rs
use std::rc::Rc;

let shared_tail = Rc::new(vec![3, 4, 5]);
let list_a = (vec![1, 2], Rc::clone(&shared_tail));
let list_b = (vec![10, 20], Rc::clone(&shared_tail));

println!("list_a tail: {:?}", list_a.1);
println!("same data: {}", Rc::ptr_eq(&list_a.1, &list_b.1));
输出
list_a tail: [3, 4, 5]
same data: true

常见错误：循环引用导致泄漏、Rc 默认不可变（需组合 RefCell）。

⚡ 快速检测
Rc::clone() 的性能代价是什么？

03Arc：线程安全的共享所有权

🎯 为什么需要了解这个

多线程程序中经常需要多个线程读取同一份配置/共享同一个连接池。Rc 不能跨线程，Arc 是它的线程安全版本。

🔍 核心思路

Arc<T>（Atomic Reference Counted）和 Rc 接口几乎一样，区别在于引用计数使用原子操作。

  • 实现了 Send + Sync（当 T: Send + Sync 时）
  • 代价：原子操作比普通加减慢一点，但比深拷贝便宜得多
  • Arc 只解决"共享所有权"，修改数据需搭配 Mutex/RwLock
📐 规则与语法
RUSTarc_mutex.rs
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..5 {
        let counter = Arc::clone(&counter);
        handles.push(thread::spawn(move || {
            let mut val = counter.lock().unwrap();
            *val += 1;
        }));
    }
    for h in handles { h.join().unwrap(); }
    println!("最终计数: {}", *counter.lock().unwrap());
}
输出
最终计数: 5
🌐 工程实践

共享配置：Arc<Config> 让所有请求处理线程共享只读配置。

Arc vs Arc<Mutex>：只读共享不需要 Mutex。只有需要修改时才加。RwLock 适合读多写少。

✏️ 示例分析与易错点
RUSTarc_readonly.rs
use std::sync::Arc;
use std::thread;

fn main() {
    let config = Arc::new(String::from("production"));
    let mut handles = vec![];
    for i in 0..3 {
        let config = Arc::clone(&config);
        handles.push(thread::spawn(move || {
            println!("thread {i}: env = {config}");
        }));
    }
    for h in handles { h.join().unwrap(); }
}
输出
thread 0: env = production
thread 1: env = production
thread 2: env = production

常见错误：给只读数据加 Mutex、忘记 Arc::clone、Mutex 死锁。

⚡ 快速检测
多个线程只读一份配置，最合适的方案？

04RefCell：内部可变性

🎯 为什么需要了解这个

Rust 的借用规则很严格，但有些设计模式（缓存、观察者、mock 测试）需要在持有不可变引用时修改内部状态。RefCell 把借用检查推迟到运行时，为这类场景提供安全的逃生通道。

🔍 核心思路
  • borrow() → 运行时不可变引用 Ref<T>
  • borrow_mut() → 运行时可变引用 RefMut<T>
  • 违反规则 → 运行时 panic
  • 只用于单线程——没实现 Sync

核心权衡：编译期检查更安全，RefCell 把风险推到运行时。

📐 规则与语法
RUSTrefcell_basics.rs
use std::cell::RefCell;

let data = RefCell::new(vec![1, 2, 3]);

// 不可变借用
{
    let r = data.borrow();
    println!("data: {:?}", *r);
}   // r 在这里被 drop

// 可变借用
{
    let mut w = data.borrow_mut();
    w.push(4);
}
println!("after push: {:?}", data.borrow());
输出
data: [1, 2, 3]
after push: [1, 2, 3, 4]
🌐 工程实践

Rc<RefCell<T>>：单线程中"多个所有者可修改同一数据"。

Mock 测试：在 &self 方法中记录调用次数。

尽量少用：能用编译期检查解决的不要用 RefCell。

✏️ 示例分析与易错点
RUSTrc_refcell.rs
use std::cell::RefCell;
use std::rc::Rc;

let shared = Rc::new(RefCell::new(vec![1, 2]));
let a = Rc::clone(&shared);
let b = Rc::clone(&shared);

a.borrow_mut().push(3);
b.borrow_mut().push(4);

println!("data: {:?}", shared.borrow());
输出
data: [1, 2, 3, 4]

常见错误：忘记释放 Ref/RefMut（在同一作用域先 borrow 再 borrow_mut 会 panic）。

05Cell 与 Cow

🎯 为什么需要了解这个

RefCell 有运行时开销和 panic 风险。Cell 处理 Copy 类型的简单可变性，Cow 处理"通常只读、偶尔修改"的场景——都更轻量。

🔍 核心思路

Cell<T>：整体 get/set，不返回引用 → 无借用冲突 → 零额外开销（只适合 Copy 类型）。

Cow<'a, B>：以借用开始，修改时才 clone。适合"大多数时候只读"的场景。

📐 规则与语法
RUSTcell_cow.rs
use std::cell::Cell;
use std::borrow::Cow;

// Cell: 不需要 mut
let counter = Cell::new(0);
counter.set(counter.get() + 1);
counter.set(counter.get() + 1);
println!("counter: {}", counter.get());

// Cow: 读时借用，写时克隆
fn maybe_upper(s: &str, up: bool) -> Cow<str> {
    if up { Cow::Owned(s.to_uppercase()) }
    else  { Cow::Borrowed(s) }
}
let a = maybe_upper("hello", false); // 零拷贝
let b = maybe_upper("hello", true);  // 克隆
println!("a: {a}, b: {b}");
输出
counter: 2
a: hello, b: HELLO
🌐 工程实践

Cell 适合计数器/标志：在 &self 内递增 i32 比 RefCell 轻量。

Cow 在解析/转换中常见：配置字符串大多原样使用，少数需替换——用 Cow 避免无谓克隆。

✏️ 示例分析与易错点
RUSTcow_to_mut.rs
use std::borrow::Cow;

let mut text: Cow<str> = Cow::Borrowed("hello");
println!("is_borrowed: {}", matches!(text, Cow::Borrowed(_)));

text.to_mut().push_str(" world"); // 触发克隆
println!("is_owned: {}", matches!(text, Cow::Owned(_)));
println!("result: {text}");
输出
is_borrowed: true
is_owned: true
result: hello world

常见错误：Cell 的 get() 要求 T: Copy（不能用于 String）；Cow 只在第一次 to_mut() 时克隆。
