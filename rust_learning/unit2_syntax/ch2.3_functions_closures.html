Unit 2 · 基础语法

函数与闭包

函数是代码组织的基本单元；闭包是能捕获环境的匿名函数。掌握 fn 签名规则和三种闭包 trait（Fn/FnMut/FnOnce），是理解 Rust 高阶编程和迭代器链的基础。

01fn 签名与返回值 — 表达式就是返回

🎯 为什么需要了解这个

Rust 函数签名是编译器和程序员之间的"合同"——参数类型和返回类型必须显式标注。函数体内，最后一个表达式（无分号）就是返回值——不需要写 return。这个设计看似小事，但它影响你对"语句 vs 表达式"的理解。

🔍 核心思路
  • 参数：必须标注类型，fn foo(x: i32, y: &str)。
  • 返回类型：用 -> T 标注。省略则返回 ()。
  • 返回值：函数体最后一个表达式（无分号）= 返回值。return 用于提前返回。
  • 发散函数：返回 !（never type），表示永不返回——如 panic!、无限循环。
📐 规则与语法
RUSTfunctions.rs
// 基本函数
fn add(a: i32, b: i32) -> i32 {
    a + b      // 没有分号 → 这是返回值
}

// 没有返回值（返回 ()）
fn greet(name: &str) {
    println!("Hello, {}!", name);
}

// 提前返回
fn abs(x: i32) -> i32 {
    if x < 0 {
        return -x;  // 提前返回
    }
    x               // 正常返回
}

// 发散函数
fn forever() -> ! {
    loop {
        // 永远不返回
    }
}

fn main() {
    println!("3 + 4 = {}", add(3, 4));
    greet("Rustacean");
    println!("|−5| = {}", abs(-5));
}
输出
3 + 4 = 7
Hello, Rustacean!
|−5| = 5
🌐 工程实践
  • 优先省略 return——最后一行表达式做返回值是惯用写法。只在需要提前退出时才用 return。
  • 小心分号——fn foo() -> i32 { 42; } 编译错误！尾部有分号变成了返回 ()。
  • ! 类型可以当任意类型用——let x: i32 = if cond { 5 } else { panic!("") }; 合法，因为 panic! 返回 !，! 可以强转为 i32。
✏️ 示例分析与易错点
RUSTsemicolon_trap.rs
// ❌ 经典错误
// fn double(x: i32) -> i32 {
//     x * 2;  // 分号使这成为语句，函数返回 ()
// }           // error: expected `i32`, found `()`

// ✅ 正确
fn double(x: i32) -> i32 {
    x * 2   // 无分号 → 表达式作返回值
}

// 发散函数在 match 中的用途
fn parse_or_die(s: &str) -> i32 {
    match s.parse::<i32>() {
        Ok(n) => n,
        Err(_) => panic!("无法解析: {}", s), // ! 可以当 i32
    }
}

fn main() {
    println!("double(21) = {}", double(21));
    println!("parse = {}", parse_or_die("42"));
}
输出
double(21) = 42
parse = 42
易错：函数签名中参数类型不能省略（不像闭包）。fn add(a, b) { a + b } 编译错误——必须写 fn add(a: i32, b: i32) -> i32。

02闭包语法与环境捕获

🎯 为什么需要了解这个

闭包是 Rust 中最常用的工具之一——你会在迭代器链 (.map())、排序 (.sort_by())、线程 (thread::spawn()) 中大量使用它们。区别于普通函数，闭包的超能力是捕获环境变量。理解它如何捕获、以什么方式捕获，是用好闭包的关键。

🔍 核心思路

闭包 = 匿名函数 + 环境捕获。语法：|params| body。

  • 参数类型可以省略（编译器推断），函数不能省略。
  • 捕获方式由编译器自动选择——尽量选"最低权限"：&T > &mut T > T。
  • move 关键字强制所有捕获都是"按值"（取走所有权）。
📐 规则与语法
RUSTclosures.rs
fn main() {
    // 基本语法
    let add = |a: i32, b: i32| -> i32 { a + b };  // 完整写法
    let add = |a, b| a + b;                         // 省略写法
    println!("add(3,4) = {}", add(3, 4));

    // 环境捕获——不可变引用
    let name = String::from("Rust");
    let greet = || println!("Hello, {}!", name);    // 捕获 &name
    greet();
    println!("name still usable: {}", name);        // ✅ name 未被 move

    // 环境捕获——可变引用
    let mut count = 0;
    let mut increment = || { count += 1; };         // 捕获 &mut count
    increment();
    increment();
    println!("count = {}", count);                   // 2

    // move 闭包——获取所有权
    let data = vec![1, 2, 3];
    let owns_data = move || println!("{:?}", data);  // move data 进闭包
    owns_data();
    // println!("{:?}", data); // ❌ data 已被 move
}
输出
add(3,4) = 7
Hello, Rust!
name still usable: Rust
count = 2
[1, 2, 3]
🌐 工程实践
  • 迭代器链——vec.iter().filter(|x| **x > 5).map(|x| x * 2)——闭包是迭代器的灵魂。
  • 线程必须 move——thread::spawn(move || { ... })，因为新线程可能比当前作用域活得更久。
  • 避免不必要的 move——让编译器自动选择最低权限的捕获方式，只在跨线程或返回闭包时才 move。
✏️ 示例分析与易错点
RUSTcapture_analysis.rs
fn main() {
    let mut s = String::from("hello");

    // 闭包 1：只读取 s → 捕获 &s
    let len = || s.len();
    println!("len = {}", len());
    println!("s = {}", s);  // ✅ s 未被 move

    // 闭包 2：修改 s → 捕获 &mut s
    let mut append = || s.push_str(" world");
    append();
    // println!("{}", s); // ❌ 在 append 的借用期间不能读 s
    drop(append);         // 结束借用
    println!("s = {}", s); // ✅ 现在可以了

    // 闭包 3：消耗 s → 捕获 s (move)
    let consume = || drop(s);
    consume();
    // consume(); // ❌ FnOnce，只能调用一次
    // println!("{}", s); // ❌ s 已被 move
}
输出
len = 5
s = hello
s = hello world
易错：闭包持有 &mut 引用时，原变量在闭包活着的期间不能被访问。用 drop(closure) 或让闭包离开作用域来释放借用。
⚡ 快速检测
闭包 || println!("{}", x) 会以什么方式捕获 x？

03Fn / FnMut / FnOnce — 三种闭包 trait

🎯 为什么需要了解这个

当你写一个接受闭包的函数时，怎么声明参数类型？Rust 用三个 trait 来分类闭包的行为——Fn、FnMut、FnOnce。选错了 trait，你的 API 要么太严格（限制了调用者），要么太宽松（无法满足需求）。

🔍 核心思路
Traitself 参数含义可调用次数
FnOnceself获取闭包的所有权1 次
FnMut&mut self可变借用闭包多次
Fn&self不可变借用闭包多次

继承关系：Fn: FnMut: FnOnce。满足 Fn 的一定满足 FnMut 和 FnOnce。

选择原则：函数参数用最宽松的 trait（FnOnce 最宽松、Fn 最严格）。要多次调用就至少用 FnMut。

📐 规则与语法
RUSTfn_traits.rs
// 接受 FnOnce：最灵活，但只能调用一次
fn call_once<F: FnOnce() -> String>(f: F) -> String {
    f()
}

// 接受 FnMut：可以多次调用
fn call_twice<F: FnMut()>(mut f: F) {
    f();
    f();
}

// 接受 Fn：可以多次调用，且闭包不修改状态
fn call_many<F: Fn() -> i32>(f: F) {
    for _ in 0..3 {
        println!("result: {}", f());
    }
}

fn main() {
    // FnOnce：消耗捕获的变量
    let name = String::from("world");
    let greeting = call_once(move || format!("Hello, {}!", name));
    println!("{}", greeting);

    // FnMut：修改捕获的变量
    let mut total = 0;
    call_twice(|| { total += 10; });
    println!("total = {}", total);

    // Fn：只读取
    let x = 42;
    call_many(|| x);
}
输出
Hello, world!
total = 20
result: 42
result: 42
result: 42
🌐 工程实践
  • API 设计原则：参数用 FnOnce（最灵活，谁都能传）。只在需要多次调用时升级到 FnMut 或 Fn。
  • 标准库示例：Option::map 接受 FnOnce（只调用一次），Iterator::for_each 接受 FnMut。
  • 普通函数也满足 Fn trait——fn double(x: i32) -> i32 { x * 2 } 可以传给接受 Fn(i32)->i32 的参数。
✏️ 示例分析与易错点
RUSTtrait_mismatch.rs
fn repeat_fn<F: Fn()>(f: F, n: usize) {
    for _ in 0..n {
        f();
    }
}

fn main() {
    let msg = String::from("hi");

    // ✅ 只读捕获，满足 Fn
    repeat_fn(|| println!("{}", msg), 3);

    // ❌ 修改捕获变量，只满足 FnMut，不满足 Fn
    // let mut count = 0;
    // repeat_fn(|| { count += 1; }, 3); // error: cannot borrow as mutable

    // ❌ 消耗捕获变量，只满足 FnOnce
    // let data = vec![1, 2, 3];
    // repeat_fn(|| { drop(data); }, 3); // error: cannot move out
}
输出
hi
hi
hi
易错：把 FnMut 闭包传给需要 Fn 的参数——编译错误。但反过来没问题：Fn 闭包可以传给 FnMut 或 FnOnce 参数。
⚡ 快速检测
如果函数需要多次调用闭包且闭包会修改状态，应该用哪个 trait bound？

04闭包作为参数和返回值

🎯 为什么需要了解这个

高阶函数——接受函数作参数或返回函数——是现代编程的核心模式。Rust 中传递闭包有两种分发方式：静态分发（impl Fn/泛型）和动态分发（dyn Fn）。选择哪种影响性能、灵活性和编译时间。

🔍 核心思路
方式语法分发适用场景
泛型参数fn foo<F: Fn()>(f: F)静态（编译时单态化）性能敏感，常见
impl Trait 参数fn foo(f: impl Fn())静态泛型的语法糖
trait 对象fn foo(f: &dyn Fn())动态（vtable）需要异构集合
impl Trait 返回fn foo() -> impl Fn()静态返回闭包的常见方式
Box<dyn> 返回fn foo() -> Box<dyn Fn()>动态返回不同类型的闭包
📐 规则与语法
RUSThigher_order.rs
// 1. 闭包作为参数——静态分发
fn apply<F: Fn(i32) -> i32>(f: F, x: i32) -> i32 {
    f(x)
}

// 等价写法
fn apply2(f: impl Fn(i32) -> i32, x: i32) -> i32 {
    f(x)
}

// 2. 闭包作为返回值——impl Trait
fn make_adder(n: i32) -> impl Fn(i32) -> i32 {
    move |x| x + n
}

// 3. 动态分发——trait 对象
fn apply_boxed(f: &dyn Fn(i32) -> i32, x: i32) -> i32 {
    f(x)
}

// 4. 返回不同闭包需要 Box<dyn>
fn make_op(op: &str) -> Box<dyn Fn(i32, i32) -> i32> {
    match op {
        "add" => Box::new(|a, b| a + b),
        "mul" => Box::new(|a, b| a * b),
        _     => Box::new(|a, b| a - b),
    }
}

fn main() {
    println!("{}", apply(|x| x * 2, 21));
    println!("{}", apply2(|x| x + 10, 32));

    let add5 = make_adder(5);
    println!("add5(10) = {}", add5(10));

    let op = make_op("mul");
    println!("mul(6, 7) = {}", op(6, 7));
}
输出
42
42
add5(10) = 15
mul(6, 7) = 42
🌐 工程实践
  • 参数优先用 impl Fn / 泛型——零开销抽象，编译器内联优化。
  • 返回单一闭包用 impl Fn——fn foo() -> impl Fn()。
  • 返回不同类型闭包用 Box<dyn Fn>——需要堆分配，但灵活。
  • 函数指针 fn(i32)->i32——当闭包不捕获环境时，可以用函数指针类型，更轻量。
✏️ 示例分析与易错点
RUSTstrategy_pattern.rs
struct Processor {
    strategy: Box<dyn Fn(i32) -> i32>,
}

impl Processor {
    fn new(strategy: Box<dyn Fn(i32) -> i32>) -> Self {
        Processor { strategy }
    }

    fn process(&self, value: i32) -> i32 {
        (self.strategy)(value)
    }
}

fn main() {
    let doubler = Processor::new(Box::new(|x| x * 2));
    let negator = Processor::new(Box::new(|x| -x));

    println!("double 21 = {}", doubler.process(21));
    println!("negate 42 = {}", negator.process(42));
}
输出
double 21 = 42
negate 42 = -42
易错：impl Fn 返回值时，函数内所有 return 路径必须返回同一个闭包类型。需要根据条件返回不同闭包时，必须用 Box<dyn Fn>。
