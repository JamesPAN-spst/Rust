Unit 2 · 基础语法

Struct 与 Enum

Struct 和 Enum 是 Rust 的复合类型基石。Struct 让你组合相关数据，Enum 让你表达"多选一"的状态。标准库的 Option 和 Result 就是 enum 的经典应用——它们让错误处理成为类型系统的一部分。

01struct 三种形式 — 命名、元组、单元

🎯 为什么需要了解这个

当基本类型不够用时，你需要自定义类型来组合数据。Rust 的 struct 提供了三种形式，各有适用场景。理解它们的差异，是设计清晰数据模型的第一步。

🔍 核心思路
形式语法适用场景
命名字段struct User { name: String, age: u32 }多字段数据，可读性最好
元组 structstruct Meters(f64)Newtype 模式、少量无名字段
单元 structstruct Marker;类型标记、空 trait 实现
📐 规则与语法
RUSTstructs.rs
// 1. 命名字段 struct
struct User {
    name: String,
    email: String,
    age: u32,
    active: bool,
}

// 2. 元组 struct（Newtype 模式）
struct Meters(f64);
struct Seconds(f64);
struct Color(u8, u8, u8);

// 3. 单元 struct
struct Marker;

fn main() {
    // 命名字段——必须指定所有字段
    let user = User {
        name: String::from("Alice"),
        email: String::from("alice@example.com"),
        age: 30,
        active: true,
    };
    println!("{} is {} years old", user.name, user.age);

    // struct update syntax——从另一个实例复制未指定的字段
    let user2 = User {
        name: String::from("Bob"),
        ..user  // 注意：user.email 被 move 了！
    };
    // println!("{}", user.email); // ❌ email 被 move
    println!("{}", user.age);     // ✅ u32 是 Copy

    // 元组 struct
    let distance = Meters(100.0);
    let duration = Seconds(9.58);
    println!("{} meters in {} seconds", distance.0, duration.0);
    // Meters 和 Seconds 是不同类型，不能混用！

    // 单元 struct
    let _m = Marker;
}
输出
Alice is 30 years old
30
100 meters in 9.58 seconds
🌐 工程实践
  • Newtype 模式——struct UserId(u64) 防止把 user_id 和 order_id 搞混。编译期类型安全，零运行时开销。
  • 字段可见性——struct 字段默认私有。跨模块访问需要 pub：pub struct Config { pub timeout: u64, ... }。
  • builder 模式——字段多且有默认值时，用 builder 模式替代冗长的构造。
✏️ 示例分析与易错点
RUSTnewtype.rs
struct UserId(u64);
struct OrderId(u64);

fn get_user(id: UserId) {
    println!("Getting user #{}", id.0);
}

fn main() {
    let uid = UserId(42);
    let oid = OrderId(100);

    get_user(uid);
    // get_user(oid); // ❌ 编译错误：expected UserId, found OrderId
    // 即使底层都是 u64，类型系统也不允许混用！
}
输出
Getting user #42
易错：..other struct update 语法会 move 非 Copy 字段。如果 other 有 String 字段被 move，后续不能再访问那些字段。Copy 字段（如 i32、bool）不受影响。

02impl 块与方法 — 给类型添加行为

🎯 为什么需要了解这个

Struct 只定义数据。要让类型"会做事"，需要在 impl 块中定义方法和关联函数。impl 是 Rust 中"数据 + 行为"的组合方式，类似其他语言中的方法定义，但没有继承。

🔍 核心思路
self 参数含义调用后原变量
&self不可变借用仍可用
&mut self可变借用仍可用
self获取所有权不可用（被 move）
（无 self）关联函数—
📐 规则与语法
RUSTimpl_block.rs
struct Rectangle {
    width: f64,
    height: f64,
}

impl Rectangle {
    // 关联函数（没有 self）——类似"构造器"
    fn new(w: f64, h: f64) -> Self {
        Rectangle { width: w, height: h }
    }

    fn square(size: f64) -> Self {
        Rectangle { width: size, height: size }
    }

    // 方法——&self 只读
    fn area(&self) -> f64 {
        self.width * self.height
    }

    fn perimeter(&self) -> f64 {
        2.0 * (self.width + self.height)
    }

    // 方法——&mut self 修改
    fn scale(&mut self, factor: f64) {
        self.width *= factor;
        self.height *= factor;
    }

    // 方法——self 获取所有权
    fn into_square(self) -> Rectangle {
        let side = (self.width * self.height).sqrt();
        Rectangle { width: side, height: side }
    }
}

fn main() {
    let mut rect = Rectangle::new(10.0, 5.0);  // 关联函数
    println!("area = {}", rect.area());          // 方法
    println!("perimeter = {}", rect.perimeter());

    rect.scale(2.0);                             // &mut self
    println!("scaled area = {}", rect.area());

    let sq = rect.into_square();                 // self (move)
    println!("square side = {}", sq.width);
    // println!("{}", rect.area()); // ❌ rect 已被 move
}
输出
area = 50
perimeter = 30
scaled area = 200
square side = 14.142135623730951
🌐 工程实践
  • new() 命名约定——关联函数 Type::new(...) 是 Rust 社区的构造器惯例。
  • 方法链——返回 &mut Self 支持链式调用：builder.set_name("x").set_age(20).build()。
  • 多个 impl 块——同一类型可以有多个 impl 块，按功能分组（如泛型 impl 和普通 impl 分开）。
  • Self 关键字——在 impl 块内，Self 是当前类型的别名。
✏️ 示例分析与易错点
RUSTmethod_chain.rs
struct QueryBuilder {
    table: String,
    conditions: Vec<String>,
    limit: Option<usize>,
}

impl QueryBuilder {
    fn new(table: &str) -> Self {
        QueryBuilder {
            table: table.to_string(),
            conditions: Vec::new(),
            limit: None,
        }
    }

    fn where_clause(&mut self, cond: &str) -> &mut Self {
        self.conditions.push(cond.to_string());
        self  // 返回 &mut Self 实现链式调用
    }

    fn limit(&mut self, n: usize) -> &mut Self {
        self.limit = Some(n);
        self
    }

    fn build(&self) -> String {
        let mut q = format!("SELECT * FROM {}", self.table);
        if !self.conditions.is_empty() {
            q.push_str(&format!(" WHERE {}", self.conditions.join(" AND ")));
        }
        if let Some(n) = self.limit {
            q.push_str(&format!(" LIMIT {}", n));
        }
        q
    }
}

fn main() {
    let query = QueryBuilder::new("users")
        .where_clause("age > 18")
        .where_clause("active = true")
        .limit(10)
        .build();
    println!("{}", query);
}
输出
SELECT * FROM users WHERE age > 18 AND active = true LIMIT 10
易错：self 方法会消耗实例。如果你不想转移所有权，用 &self 或 &mut self。
⚡ 快速检测

03enum 与代数数据类型

🎯 为什么需要了解这个

C/Java 的 enum 只是命名整数。Rust 的 enum 是代数数据类型（ADT）——每个变体可以携带不同数据。这让你能精确建模"多种可能之一"的状态，配合模式匹配，编译器保证你处理所有情况。

🔍 核心思路

Rust 的 enum = sum type（和类型）。变体可以是：

  • 无数据：Quit
  • 元组数据：Move(i32, i32)
  • 命名字段：Resize { w: u32, h: u32 }

内存布局：tag（判别值）+ 最大变体的空间。编译器会做优化（如 null pointer optimization）。

📐 规则与语法
RUSTenums.rs
// 变体可以携带不同类型的数据
enum Message {
    Quit,                          // 无数据
    Echo(String),                  // 元组数据
    Move { x: i32, y: i32 },      // 命名字段
    ChangeColor(u8, u8, u8),      // 多个元组字段
}

impl Message {
    fn process(&self) {
        match self {
            Message::Quit => println!("退出"),
            Message::Echo(text) => println!("回显: {}", text),
            Message::Move { x, y } => println!("移动到 ({}, {})", x, y),
            Message::ChangeColor(r, g, b) => println!("颜色: ({},{},{})", r, g, b),
        }
    }
}

fn main() {
    let msgs = vec![
        Message::Echo("hello".into()),
        Message::Move { x: 10, y: 20 },
        Message::ChangeColor(255, 0, 128),
        Message::Quit,
    ];

    for msg in &msgs {
        msg.process();
    }

    // enum 也可以有 impl
    println!("size: {} bytes", std::mem::size_of::<Message>());
}
输出
回显: hello
移动到 (10, 20)
颜色: (255,0,128)
退出
size: 32 bytes
🌐 工程实践
  • 状态机建模——enum State { Idle, Running(Progress), Done(Result) }，每种状态携带不同数据。
  • 消息/命令模式——enum Command { Start, Stop, SetSpeed(f64) }。
  • 替代 trait 对象——当变体有限时，enum dispatch 比 dyn trait 更高效（无虚表开销）。
  • 注意大小——enum 大小 = 最大变体 + tag。如果一个变体特别大，考虑 Box 它：LargeVariant(Box<[u8; 1024]>)。
✏️ 示例分析与易错点
RUSTshape.rs
enum Shape {
    Circle(f64),                     // 半径
    Rectangle { width: f64, height: f64 },
    Triangle(f64, f64, f64),         // 三条边
}

impl Shape {
    fn area(&self) -> f64 {
        match self {
            Shape::Circle(r) => std::f64::consts::PI * r * r,
            Shape::Rectangle { width, height } => width * height,
            Shape::Triangle(a, b, c) => {
                let s = (a + b + c) / 2.0;
                (s * (s - a) * (s - b) * (s - c)).sqrt()
            }
        }
    }
}

fn main() {
    let shapes: Vec<Shape> = vec![
        Shape::Circle(5.0),
        Shape::Rectangle { width: 10.0, height: 3.0 },
        Shape::Triangle(3.0, 4.0, 5.0),
    ];
    for shape in &shapes {
        println!("area = {:.2}", shape.area());
    }
}
输出
area = 78.54
area = 30.00
area = 6.00
易错：enum 的大小由最大变体决定。如果有一个变体特别大（如携带大数组），整个 enum 都会变大。可以用 Box 包裹大数据来减小 enum 体积。

04Option<T> — 空值的安全替代

🎯 为什么需要了解这个

Tony Hoare 称 null 是"十亿美元的错误"。Java 的 NullPointerException、C 的 segfault，都源于未检查的空值。Rust 没有 null——取而代之的是 Option<T>，它让"可能为空"成为类型的一部分，编译器强制你处理空值情况。

🔍 核心思路
RUSToption_def.rs
// 标准库定义（简化）
enum Option<T> {
    Some(T),  // 有值
    None,     // 无值
}

核心优势：如果函数返回 Option<T>，调用者必须处理 None——编译器不让你假装值一定存在。

📐 规则与语法
RUSToption_usage.rs
fn find_first_even(nums: &[i32]) -> Option<i32> {
    for &n in nums {
        if n % 2 == 0 {
            return Some(n);
        }
    }
    None
}

fn main() {
    let data = vec![1, 3, 5, 4, 7];

    // 1. match——最完整
    match find_first_even(&data) {
        Some(n) => println!("found: {}", n),
        None => println!("no even numbers"),
    }

    // 2. if let——只关心 Some
    if let Some(n) = find_first_even(&data) {
        println!("first even: {}", n);
    }

    // 3. unwrap_or——提供默认值
    let val = find_first_even(&[1, 3, 5]).unwrap_or(0);
    println!("value or default: {}", val);

    // 4. map——变换内部值
    let doubled = find_first_even(&data).map(|n| n * 2);
    println!("doubled: {:?}", doubled);

    // 5. ?——在返回 Option 的函数中使用
    fn half_of_even(nums: &[i32]) -> Option<i32> {
        let n = find_first_even(nums)?;  // None 则提前返回 None
        Some(n / 2)
    }
    println!("half: {:?}", half_of_even(&data));
}
输出
found: 4
first even: 4
value or default: 0
doubled: Some(8)
half: Some(2)
🌐 工程实践
  • 避免 unwrap()——在生产代码中用 match/if let/unwrap_or/? 替代。unwrap() 在 None 时 panic。
  • 链式方法——opt.map(f).and_then(g).unwrap_or(default) 构建处理管道。
  • 零开销——Option<&T> 和 &T 大小相同（null pointer optimization）。
✏️ 示例分析与易错点
RUSToption_chain.rs
fn parse_port(config: &str) -> Option<u16> {
    config
        .split(':')           // "host:8080" → ["host", "8080"]
        .nth(1)               // Option<&str> → Some("8080")
        .and_then(|s| s.parse().ok())  // Option<u16>
}

fn main() {
    println!("{:?}", parse_port("localhost:8080")); // Some(8080)
    println!("{:?}", parse_port("localhost"));       // None
    println!("{:?}", parse_port("localhost:abc"));   // None
}
输出
Some(8080)
None
None
易错：Option<T> 和 T 是不同类型！不能直接把 Option<i32> 当 i32 用——必须先"拆包"。
⚡ 快速检测

05Result<T, E> — 错误处理基础

🎯 为什么需要了解这个

异常（exception）的问题是看不出哪个函数会抛——你不知道什么时候需要 try/catch。Rust 没有异常，用 Result<T, E> 把"成功或失败"编码到返回类型中。调用者必须处理失败情况，否则编译不过。

🔍 核心思路
RUSTresult_def.rs
// 标准库定义（简化）
enum Result<T, E> {
    Ok(T),   // 成功，携带值
    Err(E),  // 失败，携带错误信息
}

Option 表示"有或没有"，Result 表示"成功或失败（附带原因）"。

📐 规则与语法
RUSTresult_usage.rs
use std::num::ParseIntError;

fn parse_number(s: &str) -> Result<i32, ParseIntError> {
    s.trim().parse::<i32>()
}

fn double_parse(s: &str) -> Result<i32, ParseIntError> {
    let n = parse_number(s)?;   // ? 直接传播错误
    Ok(n * 2)
}

fn main() {
    // 1. match
    match parse_number("42") {
        Ok(n) => println!("parsed: {}", n),
        Err(e) => println!("error: {}", e),
    }

    // 2. unwrap_or_else
    let val = parse_number("abc").unwrap_or_else(|e| {
        println!("fallback due to: {}", e);
        0
    });
    println!("val = {}", val);

    // 3. ? 运算符
    match double_parse("21") {
        Ok(n) => println!("doubled: {}", n),
        Err(e) => println!("error: {}", e),
    }

    match double_parse("xyz") {
        Ok(n) => println!("doubled: {}", n),
        Err(e) => println!("error: {}", e),
    }

    // 4. map / and_then
    let result = parse_number("10")
        .map(|n| n * 3)
        .map(|n| format!("value is {}", n));
    println!("{:?}", result);
}
输出
parsed: 42
fallback due to: invalid digit found in string
val = 0
doubled: 42
error: invalid digit found in string
Ok("value is 30")
🌐 工程实践
  • ? 是最常用的错误传播方式——自动将 Err 返回给调用者。函数返回类型必须是 Result。
  • main 也可以返回 Result——fn main() -> Result<(), Box<dyn Error>>。
  • 自定义错误类型——大项目用 thiserror 或 anyhow crate。
  • 避免 .unwrap()——只在确定不会失败时使用（如字面量 parse）。
✏️ 示例分析与易错点
RUSTresult_chain.rs
use std::fs;
use std::io;

fn read_username() -> Result<String, io::Error> {
    let content = fs::read_to_string("username.txt")?;
    Ok(content.trim().to_string())
}

fn main() {
    match read_username() {
        Ok(name) => println!("Welcome, {}!", name),
        Err(e) => println!("Could not read username: {}", e),
    }

    // Result 和 Option 的转换
    let opt: Option<i32> = Some(42);
    let res: Result<i32, &str> = opt.ok_or("missing value");
    println!("{:?}", res);

    let res2: Result<i32, &str> = Err("oops");
    let opt2: Option<i32> = res2.ok();
    println!("{:?}", opt2);
}
输出
Could not read username: No such file or directory (os error 2)
Ok(42)
None
易错：? 运算符只能在返回 Result 或 Option 的函数中使用。在 main() 中使用 ? 要把 main 的返回类型改为 Result。
