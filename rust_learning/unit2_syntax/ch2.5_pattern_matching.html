Unit 2 · 基础语法

模式匹配

如果说所有权是 Rust 的灵魂，那模式匹配就是 Rust 的手术刀——它让你精确地拆解数据结构、分派逻辑分支，并在编译期保证穷尽所有情况。从 match 到 if let，从解构到守卫，本章全面覆盖。

01match 表达式 — 穷尽性检查

🎯 为什么需要了解这个

C 的 switch 不强制处理所有 case，漏掉一个就是静默 bug。Rust 的 match 要求穷尽所有可能——编译器会检查你是不是遗漏了某个 enum 变体。这让你在添加新变体时，编译器会立刻告诉你哪些地方需要更新。

🔍 核心思路
  • match 是表达式——有返回值，可以赋给变量。
  • 穷尽性：必须覆盖被匹配类型的所有可能值。
  • 从上到下匹配，第一个匹配的分支执行。
  • 所有分支的返回类型必须相同。
  • _ 是通配符，匹配任何值。
📐 规则与语法
RUSTmatch_basics.rs
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: &Coin) -> u32 {
    match coin {
        Coin::Penny => {
            println!("Lucky penny!");
            1
        }
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
    // 删除任何一个变体的处理 → 编译错误！
}

fn main() {
    let coins = [Coin::Penny, Coin::Dime, Coin::Quarter];
    let total: u32 = coins.iter().map(value_in_cents).sum();
    println!("Total: {} cents", total);

    // match 作为表达式
    let x = 42;
    let description = match x {
        0 => "zero",
        1..=9 => "single digit",
        10..=99 => "double digit",
        _ => "large",          // 通配符兜底
    };
    println!("{} is {}", x, description);
}
输出
Lucky penny!
Total: 36 cents
42 is double digit
🌐 工程实践
  • enum dispatch——match 是处理 enum 的标准方式，编译器保证不漏分支。
  • 添加变体时——新增 enum 变体后，所有 match 都会编译报错，提醒你更新对应处理逻辑。
  • 避免过度 _——虽然方便，但通配符可能隐藏新增变体的遗漏。只在确定不关心"其余情况"时使用。
  • OR 模式——Coin::Penny | Coin::Nickel => "small" 合并相同处理的分支。
✏️ 示例分析与易错点
RUSTmatch_option.rs
fn describe_number(opt: Option<i32>) -> String {
    match opt {
        Some(n) if n > 0 => format!("positive: {}", n),
        Some(0) => "zero".to_string(),
        Some(n) => format!("negative: {}", n),
        None => "nothing".to_string(),
    }
}

fn main() {
    println!("{}", describe_number(Some(42)));
    println!("{}", describe_number(Some(0)));
    println!("{}", describe_number(Some(-3)));
    println!("{}", describe_number(None));
}
输出
positive: 42
zero
negative: -3
nothing
易错：match 分支顺序很重要。如果把通配符 _ 放在第一个，后面的分支永远不会执行。编译器会发出"unreachable pattern"警告。

02if let / while let — 简化单分支匹配

🎯 为什么需要了解这个

很多时候你只关心一个模式（比如 Some(x)），其余情况什么都不做。用完整的 match 就太啰嗦了。if let 是语法糖，让单分支匹配简洁明了。while let 则是"持续匹配直到失败"的循环版本。

🔍 核心思路
语法等价 match用途
if let Pat = expr { A } else { B }match expr { Pat => A, _ => B }只关心一个模式
while let Pat = expr { body }循环 match 直到匹配失败逐个取元素
let else（Rust 1.65+）let Ok(x) = expr else { return/break }不匹配时提前退出
📐 规则与语法
RUSTif_let.rs
fn main() {
    let config_max: Option<u32> = Some(100);

    // 啰嗦版
    match config_max {
        Some(max) => println!("Max is {}", max),
        _ => (),
    }

    // 简洁版：if let
    if let Some(max) = config_max {
        println!("Max is {}", max);
    }

    // 带 else
    if let Some(max) = config_max {
        println!("Max: {}", max);
    } else {
        println!("No max configured");
    }

    // while let — 逐个弹出
    let mut stack = vec![1, 2, 3];
    while let Some(top) = stack.pop() {
        println!("popped: {}", top);
    }
    println!("stack is now empty: {:?}", stack);

    // let-else（Rust 1.65+）
    fn parse_or_return(s: &str) -> Option<i32> {
        let Ok(n) = s.parse::<i32>() else {
            return None;   // 不匹配时必须发散（return/break/continue/panic）
        };
        Some(n * 2)
    }
    println!("parse: {:?}", parse_or_return("21"));
    println!("parse: {:?}", parse_or_return("abc"));
}
输出
Max is 100
Max is 100
Max: 100
popped: 3
popped: 2
popped: 1
stack is now empty: []
parse: Some(42)
parse: None
🌐 工程实践
  • if let 替代 is_some() + unwrap()——if let Some(x) = opt { use(x) } 比 if opt.is_some() { use(opt.unwrap()) } 更安全简洁。
  • while let 处理通道——while let Ok(msg) = receiver.recv() { handle(msg) }。
  • let-else 做早期卫语句——let Some(x) = get_thing() else { return }; 简化错误退出。
✏️ 示例分析与易错点
RUSTlet_else.rs
fn process_input(input: &str) -> Result<String, String> {
    // let-else 做卫语句——不匹配就提前退出
    let Some(first_word) = input.split_whitespace().next() else {
        return Err("empty input".into());
    };

    let Ok(number) = first_word.parse::<i32>() else {
        return Err(format!("'{}' is not a number", first_word));
    };

    Ok(format!("Parsed: {}", number * 2))
}

fn main() {
    println!("{:?}", process_input("42 hello"));
    println!("{:?}", process_input("abc"));
    println!("{:?}", process_input(""));
}
输出
Ok("Parsed: 84")
Err("'abc' is not a number")
Err("empty input")
易错：let-else 的 else 块必须"发散"（diverge）——即 return、break、continue 或 panic。不能只是赋默认值。
⚡ 快速检测
while let Some(x) = v.pop() 何时停止？

03解构 — 拆解 struct / enum / tuple / slice

🎯 为什么需要了解这个

解构（destructuring）是模式匹配最实用的能力——直接把复合类型"拆"成独立的部分。Rust 的解构非常强大，支持嵌套、忽略、切片模式等。掌握解构让你的代码简洁且表意清晰。

🔍 核心思路

可以解构的地方：let、match、if let、for、函数参数。

可以解构的类型：struct、enum、tuple、数组/切片、引用。

📐 规则与语法
RUSTdestructuring.rs
struct Point {
    x: f64,
    y: f64,
}

enum Shape {
    Circle(f64),
    Rect { w: f64, h: f64 },
}

fn main() {
    // 1. 解构 struct
    let p = Point { x: 3.0, y: 4.0 };
    let Point { x, y } = p;
    println!("x={}, y={}", x, y);

    // 重命名
    let Point { x: px, y: py } = Point { x: 1.0, y: 2.0 };
    println!("px={}, py={}", px, py);

    // 2. 解构 enum
    let shapes = vec![Shape::Circle(5.0), Shape::Rect { w: 10.0, h: 3.0 }];
    for s in &shapes {
        match s {
            Shape::Circle(r) => println!("circle r={}", r),
            Shape::Rect { w, h } => println!("rect {}x{}", w, h),
        }
    }

    // 3. 解构 tuple
    let (a, b, c) = (1, "hello", true);
    println!("a={}, b={}, c={}", a, b, c);

    // 忽略部分字段
    let (first, ..) = (1, 2, 3, 4);  // .. 忽略其余
    println!("first={}", first);

    // 4. 解构切片
    let numbers = [1, 2, 3, 4, 5];
    match numbers.as_slice() {
        [first, second, rest @ ..] => {
            println!("first={}, second={}, rest={:?}", first, second, rest);
        }
        _ => {}
    }

    // 5. 嵌套解构
    let nested = Some(Point { x: 10.0, y: 20.0 });
    if let Some(Point { x, y }) = nested {
        println!("nested: ({}, {})", x, y);
    }

    // 6. 函数参数解构
    fn distance(&Point { x, y }: &Point) -> f64 {
        (x * x + y * y).sqrt()
    }
    let p = Point { x: 3.0, y: 4.0 };
    println!("distance = {}", distance(&p));
}
输出
x=3, y=4
px=1, py=2
circle r=5
rect 10x3
a=1, b=hello, c=true
first=1
first=1, second=2, rest=[3, 4, 5]
nested: (10, 20)
distance = 5
🌐 工程实践
  • for 循环解构——for (i, val) in data.iter().enumerate() 直接解构索引和值。
  • 忽略不关心的字段——let Point { x, .. } = p; 只取 x，忽略其余。
  • 切片模式——[first, .., last] 取首尾，[head, tail @ ..] 取头和尾切片。
  • 避免过深嵌套——超过 3 层的嵌套解构会降低可读性，考虑分步处理。
✏️ 示例分析与易错点
RUSTslice_patterns.rs
fn describe(list: &[i32]) -> &str {
    match list {
        [] => "empty",
        [_] => "single element",
        [_, _] => "pair",
        [1, ..] => "starts with one",
        [.., 0] => "ends with zero",
        _ => "something else",
    }
}

fn main() {
    println!("{}", describe(&[]));
    println!("{}", describe(&[42]));
    println!("{}", describe(&[1, 2]));
    println!("{}", describe(&[1, 2, 3]));
    println!("{}", describe(&[5, 4, 3, 0]));
    println!("{}", describe(&[9, 8, 7]));
}
输出
empty
single element
pair
starts with one
ends with zero
something else
易错：.. 在模式中只能出现一次（因为有歧义）。[.., x, ..] 不合法。

04守卫与 @ 绑定 — 精细化匹配

🎯 为什么需要了解这个

有时候纯模式不够用：你想匹配 Some(x) 但只在 x > 0 时执行；或者想匹配一个范围并同时获得具体值。Match guard 和 @ 绑定是模式匹配的高级武器，让你写出更精确的分支。

🔍 核心思路
  • Match guard：模式 if 条件 => 分支——在结构匹配后加额外布尔条件。
  • @ 绑定：name @ 模式——在检查模式的同时把值绑定到变量。
  • OR 模式：A | B => ...——多个模式共享同一处理。
📐 规则与语法
RUSTguards_binding.rs
fn main() {
    // 1. Match guard
    let num = Some(4);
    match num {
        Some(x) if x < 0 => println!("negative: {}", x),
        Some(x) if x == 0 => println!("zero"),
        Some(x) => println!("positive: {}", x),
        None => println!("nothing"),
    }

    // 2. @ 绑定——匹配范围并获取具体值
    let age = 25;
    let category = match age {
        n @ 0..=12 => format!("child ({})", n),
        n @ 13..=17 => format!("teen ({})", n),
        n @ 18..=64 => format!("adult ({})", n),
        n @ 65.. => format!("senior ({})", n),
        _ => unreachable!(),
    };
    println!("{}", category);

    // 3. @ 在 enum 中
    enum Event {
        Click { x: i32, y: i32 },
        KeyPress(char),
    }

    let event = Event::Click { x: 100, y: 200 };
    match event {
        Event::Click { x: x @ 0..=800, y: y @ 0..=600 } => {
            println!("valid click at ({}, {})", x, y);
        }
        Event::Click { x, y } => println!("out of bounds: ({}, {})", x, y),
        Event::KeyPress(c @ 'a'..='z') => println!("lowercase: {}", c),
        Event::KeyPress(c) => println!("key: {}", c),
    }

    // 4. OR 模式
    let c = 'e';
    let is_vowel = matches!(c, 'a' | 'e' | 'i' | 'o' | 'u');
    println!("'{}' is vowel: {}", c, is_vowel);

    // OR 模式在 match 中
    let code = 404;
    match code {
        200 | 201 => println!("success"),
        301 | 302 => println!("redirect"),
        400..=499 => println!("client error"),
        500..=599 => println!("server error"),
        _ => println!("other"),
    }
}
输出
positive: 4
adult (25)
valid click at (100, 200)
'e' is vowel: true
client error
🌐 工程实践
  • matches! 宏——快速检查值是否匹配模式：matches!(x, Some(1..=10)) 返回 bool。
  • Guard 不影响穷尽性——编译器无法分析 guard 的完整性，所以即使你的 guard 覆盖了所有情况，仍需要通配分支。
  • @ 替代重复代码——不用先匹配范围再在分支内获取值，一步到位。
✏️ 示例分析与易错点
RUSThttp_status.rs
fn describe_status(code: u16) -> &'static str {
    match code {
        200 => "OK",
        201 => "Created",
        204 => "No Content",
        301 | 302 => "Redirect",
        400 => "Bad Request",
        401 => "Unauthorized",
        403 => "Forbidden",
        404 => "Not Found",
        c @ 405..=499 => {
            // 可以用 c 做日志，但这里只返回通用描述
            let _ = c;
            "Client Error"
        }
        500 => "Internal Server Error",
        502 | 503 => "Service Unavailable",
        _ => "Unknown",
    }
}

fn main() {
    for code in [200, 301, 404, 418, 500, 999] {
        println!("{}: {}", code, describe_status(code));
    }
}
输出
200: OK
301: Redirect
404: Not Found
418: Client Error
500: Internal Server Error
999: Unknown
易错：Match guard 会"偷走"模式的穷尽性保证。Some(x) if x > 0 和 Some(x) if x <= 0 看起来覆盖了所有 Some，但编译器不会这么推导——你仍需要一个兜底分支。
⚡ 快速检测
n @ 1..=12 中 n 绑定到什么？
