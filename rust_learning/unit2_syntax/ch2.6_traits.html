Unit 2 · 基础语法

Trait 系统

Trait 是 Rust 实现多态和代码复用的核心机制。它定义共享行为、约束泛型参数、支持运算符重载，是理解 Rust 类型系统的关键一步。本章从定义到实践，系统覆盖 trait 基础。

01定义与实现 Trait

🎯 为什么需要了解这个

如果你有很多类型都需要"可打印""可比较""可序列化"这种共享行为，你不想为每个类型各写一套。Trait 就是 Rust 的"接口"——定义一组方法签名，任何类型都可以实现它。标准库的 Display、Clone、Iterator 都是 trait。

🔍 核心思路
  • trait Name { fn method(&self); }——定义 trait。
  • impl Name for Type { fn method(&self) { ... } }——为类型实现 trait。
  • 孤儿规则：trait 或 type 至少有一个是当前 crate 的，才能写 impl。
  • Trait 方法通过 .method() 调用，需要 trait 在作用域内。
📐 规则与语法
RUSTtrait_basics.rs
// 定义一个 trait
trait Summary {
    fn summarize(&self) -> String;
}

// 为不同类型实现
struct Article {
    title: String,
    author: String,
    content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{}, by {} — {}...",
            self.title, self.author, &self.content[..20])
    }
}

struct Tweet {
    username: String,
    content: String,
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("@{}: {}", self.username, self.content)
    }
}

fn main() {
    let article = Article {
        title: "AI in 2025".into(),
        author: "Alice".into(),
        content: "Artificial intelligence has transformed many industries...".into(),
    };
    let tweet = Tweet {
        username: "rustlang".into(),
        content: "Rust 2025 roadmap is out!".into(),
    };

    println!("{}", article.summarize());
    println!("{}", tweet.summarize());
}
输出
AI in 2025, by Alice — Artificial intelligen...
@rustlang: Rust 2025 roadmap is out!
🌐 工程实践
  • trait 就是契约——定义 trait 时就是在声明"任何实现者必须提供这些能力"。
  • use 导入 trait——调用 trait 方法前，相应 trait 必须在作用域中（use std::fmt::Display;）。
  • 关联函数——trait 也可以有 fn new() -> Self 这类无 self 的关联函数。
  • 命名惯例——trait 名是形容词或能力描述：Printable, Sendable, Iterator。
✏️ 示例分析与易错点
RUSTorphan_rule.rs
use std::fmt;

// ✅ 可以为自己的类型实现外部 trait
struct MyType(i32);
impl fmt::Display for MyType {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "MyType({})", self.0)
    }
}

// ✅ 可以为外部类型实现自己的 trait
trait Describe {
    fn describe(&self) -> String;
}
impl Describe for Vec<i32> {
    fn describe(&self) -> String {
        format!("Vec with {} elements", self.len())
    }
}

// ❌ 不能为外部类型实现外部 trait
// impl fmt::Display for Vec<i32> { ... }  // 编译错误！

fn main() {
    let m = MyType(42);
    println!("{}", m);                    // Display

    let v = vec![1, 2, 3];
    println!("{}", v.describe());         // 自定义 trait
}
输出
MyType(42)
Vec with 3 elements
易错：忘记导入 trait 是常见编译错误。明明类型有 .method() 却报"method not found"——大概率是忘了 use 对应的 trait。

02默认方法 — 减少模板代码

🎯 为什么需要了解这个

如果 trait 有 10 个方法，每个实现者都得写 10 个，那太繁琐了。默认方法让 trait 在声明时就提供方法体。实现者只需重写需要自定义的方法，其余沿用默认逻辑。这是 Rust 版的"模板方法"模式。

🔍 核心思路
  • trait 中的方法可以有方法体——这就是默认方法。
  • 实现者可以选择覆盖（override）默认方法。
  • 默认方法可以调用同一 trait 中的其他方法（包括无默认实现的）。
  • 无法在 override 后再调用原始默认实现（Rust 无 super.method()）。
📐 规则与语法
RUSTdefault_methods.rs
trait Summary {
    fn author(&self) -> String;                  // 必须实现

    fn summarize(&self) -> String {               // 默认方法
        format!("(Read more from {}...)", self.author())
    }

    fn preview(&self) -> String {                 // 默认方法
        format!("[Preview] {}", self.summarize())
    }
}

struct Article {
    title: String,
    author: String,
}

// 只实现必须的方法，其余用默认
impl Summary for Article {
    fn author(&self) -> String {
        self.author.clone()
    }
    // summarize 和 preview 使用默认实现
}

struct Tweet {
    username: String,
    content: String,
}

impl Summary for Tweet {
    fn author(&self) -> String {
        format!("@{}", self.username)
    }

    // 覆盖 summarize 的默认实现
    fn summarize(&self) -> String {
        format!("{}: {}", self.author(), self.content)
    }
    // preview 用默认实现（会调用覆盖后的 summarize）
}

fn main() {
    let article = Article {
        title: "Rust Tips".into(),
        author: "Alice".into(),
    };
    println!("{}", article.summarize());
    println!("{}", article.preview());

    let tweet = Tweet {
        username: "bob".into(),
        content: "Hello Rust!".into(),
    };
    println!("{}", tweet.summarize());
    println!("{}", tweet.preview());
}
输出
(Read more from Alice...)
[Preview] (Read more from Alice...)
@bob: Hello Rust!
[Preview] @bob: Hello Rust!
🌐 工程实践
  • 最小实现原则——trait 只要求实现者提供"核心"方法，其余用默认方法组合出来。如 Iterator 只要求 next()，其余 map/filter/collect 等上百个方法都是默认方法。
  • 默认调用其他方法——默认方法可以依赖同 trait 中的其他方法，形成链式委托。
  • 小心覆盖连锁效应——覆盖基础方法后，所有调用它的默认方法行为都会改变。这是有意设计。
✏️ 示例分析与易错点
RUSTiterator_methods.rs
struct Counter {
    count: u32,
    max: u32,
}

impl Counter {
    fn new(max: u32) -> Self {
        Counter { count: 0, max }
    }
}

// 只实现 next()——获得所有迭代器方法
impl Iterator for Counter {
    type Item = u32;
    fn next(&mut self) -> Option<Self::Item> {
        if self.count < self.max {
            self.count += 1;
            Some(self.count)
        } else {
            None
        }
    }
}

fn main() {
    // 免费获得 sum, map, filter, collect, zip...
    let total: u32 = Counter::new(5).sum();
    println!("1+2+3+4+5 = {}", total);

    let doubled: Vec<u32> = Counter::new(3).map(|x| x * 2).collect();
    println!("doubled: {:?}", doubled);

    let pairs: Vec<_> = Counter::new(3)
        .zip(Counter::new(3).skip(1))
        .collect();
    println!("pairs: {:?}", pairs);
}
输出
1+2+3+4+5 = 15
doubled: [2, 4, 6]
pairs: [(1, 2), (2, 3)]
要点：Iterator trait 有 70+ 默认方法，你只需实现 next()。这就是默认方法的威力。
⚡ 快速检测
覆盖了 summarize() 后，默认方法 preview() 里调用的是哪个版本？

03Trait Bound — 泛型约束

🎯 为什么需要了解这个

泛型函数 fn foo<T>(x: T) 默认对 T 一无所知——不能打印、不能比较、什么都不能做。Trait bound 告诉编译器："T 必须实现某些 trait"，这样你就能在泛型函数内调用对应的方法。这是 Rust 泛型系统的基石。

🔍 核心思路
语法含义
T: DisplayT 必须实现 Display
T: Display + Clone同时满足多个 trait
where T: Display等价但更清晰的写法
impl Display 参数语法糖，等价于泛型 + trait bound
T: 'a生命周期 bound
📐 规则与语法
RUSTtrait_bounds.rs
use std::fmt::Display;

// 内联 trait bound
fn print_twice<T: Display>(item: &T) {
    println!("{}", item);
    println!("{}", item);
}

// 多个 trait bound
fn print_and_clone<T: Display + Clone>(item: &T) {
    let cloned = item.clone();
    println!("original: {}, copy: {}", item, cloned);
}

// where 子句 — 更清晰
fn compare_and_display<T, U>(t: &T, u: &U)
where
    T: Display + PartialOrd,
    U: Display + Into<f64> + Copy,
{
    println!("t={}, u={}", t, u);
}

// impl Trait 作为参数（语法糖）
fn notify(item: &impl Display) {
    println!("Breaking news: {}", item);
}

fn main() {
    print_twice(&"hello");
    print_and_clone(&42);
    compare_and_display(&3.14, &2.0_f64);
    notify(&"Rust 2.0 released!");
}
输出
hello
hello
original: 42, copy: 42
t=3.14, u=2
Breaking news: Rust 2.0 released!
🌐 工程实践
  • 编译时检查——trait bound 是零成本抽象。编译器单态化泛型，运行时无开销。
  • 错误信息——不满足 trait bound 的编译错误信息清晰，会告诉你缺少哪个 trait 实现。
  • 条件实现——impl<T: Display> ToString for T { ... } 标准库用这种方式为所有 Display 类型自动实现 ToString。
  • 选择策略——参数少、bound 简单用内联；多参数、复杂 bound 用 where。
✏️ 示例分析与易错点
RUSTconditional_impl.rs
#[derive(Debug)]
struct Pair<T> {
    first: T,
    second: T,
}

// 对所有 T 都有 new
impl<T> Pair<T> {
    fn new(first: T, second: T) -> Self {
        Self { first, second }
    }
}

// 只有 T: Display + PartialOrd 时才有 cmp_display
impl<T: std::fmt::Display + PartialOrd> Pair<T> {
    fn cmp_display(&self) {
        if self.first >= self.second {
            println!("larger = {}", self.first);
        } else {
            println!("larger = {}", self.second);
        }
    }
}

fn main() {
    let p1 = Pair::new(10, 20);
    p1.cmp_display();   // ✅ i32 实现了 Display + PartialOrd

    let p2 = Pair::new(vec![1], vec![2]);
    // p2.cmp_display();  // ❌ Vec 没实现 Display
    println!("{:?}", p2);  // ✅ Debug 可以
}
输出
larger = 20
Pair { first: [1], second: [2] }
要点：条件 impl 是 Rust 强大的特性——同一泛型类型，不同 T 可以有不同的方法集合。

04derive 宏 — 自动实现 Trait

🎯 为什么需要了解这个

手动实现 Debug、Clone、PartialEq 这些常见 trait 非常繁琐且容易出错。#[derive(...)] 让编译器自动生成这些实现，几乎每个 Rust 项目都会大量使用。

🔍 核心思路
可 derive 的 Trait功能
Debug{:?} 格式化输出
Clone显式 .clone()
Copy隐式按位复制（需先 Clone）
PartialEq / Eq== / != 比较
PartialOrd / Ord< > <= >= 比较
Hash哈希值（用于 HashMap key）
Default默认值（Default::default()）
📐 规则与语法
RUSTderive_example.rs
#[derive(Debug, Clone, PartialEq)]
struct Color {
    r: u8,
    g: u8,
    b: u8,
}

#[derive(Debug, Clone, Copy, PartialEq, Default)]
struct Point {
    x: f64,
    y: f64,
}

fn main() {
    // Debug — {:?} 打印
    let red = Color { r: 255, g: 0, b: 0 };
    println!("{:?}", red);
    println!("{:#?}", red);  // 漂亮打印

    // Clone
    let red2 = red.clone();
    println!("cloned: {:?}", red2);

    // PartialEq — == 比较
    println!("equal: {}", red == red2);

    // Copy — 隐式复制（Point 是 Copy）
    let p1 = Point { x: 1.0, y: 2.0 };
    let p2 = p1;       // Copy，不是 move
    println!("p1={:?}, p2={:?}", p1, p2);  // 两个都能用

    // Default
    let origin = Point::default();  // x: 0.0, y: 0.0
    println!("origin: {:?}", origin);
}
输出
Color { r: 255, g: 0, b: 0 }
Color {
    r: 255,
    g: 0,
    b: 0,
}
cloned: Color { r: 255, g: 0, b: 0 }
equal: true
p1=Point { x: 1.0, y: 2.0 }, p2=Point { x: 1.0, y: 2.0 }
origin: Point { x: 0.0, y: 0.0 }
🌐 工程实践
  • 最佳实践——几乎所有自定义类型都应该 derive Debug。
  • Copy 的限制——所有字段都是 Copy 才行。包含 String、Vec 等的类型不能 Copy。
  • PartialEq vs Eq——Eq 保证自反性（a == a），浮点数因为 NaN 只能 PartialEq。
  • HashMap key——需要同时实现 Eq + Hash。f64 不能做 key（因为没有 Eq）。
  • 第三方 derive——serde 的 #[derive(Serialize, Deserialize)] 也是同样模式。
✏️ 示例分析与易错点
RUSTderive_enum.rs
#[derive(Debug, Clone, PartialEq)]
enum Shape {
    Circle(f64),
    Rectangle { w: f64, h: f64 },
    Triangle(f64, f64, f64),
}

impl Shape {
    fn area(&self) -> f64 {
        match self {
            Shape::Circle(r) => std::f64::consts::PI * r * r,
            Shape::Rectangle { w, h } => w * h,
            Shape::Triangle(a, b, c) => {
                let s = (a + b + c) / 2.0;
                (s * (s - a) * (s - b) * (s - c)).sqrt()
            }
        }
    }
}

fn main() {
    let shapes = vec![
        Shape::Circle(5.0),
        Shape::Rectangle { w: 4.0, h: 6.0 },
        Shape::Triangle(3.0, 4.0, 5.0),
    ];

    for s in &shapes {
        println!("{:?} → area = {:.2}", s, s.area());
    }

    // PartialEq 比较
    let c1 = Shape::Circle(5.0);
    let c2 = Shape::Circle(5.0);
    println!("c1 == c2: {}", c1 == c2);
}
输出
Circle(5.0) → area = 78.54
Rectangle { w: 4.0, h: 6.0 } → area = 24.00
Triangle(3.0, 4.0, 5.0) → area = 6.00
c1 == c2: true
易错：derive 要求所有字段都实现目标 trait。如果给包含 f64 的 struct derive Eq，会编译错误（f64 只有 PartialEq）。
⚡ 快速检测

05impl Trait — 参数与返回值中的 Trait

🎯 为什么需要了解这个

有时候你想写一个函数"接受任何可打印的东西"或"返回某个迭代器但不想暴露具体类型"。impl Trait 在参数和返回位置的两种用法，让泛型代码更简洁，同时保持零成本抽象。

🔍 核心思路
位置语法意义等价
参数fn f(x: &impl Display)接受任何 Display 类型fn f<T: Display>(x: &T)
返回fn f() -> impl Display返回某个 Display 类型（无直接等价泛型写法）
  • 参数位置：每个 impl Trait 引入独立的泛型参数。
  • 返回位置：函数内部只能返回同一个具体类型。
📐 规则与语法
RUSTimpl_trait.rs
use std::fmt::Display;

// 1. 参数位置 — 语法糖
fn log(item: &impl Display) {
    println!("[LOG] {}", item);
}

// 等价版本
fn log_explicit<T: Display>(item: &T) {
    println!("[LOG] {}", item);
}

// 2. 返回位置 — 隐藏具体类型
fn make_greeting(name: &str) -> impl Display {
    format!("Hello, {}!", name)
    // 调用者只知道返回值实现了 Display
    // 不知道具体类型是 String
}

// 3. 返回复杂迭代器 — impl Trait 最常见用途
fn even_squares(limit: u32) -> impl Iterator<Item = u32> {
    (1..=limit)
        .map(|x| x * x)
        .filter(|&x| x % 2 == 0)
    // 具体类型太复杂：Filter<Map<RangeInclusive<u32>, ...>, ...>
    // impl Iterator 让类型签名保持简洁
}

// 4. 参数和返回中同时使用
fn double_display(item: &impl Display) -> impl Display {
    format!("{} {}", item, item)
}

fn main() {
    log(&42);
    log(&"hello");
    log_explicit(&3.14);

    let greeting = make_greeting("Rustacean");
    println!("{}", greeting);

    let squares: Vec<u32> = even_squares(10).collect();
    println!("even squares: {:?}", squares);

    let doubled = double_display(&"ha");
    println!("{}", doubled);
}
输出
[LOG] 42
[LOG] hello
[LOG] 3.14
Hello, Rustacean!
even squares: [4, 16, 36, 64, 100]
ha ha
🌐 工程实践
  • 返回迭代器——-> impl Iterator 是最常见的用法，避免写出复杂的嵌套类型。
  • 返回闭包——-> impl Fn(i32) -> i32 用于返回闭包。
  • 限制：返回位置的 impl Trait 不能返回不同类型——if cond { "str" } else { 42 } 不合法（即使都 Display）。需要动态分派时用 Box<dyn Trait>。
  • impl Trait vs dyn Trait——impl 是编译时单态化（快），dyn 是运行时动态分派（灵活）。
✏️ 示例分析与易错点
RUSTreturn_closure.rs
// 返回闭包
fn make_adder(x: i32) -> impl Fn(i32) -> i32 {
    move |y| x + y
}

// ❌ 不能返回不同类型
// fn make_formatter(fancy: bool) -> impl Display {
//     if fancy { format!("✨ fancy ✨") }
//     else { 42 }  // 错！String 和 i32 是不同类型
// }

// ✅ 需要不同类型时用 Box<dyn Trait>
fn make_formatter(fancy: bool) -> Box<dyn std::fmt::Display> {
    if fancy {
        Box::new(format!("fancy output"))
    } else {
        Box::new(42)
    }
}

fn main() {
    let add5 = make_adder(5);
    println!("3 + 5 = {}", add5(3));
    println!("10 + 5 = {}", add5(10));

    println!("{}", make_formatter(true));
    println!("{}", make_formatter(false));
}
输出
3 + 5 = 8
10 + 5 = 15
fancy output
42
核心区别：impl Trait（静态分派，零开销，但只能返回一种类型）vs dyn Trait（动态分派，有间接寻址开销，但可以返回多种类型）。这是后续章节（Trait 对象）的重点。
⚡ 快速检测
返回 impl Iterator 的函数体能 return 两种不同的迭代器类型吗？
