# Rust 子 Agent 工作规则

> 本文件规定后续子 agent 在 Rust 学习内容生成中的职责与交付标准。

---

## 一、学习顺序

每个子 agent 必须按以下顺序工作：
1. 阅读 `generation_rule.md`
2. 阅读 `examples/ch1.1_ownership.html`
3. 阅读本文件
4. 规划自己负责的单元与章节
5. 记录 workflow
6. 开始生成页面
7. 结束后自评并记录问题

---

## 二、推荐分工

- agent-u0：环境与心态
- agent-u1：所有权系统
- agent-u2：基础语法
- agent-u3：标准库与项目实战
- agent-u4：并发与异步
- agent-u5：高级特性与方向分支

---

## 三、页面生成要求

### 每个知识点至少包含
- 一个引导问题
- 一段直觉解释
- 一条正式规则或概念定义
- 一个 Rust 代码示例
- 一道快速测试或思考题

### 代码示例要求
- 保持短小、可读、可运行
- 输出或报错信息要有说明
- 尽量体现 Rust 语言设计原因，而不只是语法记忆

### 练习要求
- 基础题覆盖核心知识点
- 进阶题要求综合运用
- 附加题必须贴近真实工程任务

---

## 四、风格要求

- 保持 `math_univers` 风格中的深色阅读界面、卡片结构、交互组件和节奏感
- 但标题、文案、案例、代码和练习必须全部服务于 Rust 学习
- 不得混入数学主题页面、数学命名或数学导航结构

---

## 五、workflow 记录规范

每个 agent 的 workflow 至少记录：
- 负责范围
- 章节规划
- 已完成内容
- 自评分数
- 待修改项

---

## 六、自评标准

页面低于 80 分不得视为完成。自评时必须明确：
- 哪些地方讲清楚了
- 哪些地方仍然抽象
- 哪些练习还不够有诊断性
