# Rust 子 Agent 工作规则

> 本文件规定后续子 agent 在 Rust 学习内容生成中的职责、执行顺序和交付标准。

---

## 一、总原则

- 保持 `math_univers` 风格中的深色阅读界面、卡片结构、交互组件和内容节奏
- 但所有标题、文案、案例、代码和练习必须全部服务于 Rust 学习
- 不得混入数学主题页面、数学命名或数学导航结构
- 不允许把 Rust 内容写成机械式语法索引

---

## 二、必须执行的顺序

### 步骤 1：学习阶段
每个子 agent 开始前必须阅读：
1. `rust_learning/generation_rule.md`
2. `rust_learning/sub_agent_rule.md`
3. `rust_learning/templates/chapter_template.html`
4. `rust_learning/examples/ch1.1_ownership.html`
5. 自己负责单元目录下的 `workflow.md`

### 步骤 2：规划阶段
- 明确章节目标
- 列出知识点
- 规划快速测试方向
- 规划课后练习梯度
- 将结果写入 `workflow.md`

### 步骤 3：生成阶段
- 优先完成结构正确的章节页
- 保证所有交互组件使用标准 class
- 代码示例要短小、合法、可读

### 步骤 4：自审阶段
- 用 generation rule 中的评分规则自评
- 记录优点、问题和待修改项
- 低于 80 分必须返工

### 步骤 5：汇报阶段
汇报至少包含：
- 完成章节
- 当前分数
- 仍存问题
- 下一步动作

---

## 三、推荐分工

- agent-u0：环境与心态
- agent-u1：所有权系统
- agent-u2：基础语法
- agent-u3：标准库与项目实战
- agent-u4：并发与异步
- agent-u5：高级特性
- agent-u6：方向分支

---

## 四、页面生成要求

### 每个知识点至少包含
- 一个引导问题
- 一段直觉解释
- 一条正式规则或定义
- 一个 Rust 代码示例
- 一个快速测试或折叠式答案

### 代码示例要求
- 代码必须合法
- 代码长度尽量控制在读者能快速理解的范围内
- 输出、报错或行为结果要解释
- 尽量体现语言设计动机，而不仅是语法写法

### 练习要求
- 基础题覆盖核心知识点
- 进阶题要求综合运用
- 挑战题聚焦难点误区
- 附加题必须贴近真实工程任务

---

## 五、workflow 记录要求

每个单元目录下的 `workflow.md` 必须持续更新：
- 第一步学习是否完成
- 当前章节规划
- 已完成页面
- 自评分数
- 待修订项

禁止只写“已完成”，必须写清具体完成了什么。

---

## 六、示例页的作用

`examples/ch1.1_ownership.html` 是当前风格对齐样板，重点学习：
- 页面节奏
- 提问式知识点展开
- Rust 代码块的组织方式
- 快速测试反馈方式
- 总结与练习区的收束方式

`templates/chapter_template.html` 是结构样板，重点用于复制骨架而不是复制具体内容。
