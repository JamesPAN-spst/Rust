# Rust 学习内容生成规则

> 目标：在 Rust 仓库中生成与 `math_univers` 设计语言一致、但主题完全服务于 Rust 学习路径的网页模板与内容规范。

---

## 一、设计基准

本仓库当前没有既有网页系统，因此页面设计基准采用对 `math_univers` 的风格抽象，而不是复用其主题内容：
- 深色背景 + 高对比强调色
- 固定侧边栏 + 主内容阅读区
- 卡片化知识模块
- 引导式提问、快速测试、折叠答案、代码演示
- 标题层级明确、节奏稳定、适合长篇学习内容

本项目中所有内容必须围绕 Rust 学习路径组织，不得继续生成数学主题数据。

## 三、工作步骤

1. 先阅读本规则文件
2. 学习示例页面 `examples/ch1.1_ownership.html`
3. 按单元和章节规划知识点、练习、交互方式
4. 优先生成模板正确、结构稳定的章节页面
5. 每章完成后执行自审

---

## 四、内容质量要求

- 内容必须解释“为什么要学、如何使用、容易错在哪里”
- 讲解顺序优先采用：问题 → 直觉 → 规则 → 示例 → Rust 代码 → 常见错误
- 代码示例必须是合法 Rust 代码，且尽量短小完整
- 快速测试聚焦误区，不能只是常识判断
- 练习分为“快速知识点测试”和“课后测试”两类

---

## 五、页面一致性要求

每个章节页面必须包含：
1. 章节头
2. 学习目标
3. 前置知识
4. 多个知识点模块
5. 章节总结
6. 课后练习
7. 延伸阅读

标准组件：
- `.chapter-header`
- `.learning-objectives`
- `.prerequisites`
- `.knowledge-point`
- `.think-box`
- `.code-playground`
- `.quick-quiz`
- `.collapsible`
- `.note-box`
- `.chapter-summary`
- `.exercises`
- `.further-reading`

---

## 六、练习设计要求

### 快速知识点测试
- 每个知识点后 1~2 题
- 聚焦 move、借用、生命周期、trait bound 等高频误区
- 必须给出即时反馈

### 课后测试
- 至少覆盖基础、进阶、附加题三个层级
- 附加题优先结合真实工程任务，例如调试 borrow checker 报错、补全 trait 实现、设计 API

---

## 七、交互设计要求

必须保留以下交互模式：
- 引导式提问
- 点击展开答案
- 选择题即时反馈
- 代码块 + 预期输出
- 总结卡片与错误提示卡片

---

## 八、最终审阅要求

按以下维度评分，总分 100：
- 结构完整性：20
- 风格一致性：20
- 知识准确性：20
- 讲解深度与逻辑性：15
- 交互设计质量：10
- 快速测试质量：5
- 课后练习质量：5
- 可维护性与模块化：5

审阅输出必须包含：
- 总分
- 各维度分数
- 主要优点
- 主要问题
- 需要修改的具体项
