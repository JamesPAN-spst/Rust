# Rust 学习内容生成规则

> 目标：在 Rust 仓库中生成与 `math_univers` 设计语言一致、但主题完全服务于 Rust 学习路径的网页模板与内容规范。

---

## 一、已学习到的设计风格与设计特点

本仓库当前没有既有网页系统，因此页面设计基准采用对 `math_univers` 的风格抽象，而不是复用其主题内容。

### 1.1 页面结构特征
- 固定侧边栏 + 主阅读区的双栏结构
- 章节内容按卡片分块，避免超长纯文本堆叠
- 页面按照“标题 → 学习目标 → 前置知识 → 知识点 → 总结 → 练习 → 延伸阅读”的稳定节奏展开
- 每个知识点内部都强调引导、解释、示例、反馈四段式体验

### 1.2 视觉语言特征
- 深色背景，强调色高对比
- 卡片圆角、轻阴影、统一边框
- 代码块与正文区分明显，适合长时间阅读
- 导航标题、单元标签、提示框都有稳定的层级感

### 1.3 内容组织特征
- 不是百科式罗列，而是问题驱动
- 优先解释“为什么需要这个概念”
- 用小型例子快速建立直觉，再给出规则和代码
- 高频使用提示框、折叠答案、快速测试来维持节奏

### 1.4 迁移到 Rust 模板时的约束
- 仅迁移设计风格，不迁移数学主题
- 标题、案例、导航、代码、练习必须全部服务于 Rust 学习路线
- 页面必须体现 Rust 的语言设计原因、工程动机和常见误区

## 四、工作步骤

### 步骤 1：学习阶段（第一步）
必须先学习以下材料：
1. 本文件 `generation_rule.md`
2. `sub_agent_rule.md`
3. 示例页 `examples/ch1.1_ownership.html`
4. 通用模板 `templates/chapter_template.html`
5. 根目录 `workflow.md` 中记录的风格学习结果

### 步骤 2：规划阶段
- 按单元拆分章节目标
- 细化知识点、误区、代码示例、测试题、课后题
- 记录到对应单元的 `workflow.md`

### 步骤 3：内容生成阶段
- 先生成结构正确、节奏稳定的章节页
- 统一使用标准组件 class
- 保持内容与 Rust 工程场景绑定

### 步骤 4：自审阶段
- 按评分规则自评
- 补写优点、问题、待修改项
- 低于 80 分不可视为完成

## 五、内容质量要求

### 5.1 讲解要求
- 必须解释“为什么学、怎么用、哪里容易错”
- 优先采用：问题 → 直觉 → 规则 → 代码 → 误区 → 反馈
- 不允许堆砌语法点而不解释语言设计原因
- 示例要尽量短，但上下文完整

### 5.2 Rust 代码要求
- 代码必须是合法 Rust
- 优先展示最小可运行例子
- 输出、报错或行为结果必须解释
- 尽量体现所有权、借用、trait、错误处理等真实语言语义

### 5.3 章节内容最低要求
每个知识点至少包含：
- 一个引导问题
- 一段直觉说明
- 一条正式规则或定义
- 一个 Rust 示例
- 一个快速测试或折叠式思考题

## 六、页面一致性要求

### 6.1 每章必须包含
1. 章节头
2. 学习目标
3. 前置知识
4. 多个知识点模块
5. 章节总结
6. 课后练习
7. 延伸阅读

### 6.2 标准组件
- `.chapter-header`
- `.learning-objectives`
- `.prerequisites`
- `.knowledge-point`
- `.think-box`
- `.code-playground`
- `.quick-quiz`
- `.collapsible`
- `.note-box`
- `.chapter-summary`
- `.exercises`
- `.further-reading`

### 6.3 命名要求
- 知识点 id：`kp-X`
- 练习题可使用 `ex-X`
- 快速测试答案统一放入 `data-answer`

---

## 七、练习设计要求

### 7.1 快速知识点测试
- 每个知识点后 1~2 题
- 聚焦 move、借用范围、生命周期误判、trait bound、错误传播等误区
- 必须提供即时反馈与简短解释

### 7.2 课后测试
- 至少覆盖基础、进阶、挑战、附加四档
- 附加题优先绑定真实工程场景
- 允许题型包括：解释题、调试题、补全题、方案设计题、API 设计题

---

## 八、交互设计要求

必须保留以下交互：
- 引导式提问
- 折叠答案
- 选择题即时反馈
- 代码示例 + 预期输出
- 提示框 / 警告框 / 总结卡片

---

## 九、workflow 记录规范

每个 `workflow.md` 至少包含：
- 负责范围
- 风格学习记录
- 章节规划
- 已完成内容
- 自评分数
- 主要问题
- 待修改项

推荐使用如下结构：
- 基本信息
- 第一步：学习阶段记录
- 第二步：章节规划
- 进度清单
- 工作后审阅

---

## 十、整体页面评分规则

总分 100：
- 结构完整性：20
- 风格一致性：20
- 知识准确性：20
- 讲解深度与逻辑性：15
- 交互设计质量：10
- 快速测试质量：5
- 课后练习质量：5
- 可维护性与模块化：5

评分时必须写出：
- 总分
- 各维度分数
- 主要优点
- 主要问题
- 需要修改的具体项
