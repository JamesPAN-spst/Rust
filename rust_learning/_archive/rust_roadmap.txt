───────────────────────────────────────────────
Unit 0 · 环境与心态
───────────────────────────────────────────────
Ch 0.1 工具链
  - rustup（版本管理）
  - cargo（构建 / 包管理 / 测试 / 文档一体化）
  - rust-analyzer（LSP 编辑器集成）
  - clippy（linter）
  - rustfmt（格式化）

Ch 0.2 心态建设
  - 与 C++ 的本质区别：编译期拦截 vs 运行期崩溃
  - "Fighting the borrow checker" 是正常阶段
  - 编译器错误信息是学习资源，不是障碍

───────────────────────────────────────────────
Unit 1 · 所有权系统（核心难点）
───────────────────────────────────────────────
Ch 1.1 Ownership
  - 每个值有唯一所有者
  - move 语义（赋值 = 所有权转移）
  - Copy trait（值类型的例外）
  - Drop trait（RAII 自动析构）

Ch 1.2 Borrowing
  - 不可变引用 &T（可多个）
  - 可变引用 &mut T（唯一）
  - 借用规则：aliasing XOR mutability
  - 引用必须始终有效（no dangling）

Ch 1.3 Lifetimes
  - 生命周期参数 'a 语法
  - 生命周期省略规则
  - 'static 生命周期
  - 结构体中的生命周期

Ch 1.4 所有权的实际影响
  - 为什么 Rust 没有双重释放
  - 为什么 Rust 没有数据竞争（编译期）
  - 与 C++ 智能指针的对比

───────────────────────────────────────────────
Unit 2 · 基础语法
───────────────────────────────────────────────
Ch 2.1 基本类型与变量
  - let / mut / const / static
  - 标量类型（i32, u64, f64, bool, char）
  - 复合类型（tuple, array）
  - shadowing

Ch 2.2 控制流
  - if / else（表达式）
  - loop / while / for
  - break 返回值

Ch 2.3 函数与闭包
  - fn 签名与返回值
  - 闭包语法 |x| x + 1
  - Fn / FnMut / FnOnce 三种闭包 trait

Ch 2.4 Struct 与 Enum
  - struct / tuple struct / unit struct
  - impl 块与方法
  - enum 及其代数数据类型本质
  - Option<T> / Result<T, E>

Ch 2.5 模式匹配
  - match 表达式
  - if let / while let
  - 解构 struct / enum / tuple
  - 守卫（match guards）

Ch 2.6 Trait 系统
  - trait 定义与实现
  - 默认方法
  - trait bound（泛型约束）
  - derive 宏（Debug, Clone, PartialEq, ...）

Ch 3.4 错误处理

Ch 3.5 模块系统

Ch 3.6 实战项目

Unit 4 · 并发与异步
───────────────────────────────────────────────
Ch 4.1 线程并发

  - Send / Sync trait
  - Arc<T>（原子引用计数）

Ch 4.2 异步编程

Ch 4.3 异步生态

───────────────────────────────────────────────
Unit 5 · 高级特性
───────────────────────────────────────────────
Ch 5.1 智能指针
  - Box<T>（堆分配）
  - Rc<T>（引用计数）
  - RefCell<T>（内部可变性）

Ch 5.2 泛型与 Trait 对象

  - 动态分发（dyn Trait）

Ch 5.3 宏系统

Ch 5.4 Unsafe Rust

  - 裸指针
