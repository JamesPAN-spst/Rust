2.4.1 petgraph 库介绍

🤔

在 Ch 2.2 中我们手写了邻接表和邻接矩阵。但在真实项目中，你会从零造轮子吗？ Rust 生态有一个强大的图库——petgraph。它提供了哪些开箱即用的能力？ 手写实现和使用库各有什么优劣？

为什么选择 petgraph？

petgraph 是 Rust 生态中最成熟的图数据结构库。 它提供了多种图类型、经典算法实现，以及与 Rust 所有权系统完美配合的 API 设计。

核心类型

类型 说明 适用场景
Graph<N, E> 基于邻接表的通用图（有向/无向） 大多数场景
DiGraph<N, E> 有向图（Graph 的有向特化） 依赖关系、工作流
UnGraph<N, E> 无向图（Graph 的无向特化） 社交网络、物理连接
GraphMap<N, E, Ty> 基于 HashMap 的图，节点可直接用值索引 节点是字符串/自定义类型时
StableGraph<N, E> 删除节点后索引不变 需要频繁增删节点

基本使用

Rust petgraph_basics.rs 
// Cargo.toml: petgraph = "0.6"
use petgraph::graph::{Graph, NodeIndex};
use petgraph::Undirected;

fn main() {
    // 创建无向图：社交网络
    let mut social: Graph<&str, &str, Undirected> = Graph::new_undirected();

    // 添加节点（返回 NodeIndex 句柄）
    let alice = social.add_node("Alice");
    let bob = social.add_node("Bob");
    let carol = social.add_node("Carol");
    let dave = social.add_node("Dave");

    // 添加边（关系）
    social.add_edge(alice, bob, "朋友");
    social.add_edge(alice, carol, "同事");
    social.add_edge(bob, carol, "朋友");
    social.add_edge(carol, dave, "同事");

    // 查询
    println!("节点数: {}", social.node_count());
    println!("边数: {}", social.edge_count());

    // 遍历 Alice 的邻居
    for neighbor in social.neighbors(alice) {
        println!("Alice 认识: {}", social[neighbor]);
    }
}

petgraph 内置算法

petgraph 提供了许多开箱即用的经典算法：

  • petgraph::algo::dijkstra — Dijkstra 最短路径
  • petgraph::algo::astar — A* 搜索
  • petgraph::algo::bellman_ford — Bellman-Ford（支持负权）
  • petgraph::algo::min_spanning_tree — 最小生成树
  • petgraph::algo::toposort — 拓扑排序
  • petgraph::algo::connected_components — 连通分量
  • petgraph::visit::Bfs / Dfs — BFS/DFS 遍历器
Rust petgraph_algo.rs 
use petgraph::graph::Graph;
use petgraph::algo::{dijkstra, min_spanning_tree};
use petgraph::data::FromElements;
use petgraph::Undirected;

fn main() {
    // 带权无向图：城市间距离
    let mut g: Graph<&str, u32, Undirected> = Graph::new_undirected();
    let bj = g.add_node("北京");
    let sh = g.add_node("上海");
    let gz = g.add_node("广州");
    let cd = g.add_node("成都");

    g.add_edge(bj, sh, 1200);
    g.add_edge(bj, cd, 1800);
    g.add_edge(sh, gz, 1500);
    g.add_edge(gz, cd, 1200);
    g.add_edge(sh, cd, 2000);

    // Dijkstra: 从北京出发的最短距离
    let distances = dijkstra(&g, bj, None, |e| *e.weight());
    for (node, dist) in &distances {
        println!("北京 → {}: {}km", g[*node], dist);
    }

    // 最小生成树
    let mst: Graph<&str, u32, Undirected> = Graph::from_elements(
        min_spanning_tree(&g)
    );
    println!("\nMST 边数: {}", mst.edge_count());
    let total: u32 = mst.edge_weights().sum();
    println!("MST 总权重: {}km", total);
}

2.4.2 实现一个社交网络分析工具

🤔

在一个社交网络中，谁是"最有影响力"的人？哪些人形成了紧密的小团体？ 如何找出网络中的关键节点——移除它会导致网络断裂？ 这些问题都可以用图论的工具来回答。

度中心性（Degree Centrality）

最简单的"影响力"指标是度中心性——一个节点的连接数占总可能连接数的比例：

C_D(v) = \frac{\deg(v)}{n - 1}

其中 \deg(v) 是节点 v 的度（邻居数）， n 是图中节点总数。度中心性的值在 0 到 1 之间。

连通分量（Connected Components）

连通分量是图中的极大连通子图——在同一个分量中的任意两个节点之间都存在路径。 社交网络中，每个连通分量就是一个独立的"社交圈"。

完整实现

Rust social_network.rs 
use std::collections::{HashMap, HashSet, VecDeque};

/// 无向图的邻接表表示
struct SocialGraph {
    adj: HashMap<String, HashSet<String>>,
}

impl SocialGraph {
    fn new() -> Self {
        SocialGraph { adj: HashMap::new() }
    }

    fn add_person(&mut self, name: &str) {
        self.adj.entry(name.to_string()).or_insert_with(HashSet::new);
    }

    fn add_friendship(&mut self, a: &str, b: &str) {
        self.adj.entry(a.to_string()).or_default().insert(b.to_string());
        self.adj.entry(b.to_string()).or_default().insert(a.to_string());
    }

    /// 计算每个节点的度中心性
    fn degree_centrality(&self) -> Vec<(&str, f64)> {
        let n = self.adj.len() as f64;
        if n <= 1.0 { return vec![]; }

        let mut result: Vec<(&str, f64)> = self.adj.iter()
            .map(|(name, neighbors)| {
                let centrality = neighbors.len() as f64 / (n - 1.0);
                (name.as_str(), centrality)
            })
            .collect();

        // 按中心性降序排列
        result.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());
        result
    }

    /// BFS 查找连通分量
    fn connected_components(&self) -> Vec<Vec<&str>> {
        let mut visited: HashSet<&str> = HashSet::new();
        let mut components = Vec::new();

        for node in self.adj.keys() {
            if visited.contains(node.as_str()) { continue; }

            // BFS 发现一个连通分量
            let mut component = Vec::new();
            let mut queue = VecDeque::new();
            queue.push_back(node.as_str());
            visited.insert(node.as_str());

            while let Some(current) = queue.pop_front() {
                component.push(current);
                if let Some(neighbors) = self.adj.get(current) {
                    for neighbor in neighbors {
                        if !visited.contains(neighbor.as_str()) {
                            visited.insert(neighbor.as_str());
                            queue.push_back(
                                // 安全：neighbor 的生命周期与 self 相同
                                self.adj.get_key_value(neighbor.as_str())
                                    .unwrap().0.as_str()
                            );
                        }
                    }
                }
            }
            component.sort(); // 稳定输出
            components.push(component);
        }
        components
    }
}

fn main() {
    let mut network = SocialGraph::new();

    // 社交圈 1
    for name in &["Alice", "Bob", "Carol", "Dave"] {
        network.add_person(name);
    }
    network.add_friendship("Alice", "Bob");
    network.add_friendship("Alice", "Carol");
    network.add_friendship("Bob", "Carol");
    network.add_friendship("Carol", "Dave");

    // 社交圈 2（孤立群体）
    for name in &["Eve", "Frank"] {
        network.add_person(name);
    }
    network.add_friendship("Eve", "Frank");

    // 度中心性分析
    println!("=== 度中心性排名 ===");
    for (name, centrality) in network.degree_centrality() {
        println!("  {}: {:.3}", name, centrality);
    }

    // 连通分量分析
    println!("\n=== 连通分量 ===");
    let components = network.connected_components();
    println!("共 {} 个社交圈", components.len());
    for (i, comp) in components.iter().enumerate() {
        println!("  圈 {}: {:?}", i + 1, comp);
    }
}

  Carol: 0.600
  Alice: 0.400
  Bob: 0.400
  Dave: 0.200
  Eve: 0.200

Carol 连接了 3 个人（Alice、Bob、Dave），在 6 个人的网络中度中心性为 3/5 = 0.6。 她是社交圈 1 中连接最紧密的"枢纽"节点。在真实社交网络中，这类人通常是信息传播的关键节点。

2.4.3 迷宫求解器

🤔

一个二维网格迷宫，0 表示通路，1 表示墙壁。从左上角走到右下角， 最短路径是什么？这其实就是图上的 BFS 问题—— 每个格子是节点，相邻可通行格子之间有边。

迷宫建模

将迷宫看作隐式图（implicit graph）：我们不需要显式构建邻接表， 而是在遍历时动态计算邻居。对于坐标 (r, c)， 它的邻居是上下左右四个方向中值为 0 的格子。

BFS 求最短路径

BFS 保证先找到的路径是最短的（在无权图中），因为它按"层"扩展。 我们只需在 BFS 过程中记录每个节点的前驱，最后回溯即可得到路径。

Rust maze_solver.rs 
use std::collections::VecDeque;

/// 迷宫求解器：BFS 找最短路径
fn solve_maze(maze: &[Vec<u8>]) -> Option<Vec<(usize, usize)>> {
    let rows = maze.len();
    if rows == 0 { return None; }
    let cols = maze[0].len();

    // 起点或终点是墙壁
    if maze[0][0] == 1 || maze[rows - 1][cols - 1] == 1 {
        return None;
    }

    // BFS 状态：visited 数组 + 前驱记录
    let mut visited = vec![vec![false; cols]; rows];
    let mut prev: Vec<Vec<Option<(usize, usize)>>> = vec![vec![None; cols]; rows];
    let mut queue = VecDeque::new();

    // 从起点开始
    visited[0][0] = true;
    queue.push_back((0usize, 0usize));

    // 四个方向：上、下、左、右
    let directions: [(i32, i32); 4] = [(-1, 0), (1, 0), (0, -1), (0, 1)];

    while let Some((r, c)) = queue.pop_front() {
        // 到达终点
        if r == rows - 1 && c == cols - 1 {
            // 回溯路径
            let mut path = Vec::new();
            let mut cur = Some((r, c));
            while let Some(pos) = cur {
                path.push(pos);
                cur = prev[pos.0][pos.1];
            }
            path.reverse();
            return Some(path);
        }

        // 扩展邻居
        for (dr, dc) in &directions {
            let nr = r as i32 + dr;
            let nc = c as i32 + dc;

            if nr >= 0 && nr < rows as i32
                && nc >= 0 && nc < cols as i32
            {
                let (nr, nc) = (nr as usize, nc as usize);
                if !visited[nr][nc] && maze[nr][nc] == 0 {
                    visited[nr][nc] = true;
                    prev[nr][nc] = Some((r, c));
                    queue.push_back((nr, nc));
                }
            }
        }
    }

    None // 无路径
}

/// 打印迷宫和路径
fn print_maze_with_path(maze: &[Vec<u8>], path: &[(usize, usize)]) {
    let path_set: std::collections::HashSet<(usize, usize)> =
        path.iter().copied().collect();

    for (r, row) in maze.iter().enumerate() {
        for (c, &cell) in row.iter().enumerate() {
            if path_set.contains(&(r, c)) {
                print!("★ ");
            } else if cell == 1 {
                print!("█ ");
            } else {
                print!("· ");
            }
        }
        println!();
    }
}

fn main() {
    let maze = vec![
        vec![0, 0, 1, 0, 0],
        vec![1, 0, 1, 0, 1],
        vec![0, 0, 0, 0, 0],
        vec![0, 1, 1, 1, 0],
        vec![0, 0, 0, 0, 0],
    ];

    println!("迷宫（0=通路, 1=墙壁）:");
    for row in &maze {
        for &cell in row {
            print!("{} ", if cell == 1 { "█" } else { "·" });
        }
        println!();
    }

    match solve_maze(&maze) {
        Some(path) => {
            println!("\n找到最短路径（长度 {}）:", path.len() - 1);
            print_maze_with_path(&maze, &path);
            println!("\n路径坐标: {:?}", path);
        }
        None => println!("无解！"),
    }
}

DFS 求解：找到任意一条路径

如果不需要最短路径，DFS 通常更节省内存（栈深度 vs 队列宽度）：

Rust maze_dfs.rs 
/// DFS 迷宫求解（找任意路径，不保证最短）
fn solve_maze_dfs(maze: &[Vec<u8>]) -> Option<Vec<(usize, usize)>> {
    let rows = maze.len();
    if rows == 0 { return None; }
    let cols = maze[0].len();

    let mut visited = vec![vec![false; cols]; rows];
    let mut path = Vec::new();

    fn dfs(
        maze: &[Vec<u8>],
        visited: &mut Vec<Vec<bool>>,
        path: &mut Vec<(usize, usize)>,
        r: usize, c: usize,
        target: (usize, usize),
    ) -> bool {
        if r == target.0 && c == target.1 {
            path.push((r, c));
            return true;
        }

        visited[r][c] = true;
        path.push((r, c));

        let directions: [(i32, i32); 4] = [(-1,0),(1,0),(0,-1),(0,1)];
        for (dr, dc) in &directions {
            let nr = r as i32 + dr;
            let nc = c as i32 + dc;
            if nr >= 0 && nr < maze.len() as i32
                && nc >= 0 && nc < maze[0].len() as i32
            {
                let (nr, nc) = (nr as usize, nc as usize);
                if !visited[nr][nc] && maze[nr][nc] == 0 {
                    if dfs(maze, visited, path, nr, nc, target) {
                        return true;
                    }
                }
            }
        }

        path.pop(); // 回溯
        false
    }

    if maze[0][0] == 1 { return None; }
    if dfs(maze, &mut visited, &mut path, 0, 0, (rows-1, cols-1)) {
        Some(path)
    } else {
        None
    }
}
输出 
DFS 找到路径（不一定最短）
特性 BFS 求解 DFS 求解
路径最短性 ✅ 保证最短 ❌ 不保证
空间复杂度 O(n \times m)（队列可能很宽） O(n \times m)（递归栈深度）
实际内存 通常更大（宽度优先） 通常更小（深度优先）
实现方式 VecDeque 迭代 递归 / 显式栈
适用场景 需要最短路径 只需判断是否可达

⚠️ Rust 递归深度限制
DFS 递归版在大迷宫上可能导致栈溢出。Rust 默认栈大小为 8MB。 对于超大迷宫（如 1000×1000），建议使用显式栈的迭代 DFS， 或使用 std::thread::Builder::new().stack_size() 增大栈空间。

2.4.4 性能分析与算法选择

🤔

你实现了 Dijkstra 和 BFS 两种最短路径算法。在什么情况下该用哪个？ 如何测量它们在实际数据上的性能差异？Rust 提供了哪些工具帮助我们做这些分析？

时间复杂度对比

算法 时间复杂度 空间复杂度 适用场景
BFS O(V + E) O(V) 无权图最短路径
DFS O(V + E) O(V) 连通性检测、拓扑排序
Dijkstra（二叉堆） O((V + E) \log V) O(V) 带权图最短路径（非负权）
Bellman-Ford O(V \cdot E) O(V) 含负权边的最短路径
Kruskal (MST) O(E \log E) O(V) 稀疏图的最小生成树
Prim (MST) O((V + E) \log V) O(V) 稠密图的最小生成树

use std::time::Instant;
use std::collections::{VecDeque, BinaryHeap, HashMap};
use std::cmp::Reverse;

/// 生成 n×n 的网格图（模拟迷宫）
fn generate_grid(n: usize) -> Vec<Vec<Vec<(usize, usize, u32)>>> {
    let mut grid = vec![vec![Vec::new(); n]; n];
    for r in 0..n {
        for c in 0..n {
            if r + 1 < n {
                grid[r][c].push((r + 1, c, 1));
                grid[r + 1][c].push((r, c, 1));
            }
            if c + 1 < n {
                grid[r][c].push((r, c + 1, 1));
                grid[r][c + 1].push((r, c, 1));
            }
        }
    }
    grid
}

/// BFS（无权图最短路径）
fn bfs_shortest(grid: &[Vec<Vec<(usize, usize, u32)>>], n: usize) -> u32 {
    let mut dist = vec![vec![u32::MAX; n]; n];
    let mut queue = VecDeque::new();
    dist[0][0] = 0;
    queue.push_back((0, 0));

    while let Some((r, c)) = queue.pop_front() {
        for &(nr, nc, _) in &grid[r][c] {
            if dist[nr][nc] == u32::MAX {
                dist[nr][nc] = dist[r][c] + 1;
                queue.push_back((nr, nc));
            }
        }
    }
    dist[n - 1][n - 1]
}

/// Dijkstra（带权图最短路径）
fn dijkstra_shortest(
    grid: &[Vec<Vec<(usize, usize, u32)>>], n: usize
) -> u32 {
    let mut dist = vec![vec![u32::MAX; n]; n];
    let mut heap = BinaryHeap::new();
    dist[0][0] = 0;
    heap.push(Reverse((0u32, 0usize, 0usize)));

    while let Some(Reverse((d, r, c))) = heap.pop() {
        if d > dist[r][c] { continue; }
        for &(nr, nc, w) in &grid[r][c] {
            let new_dist = d + w;
            if new_dist < dist[nr][nc] {
                dist[nr][nc] = new_dist;
                heap.push(Reverse((new_dist, nr, nc)));
            }
        }
    }
    dist[n - 1][n - 1]
}

fn main() {
    for &n in &[50, 100, 200, 500] {
        let grid = generate_grid(n);

        let start = Instant::now();
        let d1 = bfs_shortest(&grid, n);
        let bfs_time = start.elapsed();

        let start = Instant::now();
        let d2 = dijkstra_shortest(&grid, n);
        let dijk_time = start.elapsed();

        println!("网格 {}×{}: BFS={:?} Dijkstra={:?} (距离={}, {})",
            n, n, bfs_time, dijk_time, d1, d2);
    }
}

📝 章节总结

  • petgraph 是 Rust 生态中最成熟的图库，提供 Graph、DiGraph、UnGraph、GraphMap 等多种类型和丰富的内置算法
  • 度中心性衡量节点的连接重要性：C_D(v) = \frac{\deg(v)}{n-1}
  • 连通分量将图分割为独立的连通子图，用 BFS/DFS 可在 O(V+E) 时间内找到所有分量
  • 迷宫求解本质是图的遍历——BFS 保证最短路径，DFS 更省内存但不保证最短
  • 算法选择取决于问题类型和图特征：无权→BFS，非负权→Dijkstra，负权→Bellman-Ford
  • 性能分析使用 std::time::Instant、criterion 或系统级工具（perf、flamegraph）验证理论分析
  • Rust 的零成本抽象和高效标准库使其成为图算法实现的优秀选择

2. 在社交网络分析代码中，添加一个 shortest_path 方法，使用 BFS 找两个人之间的最短社交路径（"六度分隔"问题）。

从起始节点 BFS，记录前驱节点。当到达目标节点时，回溯前驱得到路径。如果 BFS 结束都没有到达目标，返回 None。

⭐ 基础

3. 修改迷宫求解器，支持对角线移动（8 个方向而非 4 个）。对角线移动的代价是否应该和直线移动相同？为什么？

将方向数组扩展为 8 个：加入 (-1,-1), (-1,1), (1,-1), (1,1)。 对角线移动的实际距离是 \sqrt{2} \approx 1.414， 如果要精确计算最短路径，应使用 Dijkstra 而非 BFS，并设置对角线权重为 1.414。 如果简化为等权，BFS 仍可使用但距离不是欧几里得意义上的最短。

⭐ 基础

4. 解释 BFS 和 DFS 在迷宫求解中的行为差异。画出一个 3×3 迷宫的例子，展示两者可能找到不同路径。

BFS 逐层扩展，优先探索近距离格子；DFS 沿一条路走到底再回溯。 在有多条等长路径的迷宫中，BFS 会找到其中一条最短的， 而 DFS 可能绕远路。例如全通的 3×3 网格中，BFS 找到的路径长度为 4， DFS 可能走出长度为 8 的蛇形路径。

⭐ 基础

5. petgraph 中 Graph 和 StableGraph 的主要区别是什么？在什么场景下应该选择 StableGraph？

Graph 在删除节点时会交换最后一个节点到被删位置（类似 Vec::swap_remove）， 导致 NodeIndex 失效。StableGraph 保持索引稳定——删除节点后其他节点的索引不变。 当你需要在算法过程中删除节点，同时保持对其他节点的引用时，应使用 StableGraph。

⭐⭐ 进阶

6. 实现介数中心性（Betweenness Centrality）：计算每个节点出现在多少对其他节点之间的最短路径上。 介数高的节点是网络的"桥梁"。

对每对节点 (s, t) 运行 BFS 找最短路径，统计每个中间节点被经过的次数。 归一化：除以 (n-1)(n-2)/2（无向图）。 时间复杂度 O(V × (V + E))。优化版本可使用 Brandes 算法（O(VE)）。

⭐⭐ 进阶

7. 为迷宫求解器实现 A* 算法。使用曼哈顿距离作为启发函数。 对比 A* 和 BFS 在大迷宫上的性能差异（访问节点数）。

A* 使用 f(n) = g(n) + h(n)，其中 g(n) 是实际代价，h(n) 是启发函数（曼哈顿距离）。 使用 BinaryHeap + Reverse 实现最小堆。曼哈顿距离：|r1 - r2| + |c1 - c2|。 A* 通常比 BFS 访问更少的节点（被启发函数引导向目标方向）， 但在无障碍网格上差异不大；在复杂迷宫中优势明显。

⭐⭐ 进阶

8. 实现随机迷宫生成器：使用 DFS 随机打通墙壁（Recursive Backtracker 算法）， 生成一个保证有解的完美迷宫（任意两点之间恰好有一条路径）。

初始化全为墙的网格。从起点开始 DFS，每次随机选择一个未访问的邻居， 打通之间的墙壁。当没有未访问邻居时回溯。这本质上是在网格图上生成随机生成树。 使用 rand::seq::SliceRandom 的 shuffle 方法随机化方向。

⭐⭐ 进阶

9. 使用 petgraph 的 toposort 实现一个简单的构建系统：给定一组任务和依赖关系， 输出合法的执行顺序。检测循环依赖并给出有意义的错误信息。

创建 DiGraph，每个任务是节点，依赖关系是边（A 依赖 B → B → A）。 调用 petgraph::algo::toposort(&graph, None)。 如果返回 Err(cycle)，说明有循环依赖，可以用 cycle 中的 NodeId 来报告是哪个任务引起的。 成功时反转结果得到执行顺序。

⭐⭐⭐ 挑战

10. 实现强连通分量（Tarjan 算法或 Kosaraju 算法）。 在有向图中找出所有的强连通分量，并构建 SCC 的缩点图（DAG）。 分析算法的时间复杂度。

Tarjan 算法：维护 DFS 栈和 low-link 值。 对每个未访问节点执行 DFS，记录发现时间 disc[v] 和最小可达祖先 low[v]。 当 disc[v] == low[v] 时，弹出栈中 v 以上的所有节点组成一个 SCC。 时间复杂度 O(V + E)。缩点后得到 DAG，可以做拓扑排序。

⭐⭐⭐ 挑战

11. 对比手写邻接表实现和 petgraph 在以下操作上的性能： (a) 创建 10000 节点的随机图；(b) BFS 遍历全图；(c) 查找所有连通分量。 分析性能差异的原因（内存布局？间接层？泛型单态化？）。

手写 Vec<Vec<usize>> 通常在简单场景下略快，因为没有 petgraph 的泛型封装开销和节点/边索引检查。 但 petgraph 的内部使用了连续内存布局（类似 CSR），对缓存更友好。 用 std::time::Instant 或 criterion 测量，重复多次取平均值。 通常差异在 10-30% 以内，petgraph 的 API 安全性和丰富算法远超手写实现的便利性。
