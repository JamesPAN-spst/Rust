2.2.1 图的表示

🤔 想一想

在社交网络中，每个用户是一个节点，好友关系是一条边。如果有 1000 万用户，你会怎么在代码中高效地存储和查询这些关系？直接用一个 1000万 × 1000万 的矩阵？还是有更好的方式？

图的基本定义

图（Graph）是由顶点（Vertex）和边（Edge）构成的数学结构。形式化地，一个图定义为：

G = (V, E)

其中 V 是顶点的有限集合，E \subseteq V \times V 是边的集合。

图的分类

  • 无向图（Undirected Graph）：边没有方向，(u, v) = (v, u)。例如：好友关系。
  • 有向图（Directed Graph / Digraph）：边有方向，(u, v) \neq (v, u)。例如：Twitter 关注关系。
  • 加权图（Weighted Graph）：每条边有一个权重 w(u, v)。例如：城市之间的距离。
  • 无权图（Unweighted Graph）：所有边的权重相同（或视为 1）。

对于无向图，度（degree）是与顶点相连的边数。对于有向图，分为入度（in-degree）和出度（out-degree）。图论中一个基本定理——握手定理告诉我们：

\sum_{v \in V} \deg(v) = 2|E|

邻接矩阵 vs 邻接表

图在计算机中最常用的两种表示方式是邻接矩阵和邻接表。选择哪种取决于图的稀疏程度和操作需求。

特性 邻接矩阵 邻接表
空间复杂度 O(V^2) O(V + E)
查询边 (u, v) 是否存在 O(1) O(\deg(u))
遍历顶点 u 的所有邻居 O(V) O(\deg(u))
添加边 O(1) O(1)
适用场景 稠密图，频繁查询边 稀疏图，频繁遍历邻居
Rust 实现 Vec<Vec<i32>> Vec<Vec<usize>> 或 HashMap
Rust 邻接矩阵实现 
/// 使用邻接矩阵表示无向加权图
struct AdjMatrix {
    size: usize,
    matrix: Vec<Vec<i32>>, // 0 表示无边，其他值表示权重
}

impl AdjMatrix {
    fn new(size: usize) -> Self {
        AdjMatrix {
            size,
            matrix: vec![vec![0; size]; size],
        }
    }

    fn add_edge(&mut self, u: usize, v: usize, weight: i32) {
        self.matrix[u][v] = weight;
        self.matrix[v][u] = weight; // 无向图，对称
    }

    fn has_edge(&self, u: usize, v: usize) -> bool {
        self.matrix[u][v] != 0
    }

    fn neighbors(&self, u: usize) -> Vec<usize> {
        (0..self.size)
            .filter(|&v| self.matrix[u][v] != 0)
            .collect()
    }
}

fn main() {
    let mut graph = AdjMatrix::new(4);
    graph.add_edge(0, 1, 1);
    graph.add_edge(0, 2, 1);
    graph.add_edge(1, 3, 1);
    graph.add_edge(2, 3, 1);

    println!("边 (0,1) 存在: {}", graph.has_edge(0, 1));
    println!("边 (0,3) 存在: {}", graph.has_edge(0, 3));
    println!("顶点 0 的邻居: {:?}", graph.neighbors(0));
}
输出 
边 (0,1) 存在: true
边 (0,3) 存在: false
顶点 0 的邻居: [1, 2]
Rust 邻接表实现 
use std::collections::HashMap;

/// 使用邻接表表示有向加权图
struct AdjList {
    edges: HashMap<usize, Vec<(usize, i32)>>, // 邻居节点 + 权重
}

impl AdjList {
    fn new() -> Self {
        AdjList {
            edges: HashMap::new(),
        }
    }

    fn add_vertex(&mut self, v: usize) {
        self.edges.entry(v).or_insert_with(Vec::new);
    }

    fn add_edge(&mut self, from: usize, to: usize, weight: i32) {
        self.edges.entry(from).or_default().push((to, weight));
    }

    fn add_undirected_edge(&mut self, u: usize, v: usize, weight: i32) {
        self.add_edge(u, v, weight);
        self.add_edge(v, u, weight);
    }

    fn neighbors(&self, v: usize) -> &[(usize, i32)] {
        self.edges.get(&v).map_or(&[], |v| v.as_slice())
    }
}

fn main() {
    let mut graph = AdjList::new();
    graph.add_undirected_edge(0, 1, 4);
    graph.add_undirected_edge(0, 2, 1);
    graph.add_undirected_edge(1, 3, 2);
    graph.add_undirected_edge(2, 3, 5);

    println!("顶点 0 的邻居: {:?}", graph.neighbors(0));
    println!("顶点 2 的邻居: {:?}", graph.neighbors(2));
}
输出 
顶点 0 的邻居: [(1, 4), (2, 1)]
顶点 2 的邻居: [(0, 1), (3, 5)]
💡 选择建议

在实际 Rust 项目中，邻接表是更常用的选择。大多数真实世界的图都是稀疏的（社交网络中每人平均几百个好友，远小于总用户数）。Vec<Vec<(usize, i32)>> 在性能上优于 HashMap 版本，因为连续内存访问对 CPU 缓存更友好。生产环境中推荐使用 petgraph crate。

2.2.2 图遍历

🤔 想一想

在一个网络爬虫程序中，你需要从一个种子 URL 出发，发现所有可达的网页。你会选择先把当前页面的所有链接都探索完再深入，还是沿着一条链接一直探索到底？这两种策略对应着两种经典的图遍历方式。

广度优先搜索 (BFS)

BFS 像水波纹一样，从起点开始逐层向外扩展。它使用队列（FIFO）来管理待访问的节点，保证先发现的节点先被处理。

初始化

将起始节点放入队列，标记为已访问。创建一个 visited 集合来避免重复访问。

出队处理

从队列头部取出一个节点，处理该节点（例如打印、记录距离等）。

扩展邻居

遍历当前节点的所有未访问邻居，标记为已访问并加入队列尾部。

重复

重复步骤 2-3，直到队列为空。此时所有可达节点都已被访问。

BFS 的关键性质：对于无权图，BFS 能保证找到从起点到每个节点的最短路径（最少边数）。时间复杂度 O(V + E)，空间复杂度 O(V)。

深度优先搜索 (DFS)

DFS 像探险家一样，沿着一条路径一直走到底，走不通再回溯。它使用栈（LIFO，可用递归的调用栈或显式栈实现）。

初始化

将起始节点压入栈（或直接开始递归调用），标记为已访问。

取栈顶节点

弹出栈顶元素并处理。在递归版本中，这对应当前函数调用的节点。

深入邻居

对当前节点的每个未访问邻居，递归地或通过压栈来继续深入探索。

回溯

当一个节点的所有邻居都已访问时，自动回溯到上一个节点继续探索其他分支。

DFS 适合用于：检测环、拓扑排序、连通分量、迷宫求解等。时间复杂度同为 O(V + E)。

特性 BFS DFS
数据结构 队列 (VecDeque) 栈 (递归 / Vec)
遍历顺序 逐层扩展 一路到底再回溯
最短路径（无权图） ✅ 保证 ❌ 不保证
空间复杂度 O(V)（最坏情况存整层） O(V)（最坏情况存整条路径）
典型应用 最短路径、层序遍历 环检测、拓扑排序、连通性
Rust BFS 实现 
use std::collections::VecDeque;

/// BFS 遍历，返回从 start 出发可达的所有节点（按访问顺序）
fn bfs(graph: &Vec<Vec<usize>>, start: usize) -> Vec<usize> {
    let n = graph.len();
    let mut visited = vec![false; n];
    let mut order = Vec::new();
    let mut queue = VecDeque::new();

    visited[start] = true;
    queue.push_back(start);

    while let Some(node) = queue.pop_front() {
        order.push(node);
        for &neighbor in &graph[node] {
            if !visited[neighbor] {
                visited[neighbor] = true;
                queue.push_back(neighbor);
            }
        }
    }
    order
}

fn main() {
    // 邻接表：0-1, 0-2, 1-3, 2-3, 3-4
    let graph = vec![
        vec![1, 2],    // 0 的邻居
        vec![0, 3],    // 1 的邻居
        vec![0, 3],    // 2 的邻居
        vec![1, 2, 4], // 3 的邻居
        vec![3],       // 4 的邻居
    ];

    let result = bfs(&graph, 0);
    println!("BFS 遍历顺序: {:?}", result);
}
输出 
BFS 遍历顺序: [0, 1, 2, 3, 4]
Rust DFS 实现（递归版 + 迭代版） 
/// DFS 递归版
fn dfs_recursive(
    graph: &Vec<Vec<usize>>,
    node: usize,
    visited: &mut Vec<bool>,
    order: &mut Vec<usize>,
) {
    visited[node] = true;
    order.push(node);
    for &neighbor in &graph[node] {
        if !visited[neighbor] {
            dfs_recursive(graph, neighbor, visited, order);
        }
    }
}

/// DFS 迭代版（使用显式栈）
fn dfs_iterative(graph: &Vec<Vec<usize>>, start: usize) -> Vec<usize> {
    let n = graph.len();
    let mut visited = vec![false; n];
    let mut order = Vec::new();
    let mut stack = vec![start];

    while let Some(node) = stack.pop() {
        if visited[node] {
            continue;
        }
        visited[node] = true;
        order.push(node);
        // 逆序压栈以保证与递归版本相同的遍历顺序
        for &neighbor in graph[node].iter().rev() {
            if !visited[neighbor] {
                stack.push(neighbor);
            }
        }
    }
    order
}

fn main() {
    let graph = vec![
        vec![1, 2],    // 0
        vec![0, 3],    // 1
        vec![0, 3],    // 2
        vec![1, 2, 4], // 3
        vec![3],       // 4
    ];

    // 递归版
    let mut visited = vec![false; graph.len()];
    let mut order = Vec::new();
    dfs_recursive(&graph, 0, &mut visited, &mut order);
    println!("DFS 递归: {:?}", order);

    // 迭代版
    let order2 = dfs_iterative(&graph, 0);
    println!("DFS 迭代: {:?}", order2);
}
输出 
DFS 递归: [0, 1, 3, 2, 4]
DFS 迭代: [0, 1, 3, 2, 4]
🦀 Rust 技巧

std::collections::VecDeque 是 Rust 标准库中的双端队列，底层使用环形缓冲区实现，push_back 和 pop_front 均为 O(1)，非常适合 BFS。避免使用 Vec 的 remove(0) 作为队列——它是 O(n) 的！

2.2.3 最短路径

🤔 想一想

你在使用 GPS 导航时，它是如何在数百万条道路中几乎瞬间找到最短路线的？背后的核心算法之一就是 Dijkstra——一种贪心策略的最短路径算法。

Dijkstra 算法

Dijkstra 算法解决单源最短路径问题：给定一个起点，求到图中所有其他顶点的最短距离。它的核心思想是贪心：每次选择当前距离最小的未确定节点，确定其最短距离，然后更新其邻居。

松弛操作（Relaxation）是算法的核心：

d[v] = \min(d[v],\ d[u] + w(u, v))

如果经过节点 u 到达 v 的路径比当前已知的最短路径更短，则更新 d[v]。

初始化

设起点 s 的距离 d[s] = 0，其余所有顶点 d[v] = \infty。将 (0, s) 放入优先队列（最小堆）。

取最小距离节点

从优先队列中弹出距离最小的节点 u。如果 u 已确认最短距离，跳过。

松弛相邻边

对 u 的每个邻居 v，如果 d[u] + w(u,v) < d[v]，更新 d[v] 并将 (d[v], v) 加入优先队列。

重复

重复步骤 2-3，直到优先队列为空或所有节点都已确认。

使用二叉堆实现的 Dijkstra 时间复杂度为 O((V + E) \log V)。

Rust Dijkstra 算法实现 
use std::cmp::Reverse;
use std::collections::BinaryHeap;

const INF: i64 = i64::MAX;

/// Dijkstra 最短路径算法
/// 返回从 start 到每个节点的最短距离
fn dijkstra(graph: &Vec<Vec<(usize, i64)>>, start: usize) -> Vec<i64> {
    let n = graph.len();
    let mut dist = vec![INF; n];
    dist[start] = 0;

    // Rust 的 BinaryHeap 是最大堆，用 Reverse 包装变成最小堆
    let mut heap = BinaryHeap::new();
    heap.push(Reverse((0i64, start)));

    while let Some(Reverse((cost, u))) = heap.pop() {
        // 如果当前路径已经不是最优，跳过
        if cost > dist[u] {
            continue;
        }
        for &(v, w) in &graph[u] {
            let new_dist = dist[u] + w;
            if new_dist < dist[v] {
                dist[v] = new_dist;
                heap.push(Reverse((new_dist, v)));
            }
        }
    }
    dist
}

fn main() {
    // 有向加权图: 5 个顶点
    // 0 --(4)--> 1, 0 --(1)--> 2
    // 1 --(1)--> 3
    // 2 --(2)--> 1, 2 --(5)--> 3
    // 3 --(3)--> 4
    let graph = vec![
        vec![(1, 4), (2, 1)],  // 0
        vec![(3, 1)],          // 1
        vec![(1, 2), (3, 5)],  // 2
        vec![(4, 3)],          // 3
        vec![],                // 4
    ];

    let dist = dijkstra(&graph, 0);
    for (i, d) in dist.iter().enumerate() {
        let d_str = if *d == INF {
            "不可达".to_string()
        } else {
            d.to_string()
        };
        println!("0 -> {}: {}", i, d_str);
    }
}
输出 
0 -> 0: 0
0 -> 1: 3
0 -> 2: 1
0 -> 3: 4
0 -> 4: 7
⚠️ 注意

Dijkstra 算法不能处理负权边！如果图中存在负权边，贪心策略会失效——已经"确认"最短的节点可能后续通过负权边被更新为更短。此时应使用 Bellman-Ford 算法（时间复杂度 O(VE)）或 SPFA。

2.2.4 最小生成树

🤔 想一想

一家网络公司需要用光缆连接 10 个城市的数据中心。每两个城市之间铺设光缆的成本不同。如何选择线路，使得所有数据中心连通，且总铺设成本最低？

最小生成树 (MST) 定义

对于一个连通无向加权图 G = (V, E)，最小生成树是 G 的一个子图 T = (V, E')，满足：

  • T 是树（连通且无环）
  • T 包含 G 的所有顶点
  • 边的总权重最小：
    \min \sum_{(u,v) \in E'} w(u, v)

一棵生成树恰好有 |V| - 1 条边。

Prim 算法

Prim 算法类似 Dijkstra：从任意一个顶点开始，每次选择连接已选集合和未选集合的最小权边，将新顶点加入已选集合。重复直到所有顶点都被选中。时间复杂度使用二叉堆时为 O((V + E) \log V)。

Kruskal 算法

Kruskal 算法从另一个角度出发：将所有边按权重排序，从小到大依次尝试加入。如果加入一条边不会形成环，就加入；否则跳过。判断"是否形成环"正是 Union-Find（并查集）数据结构的拿手好戏。

排序所有边

按权重从小到大排序。

初始化 Union-Find

每个顶点独立为一个集合。

遍历边

对于每条边 (u, v, w)，如果 u 和 v 不在同一个集合中（find(u) != find(v)），则加入该边并合并两个集合。

终止

当已选边数等于 |V| - 1 时，MST 构建完成。

Kruskal 的时间复杂度主要来自排序：O(E \log E)。

Rust Union-Find（并查集）实现 
/// 并查集（Union-Find），支持路径压缩和按秩合并
struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<usize>,
}

impl UnionFind {
    fn new(n: usize) -> Self {
        UnionFind {
            parent: (0..n).collect(), // 初始时每个节点的父节点是自身
            rank: vec![0; n],
        }
    }

    /// 查找根节点，带路径压缩
    fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]);
        }
        self.parent[x]
    }

    /// 按秩合并两个集合，返回是否成功合并（false 表示已在同一集合）
    fn union(&mut self, x: usize, y: usize) -> bool {
        let rx = self.find(x);
        let ry = self.find(y);
        if rx == ry {
            return false;
        }
        match self.rank[rx].cmp(&self.rank[ry]) {
            std::cmp::Ordering::Less => self.parent[rx] = ry,
            std::cmp::Ordering::Greater => self.parent[ry] = rx,
            std::cmp::Ordering::Equal => {
                self.parent[ry] = rx;
                self.rank[rx] += 1;
            }
        }
        true
    }
}

fn main() {
    let mut uf = UnionFind::new(5);
    uf.union(0, 1);
    uf.union(2, 3);
    println!("0 和 1 连通: {}", uf.find(0) == uf.find(1)); // true

Rust Kruskal 最小生成树算法 
/// 边：(权重, 起点, 终点)
type Edge = (i64, usize, usize);

/// Kruskal 算法求最小生成树
/// 返回 MST 的边集和总权重
fn kruskal(n: usize, edges: &mut Vec<Edge>) -> (Vec<Edge>, i64) {
    edges.sort(); // 按权重排序

    let mut uf = UnionFind::new(n);
    let mut mst = Vec::new();
    let mut total_weight = 0i64;

    for &(w, u, v) in edges.iter() {
        if uf.union(u, v) {
            mst.push((w, u, v));
            total_weight += w;
            if mst.len() == n - 1 {
                break; // MST 完成
            }
        }
    }
    (mst, total_weight)
}

// UnionFind 结构体同上一个代码块（此处省略以避免重复）

fn main() {
    let n = 5; // 顶点数
    let mut edges: Vec<Edge> = vec![
        (4, 0, 1),
        (1, 0, 2),
        (3, 1, 2),
        (2, 1, 3),
        (5, 2, 3),
        (7, 3, 4),
        (6, 2, 4),
    ];

    let (mst, total) = kruskal(n, &mut edges);
    println!("MST 的边:");
    for (w, u, v) in &mst {
        println!("  {} -- {} (权重: {})", u, v, w);
    }
    println!("总权重: {}", total);

📝 本章总结

  • 图的表示：邻接矩阵适合稠密图和频繁边查询（O(1)），邻接表适合稀疏图和邻居遍历（空间 O(V+E)）。
  • BFS：使用队列逐层扩展，保证无权图最短路径，时间 O(V+E)。Rust 中用 VecDeque。
  • DFS：使用栈或递归深入探索，适合环检测、拓扑排序，时间 O(V+E)。
  • Dijkstra：贪心策略求加权图单源最短路径，不可处理负权边，使用 BinaryHeap + Reverse 实现最小堆，时间 O((V+E) \log V)。
  • 最小生成树：Kruskal 按边权排序 + Union-Find，Prim 类似 Dijkstra 逐步扩展。
  • Union-Find：路径压缩 + 按秩合并，近乎 O(1) 的查找和合并，是图算法中的经典辅助数据结构。

⭐ 基础

有向图中，顶点的入度和出度有什么区别？如果一个 DAG（有向无环图）有一个顶点入度为 0，这意味着什么？

出度是从该顶点出发的边数，入度是指向该顶点的边数。

入度为 0 的顶点没有任何前驱，在 DAG 中代表一个"起始"节点——没有依赖项。例如在编译依赖图中，入度为 0 的模块可以最先编译。这正是拓扑排序的起点选择策略。

⭐ 基础

解释为什么 Dijkstra 算法不能处理负权边。举一个具体的反例（3个顶点即可）。

Dijkstra 的贪心策略假设：一旦从优先队列中弹出一个节点，它的最短距离已经确定。但负权边会违反这个假设。

反例：顶点 A, B, C。边 A→B(1), A→C(3), B→C(-5)。Dijkstra 从 A 开始：先确定 d[B]=1，然后确定 d[C]=3。但实际上 A→B→C 的路径长度为 1+(-5)=-4，比直接 A→C(3) 更短。贪心策略无法回溯修正已确认的 d[C]。

⭐⭐ 进阶

用 Rust 实现一个函数 has_cycle，使用 DFS 检测有向图中是否存在环。提示：需要三种状态（未访问、正在访问、已完成）。

#[derive(Clone, PartialEq)]
enum State { White, Gray, Black }

fn has_cycle(graph: &Vec<Vec<usize>>) -> bool {
    let n = graph.len();
    let mut state = vec![State::White; n];

    for i in 0..n {
        if state[i] == State::White {
            if dfs_cycle(graph, i, &mut state) {
                return true;
            }
        }
    }
    false
}

fn dfs_cycle(graph: &Vec<Vec<usize>>, u: usize, state: &mut Vec<State>) -> bool {
    state[u] = State::Gray; // 正在访问
    for &v in &graph[u] {
        if state[v] == State::Gray {
            return true; // 发现回边，存在环
        }
        if state[v] == State::White && dfs_cycle(graph, v, state) {
            return true;
        }
    }
    state[u] = State::Black; // 完成访问
    false
}

Gray 节点表示在当前 DFS 路径上。如果遇到 Gray 邻居，说明存在环。

⭐⭐ 进阶

修改 BFS 代码，使其不仅返回遍历顺序，还返回从起点到每个节点的最短距离（边数）和前驱节点（用于还原路径）。

use std::collections::VecDeque;

fn bfs_with_path(
    graph: &Vec<Vec<usize>>,
    start: usize,
) -> (Vec<i32>, Vec<Option<usize>>) {
    let n = graph.len();
    let mut dist = vec![-1i32; n]; // -1 表示不可达
    let mut prev: Vec<Option<usize>> = vec![None; n];
    let mut queue = VecDeque::new();

    dist[start] = 0;
    queue.push_back(start);

    while let Some(u) = queue.pop_front() {
        for &v in &graph[u] {
            if dist[v] == -1 {
                dist[v] = dist[u] + 1;
                prev[v] = Some(u);
                queue.push_back(v);
            }
        }
    }
    (dist, prev)
}

// 还原从 start 到 target 的路径
fn reconstruct_path(prev: &[Option<usize>], target: usize) -> Vec<usize> {
    let mut path = Vec::new();
    let mut current = Some(target);
    while let Some(node) = current {
        path.push(node);
        current = prev[node];
    }
    path.reverse();
    path
}
⭐⭐ 进阶

实现一个函数，使用 DFS 对 DAG 进行拓扑排序。返回一个顶点的线性排序，使得对于每条有向边 (u, v)，u 都在 v 之前。

fn topological_sort(graph: &Vec<Vec<usize>>) -> Vec<usize> {
    let n = graph.len();
    let mut visited = vec![false; n];
    let mut order = Vec::new();

    for i in 0..n {
        if !visited[i] {
            topo_dfs(graph, i, &mut visited, &mut order);
        }
    }
    order.reverse(); // DFS 后序的逆序即为拓扑排序
    order
}

fn topo_dfs(
    graph: &Vec<Vec<usize>>,
    u: usize,
    visited: &mut Vec<bool>,
    order: &mut Vec<usize>,
) {
    visited[u] = true;
    for &v in &graph[u] {
        if !visited[v] {
            topo_dfs(graph, v, visited, order);
        }
    }
    order.push(u); // 后序：所有后继都已处理后再加入
}

关键思想：DFS 完成时（所有后继都已访问），将节点加入结果。最后反转结果即得拓扑排序。

⭐⭐ 进阶

修改 Dijkstra 实现，使其不仅计算最短距离，还能还原最短路径本身。给定起点 0 和终点 4（使用本章中的示例图），输出具体路径。

use std::cmp::Reverse;
use std::collections::BinaryHeap;

fn dijkstra_with_path(
    graph: &Vec<Vec<(usize, i64)>>,
    start: usize,
) -> (Vec<i64>, Vec<Option<usize>>) {
    let n = graph.len();
    let mut dist = vec![i64::MAX; n];
    let mut prev: Vec<Option<usize>> = vec![None; n];
    let mut heap = BinaryHeap::new();

    dist[start] = 0;
    heap.push(Reverse((0i64, start)));

    while let Some(Reverse((cost, u))) = heap.pop() {
        if cost > dist[u] { continue; }
        for &(v, w) in &graph[u] {
            let nd = dist[u] + w;
            if nd < dist[v] {
                dist[v] = nd;
                prev[v] = Some(u);
                heap.push(Reverse((nd, v)));
            }
        }
    }
    (dist, prev)
}
// 使用本章示例图：最短路径 0 → 2 → 1 → 3 → 4，总距离 7
⭐⭐⭐ 挑战

实现 Bellman-Ford 算法：处理可能含负权边的图的单源最短路径。额外要求：能检测负权环（如果存在负权环，返回错误而非距离数组）。

fn bellman_ford(
    n: usize,
    edges: &[(usize, usize, i64)], // (from, to, weight)
    start: usize,
) -> Result<Vec<i64>, &'static str> {
    let mut dist = vec![i64::MAX; n];
    dist[start] = 0;

    // 松弛 V-1 轮
    for _ in 0..n - 1 {
        for &(u, v, w) in edges {
            if dist[u] != i64::MAX && dist[u] + w < dist[v] {
                dist[v] = dist[u] + w;
            }
        }
    }

    // 第 V 轮检测负权环
    for &(u, v, w) in edges {
        if dist[u] != i64::MAX && dist[u] + w < dist[v] {
            return Err("图中存在负权环");
        }
    }

    Ok(dist)
}

时间复杂度 O(VE)。如果第 V 轮仍能松弛某条边，说明存在负权环。

⭐⭐⭐ 挑战

使用 BFS 实现二分图检测：判断一个无向图是否是二分图（可以将顶点分为两组，使得每条边的两个端点分别属于不同组）。提示：尝试对图进行二着色。

use std::collections::VecDeque;

fn is_bipartite(graph: &Vec<Vec<usize>>) -> bool {
    let n = graph.len();
    let mut color: Vec<i8> = vec![-1; n]; // -1 未染色，0 和 1 两种颜色

    for start in 0..n {
        if color[start] != -1 { continue; }
        color[start] = 0;
        let mut queue = VecDeque::new();
        queue.push_back(start);

        while let Some(u) = queue.pop_front() {
            for &v in &graph[u] {
                if color[v] == -1 {
                    color[v] = 1 - color[u]; // 染相反颜色
                    queue.push_back(v);
                } else if color[v] == color[u] {
                    return false; // 相邻同色，非二分图
                }
            }
        }
    }
    true
}

如果 BFS 过程中发现相邻节点已被染成相同颜色，则图不是二分图。注意要处理不连通图（外层循环遍历所有未染色节点）。

🏆 附加

实现 A* 搜索算法——Dijkstra 的启发式扩展。以二维网格为例（可以上下左右移动，某些格子是障碍），使用曼哈顿距离作为启发函数，找到从左上角到右下角的最短路径。分析：在什么条件下 A* 保证找到最优解？

A* 的核心公式是 f(n) = g(n) + h(n)，其中 g(n) 是从起点到 n 的实际代价，h(n) 是从 n 到目标的启发式估计。

use std::cmp::Reverse;
use std::collections::BinaryHeap;

fn manhattan_distance(a: (usize, usize), b: (usize, usize)) -> i64 {
    ((a.0 as i64) - (b.0 as i64)).abs()
    + ((a.1 as i64) - (b.1 as i64)).abs()
}

fn a_star(grid: &Vec<Vec<bool>>, start: (usize, usize), goal: (usize, usize)) -> Option<i64> {
    let rows = grid.len();
    let cols = grid[0].len();
    let mut g_score = vec![vec![i64::MAX; cols]; rows];
    g_score[start.0][start.1] = 0;

    // (f_score, row, col)
    let mut heap = BinaryHeap::new();
    let h = manhattan_distance(start, goal);
    heap.push(Reverse((h, start.0, start.1)));

    let dirs: [(i64, i64); 4] = [(-1,0),(1,0),(0,-1),(0,1)];

    while let Some(Reverse((_f, r, c))) = heap.pop() {
        if (r, c) == goal { return Some(g_score[r][c]); }
        for (dr, dc) in &dirs {
            let nr = r as i64 + dr;
            let nc = c as i64 + dc;
            if nr < 0 || nc < 0 { continue; }
            let (nr, nc) = (nr as usize, nc as usize);
            if nr >= rows || nc >= cols || grid[nr][nc] { continue; }
            let ng = g_score[r][c] + 1;
            if ng < g_score[nr][nc] {
                g_score[nr][nc] = ng;
                let f = ng + manhattan_distance((nr, nc), goal);
                heap.push(Reverse((f, nr, nc)));
            }
        }
    }
    None
}

最优性条件：当启发函数 h(n) 是可容许的（admissible，即不会高估实际代价）时，A* 保证找到最优解。曼哈顿距离在只能四方向移动的网格中是可容许的。
