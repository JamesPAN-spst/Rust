1.4.1 nalgebra 库入门

🤔

在前几章中，我们手写了 Vec2、Vec3 和 Matrix3。但在真实项目中， 你不会每次都从零开始——就像你不会手写 HTTP 解析器一样。 nalgebra 是 Rust 生态中最成熟的线性代数库， 被 Rapier 物理引擎和 Bevy 游戏引擎广泛使用。 那么，它和我们手写的版本有什么不同？

创建项目并添加依赖

首先用 Cargo 创建一个新项目，并添加 nalgebra 依赖：

核心类型概览

nalgebra 的类型系统围绕一个核心泛型 Matrix<T, R, C, S> 展开， 但日常使用中你只需要记住几个类型别名：

nalgebra 类型 我们手写的等价物 说明
Vector3<f64> Vec3 { x, y, z } 3D 列向量
Matrix3<f64> Mat3 { data: [[f64; 3]; 3] } 3×3 矩阵
Matrix4<f64> （未实现） 4×4 矩阵，用于齐次坐标
Point3<f64> （未实现） 3D 点——自带齐次坐标语义
Rotation3<f64> （未实现） 3D 旋转（内部存储旋转矩阵）
Isometry3<f64> （未实现） 刚体变换 = 旋转 + 平移

基本操作：从手写到 nalgebra

让我们把 Ch 1.1 的手写向量运算用 nalgebra 重写，体会 API 的简洁：

Rust nalgebra_basics.rs 
use nalgebra::{Vector3, Matrix3};

fn main() {
    // === 向量操作 ===
    let a = Vector3::new(3.0_f64, 2.0, 1.0);
    let b = Vector3::new(1.0_f64, -4.0, 2.0);

    // 加法、减法、数乘——运算符已重载
    println!("a + b = {}", a + b);
    println!("a - b = {}", a - b);
    println!("2 * a = {}", 2.0 * a);

    // 点积和叉积——方法调用
    println!("a · b = {}", a.dot(&b));
    println!("a × b = {}", a.cross(&b));

    // 长度和单位化
    println!("|a| = {:.4}", a.magnitude());
    println!("â = {}", a.normalize());

    // === 矩阵操作 ===
    let m = Matrix3::new(
        1.0, 2.0, 3.0,  // 第一行
        4.0, 5.0, 6.0,  // 第二行
        7.0, 8.0, 9.0,  // 第三行
    );

    // 矩阵 × 向量
    let result = m * a;
    println!("M * a = {}", result);

    // 转置
    println!("M^T = {}", m.transpose());

    // 行列式
    println!("det(M) = {}", m.determinant());
}

💡 nalgebra vs 手写代码：关键差异
nalgebra 在编译期检查维度——Vector3 不能和 Vector2 相加， Matrix3 不能乘 Vector2。 我们手写的版本需要自己保证维度正确，而 nalgebra 让编译器帮你检查。 此外，nalgebra 内部会根据平台自动使用 SIMD 指令加速运算。

矩阵求逆与线性方程组

nalgebra 提供了健壮的矩阵求逆和线性方程组求解，自动处理奇异矩阵：

Rust solve_linear.rs 
use nalgebra::{Matrix3, Vector3};

fn main() {
    // 求解线性方程组 Ax = b
    // | 2  1  1 |       | 1 |
    // | 4 -6  0 | * x = | 2 |
    // |-2  7  2 |       | 3 |
    let a = Matrix3::new(
         2.0,  1.0, 1.0,
         4.0, -6.0, 0.0,
        -2.0,  7.0, 2.0,
    );
    let b = Vector3::new(1.0, 2.0, 3.0);

    // 方法 1：通过逆矩阵求解
    match a.try_inverse() {
        Some(a_inv) => {
            let x = a_inv * b;
            println!("通过逆矩阵: x = {}", x);
        }
        None => println!("矩阵不可逆！"),
    }

    // 方法 2：LU 分解（数值更稳定，效率更高）
    let lu = a.lu();
    match lu.solve(&b) {
        Some(x) => {
            println!("通过 LU 分解: x = {}", x);
            // 验证：A * x 应该等于 b
            let residual = (a * x - b).magnitude();
            println!("残差 |Ax - b| = {:.2e}", residual);
        }
        None => println!("方程组无解！"),
    }
}
输出 
通过逆矩阵: x = [0.5, 0.333, -0.333]
通过 LU 分解: x = [0.5, 0.333, -0.333]
残差 |Ax - b| = 2.22e-16

⚠️ 生产环境中永远不要用逆矩阵求解线性方程组
try_inverse() 虽然直观，但计算量是 LU 分解的 2–3 倍， 且数值稳定性更差。在生产代码中，始终使用分解方法（LU、QR、SVD）。 逆矩阵仅在需要反复与多个不同 b 相乘时才值得预计算。

1.4.2 2D 图形变换实战

🤔

你在开发一个 2D 游戏，需要让一个三角形先旋转 45°，再放大 2 倍， 最后向右移动 100 像素。这三个操作能不能合并成一步完成？ 答案是可以——通过齐次坐标和矩阵链式乘法， 任意多个变换都能压缩成一次矩阵乘法。

为什么需要齐次坐标？

旋转和缩放都可以用 2×2 矩阵表示，但平移不行—— 平移是加法，不是乘法。齐次坐标的巧妙之处在于： 把 2D 点 (x, y) 升维到 (x, y, 1)，这样平移也可以用矩阵乘法表示：

\begin{pmatrix} x' \\ y' \\ 1 \end{pmatrix} = \begin{pmatrix} 1 & 0 & t_x \\ 0 & 1 & t_y \\ 0 & 0 & 1 \end{pmatrix} \begin{pmatrix} x \\ y \\ 1 \end{pmatrix}

三种基本 2D 变换

在齐次坐标下，旋转、缩放、平移的矩阵形式如下：

旋转（角度 θ）

R(\theta) = \begin{pmatrix} \cos\theta & -\sin\theta & 0 \\ \sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{pmatrix}

缩放（因子 sx, sy）

S(s_x, s_y) = \begin{pmatrix} s_x & 0 & 0 \\ 0 & s_y & 0 \\ 0 & 0 & 1 \end{pmatrix}

平移（偏移 tx, ty）

T(t_x, t_y) = \begin{pmatrix} 1 & 0 & t_x \\ 0 & 1 & t_y \\ 0 & 0 & 1 \end{pmatrix}

ℹ️ 变换顺序很重要！
矩阵乘法不满足交换律。"先旋转再平移"和"先平移再旋转"的结果完全不同。 在齐次坐标中，变换从右到左应用： T \cdot S \cdot R \cdot \vec{p} 表示先旋转、再缩放、最后平移。

完整项目：2D 变换管线

下面是一个完整的、可编译运行的 Rust 项目，实现了 2D 图形变换管线。 它对一个三角形依次应用旋转、缩放和平移：

Rust transform_2d.rs — 完整可编译项目 
use nalgebra::{Matrix3, Vector3};
use std::f64::consts::PI;

/// 创建 2D 旋转矩阵（齐次坐标）
fn rotation_2d(angle_rad: f64) -> Matrix3<f64> {
    let c = angle_rad.cos();
    let s = angle_rad.sin();
    Matrix3::new(
        c, -s, 0.0,
        s,  c, 0.0,
        0.0, 0.0, 1.0,
    )
}

/// 创建 2D 缩放矩阵（齐次坐标）
fn scaling_2d(sx: f64, sy: f64) -> Matrix3<f64> {
    Matrix3::new(
        sx,  0.0, 0.0,
        0.0, sy,  0.0,
        0.0, 0.0, 1.0,
    )
}

/// 创建 2D 平移矩阵（齐次坐标）
fn translation_2d(tx: f64, ty: f64) -> Matrix3<f64> {
    Matrix3::new(
        1.0, 0.0, tx,
        0.0, 1.0, ty,
        0.0, 0.0, 1.0,
    )
}

/// 将齐次坐标点变换后提取 2D 坐标
fn apply_transform(
    transform: &Matrix3<f64>,
    point: (f64, f64),
) -> (f64, f64) {
    let p = Vector3::new(point.0, point.1, 1.0);
    let result = transform * p;
    (result[0], result[1])
}

fn main() {
    // 定义一个三角形（三个顶点）
    let triangle = vec![
        (0.0, 0.0),   // 原点
        (1.0, 0.0),   // 右方
        (0.5, 0.866), // 上方（等边三角形）
    ];

    println!("=== 2D 图形变换管线 ===\n");
    println!("原始三角形:");
    for (i, p) in triangle.iter().enumerate() {
        println!("  P{}: ({:.3}, {:.3})", i, p.0, p.1);
    }

    // 构建变换矩阵：先旋转 45°，再缩放 2 倍，最后平移 (10, 5)
    let r = rotation_2d(PI / 4.0);       // 45° 旋转
    let s = scaling_2d(2.0, 2.0);         // 均匀缩放 2 倍
    let t = translation_2d(10.0, 5.0);    // 平移

    // 链式组合：从右到左读 → 先 R，再 S，最后 T
    let combined = t * s * r;

    println!("\n组合变换矩阵 T·S·R:");
    println!("{:.4}", combined);

    // 应用变换
    println!("变换后的三角形:");
    for (i, p) in triangle.iter().enumerate() {
        let transformed = apply_transform(&combined, *p);
        println!("  P{}: ({:.3}, {:.3})", i, transformed.0, transformed.1);
    }

    // 验证：逆变换应该恢复原始坐标
    println!("\n--- 验证：逆变换 ---");
    if let Some(inverse) = combined.try_inverse() {
        let p1_transformed = apply_transform(&combined, triangle[1]);
        let p1_recovered = apply_transform(&inverse, p1_transformed);
        println!("P1 原始: ({:.3}, {:.3})", triangle[1].0, triangle[1].1);
        println!("P1 变换后: ({:.3}, {:.3})", p1_transformed.0, p1_transformed.1);
        println!("P1 逆变换恢复: ({:.3}, {:.3})", p1_recovered.0, p1_recovered.1);
    }

    // 演示：变换顺序的影响
    println!("\n--- 变换顺序对比 ---");
    let p = (1.0, 0.0);
    let rotate_then_translate = translation_2d(5.0, 0.0) * rotation_2d(PI / 2.0);
    let translate_then_rotate = rotation_2d(PI / 2.0) * translation_2d(5.0, 0.0);

    let result1 = apply_transform(&rotate_then_translate, p);
    let result2 = apply_transform(&translate_then_rotate, p);
    println!("先旋转90°再平移(5,0): ({:.3}, {:.3})", result1.0, result1.1);
    println!("先平移(5,0)再旋转90°: ({:.3}, {:.3})", result2.0, result2.1);
    println!("→ 结果不同！矩阵乘法不可交换。");
}

💡 工程洞察：为什么游戏引擎用齐次坐标？
齐次坐标的核心价值是统一性——旋转、缩放、平移、 透视投影全部变成矩阵乘法。GPU 的硬件管线就是一连串矩阵乘法， 所以齐次坐标让 CPU 和 GPU 之间的数据传递变得极其简洁。 实际上，OpenGL 和 Vulkan 的整个变换管线都建立在 4×4 齐次矩阵之上。

1.4.3 3D 场景基础

🤔

打开任何一个 3D 游戏，你看到的画面都经过了一系列坐标变换： 物体从自己的模型空间，变换到世界空间， 再变换到摄像机空间，最后通过投影映射到 2D 屏幕上。 这条管线的核心就是两个矩阵——视图矩阵（View Matrix）和投影矩阵（Projection Matrix）。 我们能不能用 nalgebra 从零构建它们？

视图矩阵：Look-At 摄像机

视图矩阵将世界坐标系变换为摄像机坐标系。经典的 "Look-At" 摄像机由三个参数定义：

  • eye：摄像机位置
  • target：摄像机注视的目标点
  • up：世界空间的"上"方向（通常是 Y 轴）

构造过程：先用叉积计算摄像机的三个坐标轴，再组合成变换矩阵：

\text{forward} = \text{normalize}(\text{target} - \text{eye})
\text{right} = \text{normalize}(\text{forward} \times \text{up})
\text{cam\_up} = \text{right} \times \text{forward}
Rust camera.rs — Look-At 摄像机实现 
use nalgebra::{Matrix4, Vector3, Point3};

/// 构建 Look-At 视图矩阵
/// 将世界坐标变换到摄像机坐标系
fn look_at(
    eye: &Point3<f64>,
    target: &Point3<f64>,
    up: &Vector3<f64>,
) -> Matrix4<f64> {
    let forward = (target - eye).normalize();
    let right = forward.cross(up).normalize();
    let cam_up = right.cross(&forward);

    // 视图矩阵 = 旋转部分 × 平移部分
    // 注意 forward 取反——摄像机看向 -Z 方向（OpenGL 惯例）
    Matrix4::new(
         right.x,    right.y,    right.z,   -right.dot(&eye.coords),
         cam_up.x,   cam_up.y,   cam_up.z,  -cam_up.dot(&eye.coords),
        -forward.x, -forward.y, -forward.z,  forward.dot(&eye.coords),
         0.0,        0.0,        0.0,        1.0,
    )
}

/// 构建透视投影矩阵
/// fov_y: 垂直视角（弧度）
/// aspect: 宽高比
/// near/far: 近裁剪面和远裁剪面
fn perspective(
    fov_y: f64,
    aspect: f64,
    near: f64,
    far: f64,
) -> Matrix4<f64> {
    let f = 1.0 / (fov_y / 2.0).tan();
    let nf = 1.0 / (near - far);

    Matrix4::new(
        f / aspect,  0.0,  0.0,                   0.0,
        0.0,         f,    0.0,                   0.0,
        0.0,         0.0,  (far + near) * nf,     2.0 * far * near * nf,
        0.0,         0.0, -1.0,                   0.0,
    )
}

fn main() {
    // 摄像机设置
    let eye = Point3::new(0.0, 2.0, 5.0);     // 摄像机在 (0, 2, 5)
    let target = Point3::new(0.0, 0.0, 0.0);   // 看向原点
    let up = Vector3::new(0.0, 1.0, 0.0);      // Y 轴朝上

    let view = look_at(&eye, &target, &up);
    println!("=== 视图矩阵 ===");
    println!("{:.4}", view);

    // 投影设置
    let fov = std::f64::consts::PI / 4.0; // 45° 视角
    let aspect = 16.0 / 9.0;               // 16:9 宽高比
    let proj = perspective(fov, aspect, 0.1, 100.0);
    println!("=== 投影矩阵 ===");
    println!("{:.4}", proj);

    // 完整管线：世界坐标 → 裁剪坐标
    let world_point = nalgebra::Vector4::new(1.0, 0.0, 0.0, 1.0);
    let clip_point = proj * view * world_point;
    println!("=== 坐标变换管线 ===");
    println!("世界坐标: ({:.2}, {:.2}, {:.2})", world_point.x, world_point.y, world_point.z);
    println!("裁剪坐标: ({:.4}, {:.4}, {:.4}, {:.4})",
        clip_point.x, clip_point.y, clip_point.z, clip_point.w);

    // 透视除法：裁剪坐标 → NDC（归一化设备坐标）
    if clip_point.w.abs() > 1e-10 {
        let ndc = clip_point.xyz() / clip_point.w;
        println!("NDC 坐标: ({:.4}, {:.4}, {:.4})", ndc.x, ndc.y, ndc.z);
        println!("→ NDC 在 [-1,1] 范围内 = 物体在屏幕内可见");
    }
}

💡 Isometry3 vs Matrix4：语义的力量
Isometry3 在类型层面保证变换是刚体变换（保距变换）， 编译器不允许你意外加入缩放或剪切。而 Matrix4 可以表示任何变换—— 包括病态的、行列式为零的变换。在设计 API 时， 使用最精确的类型能让编译器帮你抓 bug。

use criterion::{black_box, criterion_group, criterion_main, Criterion};
use nalgebra::Matrix4;

// ========== 手写 4×4 矩阵 ==========
#[derive(Clone, Copy)]
struct Mat4 {
    data: [[f64; 4]; 4],
}

impl Mat4 {
    fn identity() -> Self {
        let mut data = [[0.0; 4]; 4];
        for i in 0..4 { data[i][i] = 1.0; }
        Mat4 { data }
    }

    fn mul(&self, other: &Mat4) -> Mat4 {
        let mut result = [[0.0; 4]; 4];
        for i in 0..4 {
            for j in 0..4 {
                for k in 0..4 {
                    result[i][j] += self.data[i][k] * other.data[k][j];
                }
            }
        }
        Mat4 { data: result }
    }
}

ℹ️ 为什么 nalgebra 更快？
nalgebra 在编译时检测 CPU 特性，自动使用 SSE/AVX SIMD 指令。 4×4 矩阵乘法的 64 次乘加运算可以被打包成 SIMD 寄存器操作， 一条指令处理 4 个 f64 乘法。此外，nalgebra 的内存布局（列优先） 对 CPU 缓存更友好。手写的三重循环虽然正确，但无法触发自动向量化。

最佳实践决策树

什么时候该用 nalgebra，什么时候该手写？这里是一个实用的决策框架：

用 nalgebra 的场景

✅ 项目需要多种矩阵运算（乘法、求逆、分解）
✅ 性能关键路径上的批量运算
✅ 需要与 Rapier 物理引擎或 Bevy 集成
✅ 需要编译期维度安全

手写实现的场景

✅ 只需要 Vec2/Vec3 的简单运算（加减、点积）
✅ 嵌入式/no_std 环境，不能依赖大型库
✅ 学习目的——理解底层原理
✅ 需要极致控制内存布局（如与 GPU buffer 对齐）

考虑 glam 的场景

✅ 只用 2D/3D/4D 向量和矩阵（不需要任意维度）
✅ 极致性能——glam 专注于游戏数学，SIMD 优化更激进
✅ 与 Bevy 引擎配合（Bevy 内部使用 glam）
✅ 需要更小的编译时间和二进制体积

⚠️ 不要过早优化
上面的纳秒级差异在大多数应用中无关紧要。 先用 nalgebra 写出正确的代码，只在 profiler 确认瓶颈后才考虑切换。 Donald Knuth 说得好："过早优化是万恶之源。" 但也不要忽视算法层面的优化——O(n³) 和 O(n²) 的差异远大于 SIMD 带来的常数因子。

📝 章节总结

  • nalgebra 是 Rust 生态最成熟的线性代数库，提供编译期维度检查和 SIMD 优化

use std::f64::consts::PI;

fn rotation(angle: f64) -> Matrix3<f64> {

    Matrix3::new(c, -s, 0.0, s, c, 0.0, 0.0, 0.0, 1.0)
}

}
fn translation(tx: f64, ty: f64) -> Matrix3<f64> {

}

fn main() {

4. 实现射线-三角形相交检测：使用 Möller–Trumbore 算法， 给定射线的起点和方向，以及三角形的三个顶点，判断射线是否穿过三角形， 并返回交点的距离参数 t。用 nalgebra 的 Vector3 和 Point3 实现。

Rust 参考答案 — Möller–Trumbore 算法 
use nalgebra::{Point3, Vector3};

/// Möller–Trumbore 射线-三角形相交测试
/// 返回 Some(t) 表示在射线 origin + t * direction 处相交
fn ray_triangle_intersect(
    origin: &Point3<f64>,
    direction: &Vector3<f64>,
    v0: &Point3<f64>,
    v1: &Point3<f64>,
    v2: &Point3<f64>,
) -> Option<f64> {
    let epsilon = 1e-10;

    let edge1 = v1 - v0;
    let edge2 = v2 - v0;
    let h = direction.cross(&edge2);
    let a = edge1.dot(&h);

    // 射线平行于三角形
    if a.abs() < epsilon {
        return None;
    }

    let f = 1.0 / a;
    let s = origin - v0;
    let u = f * s.dot(&h);

    if u < 0.0 || u > 1.0 {
        return None;
    }

    let q = s.cross(&edge1);
    let v = f * direction.dot(&q);

    if v < 0.0 || u + v > 1.0 {
        return None;
    }

    let t = f * edge2.dot(&q);

    if t > epsilon {
        Some(t)
    } else {
        None // 交点在射线后方
    }
}

fn main() {
    let origin = Point3::new(0.0, 0.0, -1.0);
    let direction = Vector3::new(0.0, 0.0, 1.0);

    let v0 = Point3::new(-1.0, -1.0, 0.0);
    let v1 = Point3::new( 1.0, -1.0, 0.0);
    let v2 = Point3::new( 0.0,  1.0, 0.0);

    match ray_triangle_intersect(&origin, &direction, &v0, &v1, &v2) {
        Some(t) => {
            let hit = origin + t * direction;
            println!("✓ 命中！t = {:.4}", t);
            println!("  交点: ({:.4}, {:.4}, {:.4})", hit.x, hit.y, hit.z);
        }
        None => println!("✗ 未命中"),
    }
    // 输出: ✓ 命中！t = 1.0000,  交点: (0.0000, 0.0000, 0.0000)
}

use std::io::{self, BufRead, Write};

fn read_matrix() -> Result<Matrix3<f64>, String> {
    let stdin = io::stdin();
    let mut values = Vec::new();
    println!("请输入 3×3 矩阵（9 个数字，空格或换行分隔）：");
    io::stdout().flush().unwrap();

    for line in stdin.lock().lines() {
        let line = line.map_err(|e| e.to_string())?;
        for token in line.split_whitespace() {
            let val: f64 = token.parse()
                .map_err(|_| format!("无法解析数字: {}", token))?;
            values.push(val);
        }
        if values.len() >= 9 { break; }
    }

    if values.len() < 9 {
        return Err("需要 9 个数字".to_string());
    }

    // nalgebra Matrix3::new 按行读入
    Ok(Matrix3::new(
        values[0], values[1], values[2],
        values[3], values[4], values[5],
        values[6], values[7], values[8],
    ))
}

use nalgebra::{Isometry3, Point3, Vector3};

struct Camera {
    position: Point3<f64>,
    yaw: f64,    // 绕 Y 轴旋转角度（弧度）
    pitch: f64,  // 绕 X 轴旋转角度（弧度）
}

impl Camera {
    fn new(pos: Point3<f64>) -> Self {
        Camera { position: pos, yaw: 0.0, pitch: 0.0 }
    }

    fn rotate(&mut self, delta_yaw: f64, delta_pitch: f64) {
        self.yaw += delta_yaw;
        self.pitch += delta_pitch;
        // 限制 pitch 在 ±89°
        let max_pitch = 89.0_f64.to_radians();
        self.pitch = self.pitch.clamp(-max_pitch, max_pitch);
    }

    fn forward(&self) -> Vector3<f64> {
        Vector3::new(
            self.yaw.cos() * self.pitch.cos(),
            self.pitch.sin(),
            self.yaw.sin() * self.pitch.cos(),
        ).normalize()
    }

    fn view_matrix(&self) -> nalgebra::Matrix4<f64> {
        let target = self.position + self.forward();
        let view = Isometry3::look_at_rh(
            &self.position, &target, &Vector3::y(),
        );
        view.to_homogeneous()
    }
}

fn main() {
    let mut cam = Camera::new(Point3::new(0.0, 2.0, 5.0));
    println!("初始朝向: {:.4}", cam.forward().transpose());

    cam.rotate(std::f64::consts::PI / 4.0, 0.1); // 右转 45°, 微微抬头
    println!("旋转后朝向: {:.4}", cam.forward().transpose());
    println!("视图矩阵:\n{:.4}", cam.view_matrix());
}

struct Ball {
    pos: Vector2<f64>,
    vel: Vector2<f64>,
}

fn reflect(vel: &Vector2<f64>, normal: &Vector2<f64>) -> Vector2<f64> {
    vel - 2.0 * vel.dot(normal) * normal
}

fn main() {
    let bounds = (10.0, 8.0); // 矩形边界
    let mut ball = Ball {
        pos: Vector2::new(1.0, 1.0),
        vel: Vector2::new(0.7, 0.5),
    };
    let dt = 1.0;

    println!("=== 弹球模拟 ({}×{}) ===", bounds.0, bounds.1);
    for frame in 0..20 {
        ball.pos += ball.vel * dt;

        // 检测并处理碰撞
        if ball.pos.x <= 0.0 || ball.pos.x >= bounds.0 {
            let nx = if ball.pos.x <= 0.0 {
                Vector2::new(1.0, 0.0)
            } else {
                Vector2::new(-1.0, 0.0)
            };
            ball.vel = reflect(&ball.vel, &nx);
            ball.pos.x = ball.pos.x.clamp(0.0, bounds.0);
        }
        if ball.pos.y <= 0.0 || ball.pos.y >= bounds.1 {
            let ny = if ball.pos.y <= 0.0 {
                Vector2::new(0.0, 1.0)
            } else {
                Vector2::new(0.0, -1.0)
            };
            ball.vel = reflect(&ball.vel, &ny);
            ball.pos.y = ball.pos.y.clamp(0.0, bounds.1);
        }

        println!("帧 {:2}: pos=({:.3}, {:.3}) vel=({:.3}, {:.3})",
            frame, ball.pos.x, ball.pos.y, ball.vel.x, ball.vel.y);
    }
}

8. 基准测试实战：使用 criterion 框架，对比以下三种方法的性能： (a) 手写循环对 1000 个 3D 点做矩阵变换； (b) nalgebra 逐点变换； (c) nalgebra 批量变换（利用矩阵乘矩阵一次性变换所有列向量）。 写出完整的 benches/transform_bench.rs 文件。

Rust 参考答案 — benches/transform_bench.rs 
use criterion::{black_box, criterion_group, criterion_main, Criterion};
use nalgebra::{Matrix3, Matrix3xX, Vector3};

fn bench_batch_transform(c: &mut Criterion) {
    let mut group = c.benchmark_group("Batch 3D Transform (1000 points)");
    let n = 1000;

    let rot = Matrix3::new(
        0.866, -0.5, 0.0,
        0.5,   0.866, 0.0,
        0.0,   0.0,   1.0,
    );

    // 生成随机点
    let points: Vec<[f64; 3]> = (0..n)
        .map(|i| {
            let t = i as f64 * 0.001;
            [t.sin(), t.cos(), t]
        })
        .collect();

    let na_points: Vec<Vector3<f64>> = points.iter()
        .map(|p| Vector3::new(p[0], p[1], p[2]))
        .collect();

    // (a) 手写循环
    group.bench_function("hand-rolled loop", |b| {
        b.iter(|| {
            let r = black_box(&rot);
            let pts = black_box(&points);
            let mut results = Vec::with_capacity(n);
            for p in pts {
                let x = r[(0,0)]*p[0] + r[(0,1)]*p[1] + r[(0,2)]*p[2];
                let y = r[(1,0)]*p[0] + r[(1,1)]*p[1] + r[(1,2)]*p[2];
                let z = r[(2,0)]*p[0] + r[(2,1)]*p[1] + r[(2,2)]*p[2];
                results.push([x, y, z]);
            }
            results
        });
    });

    // (b) nalgebra 逐点
    group.bench_function("nalgebra per-point", |b| {
        b.iter(|| {
            let r = black_box(rot);
            let pts = black_box(&na_points);
            let results: Vec<_> = pts.iter()
                .map(|p| r * p)
                .collect();
            results
        });
    });

    // (c) nalgebra 批量（矩阵×矩阵）
    let point_matrix = Matrix3xX::from_columns(&na_points);
    group.bench_function("nalgebra batch (mat*mat)", |b| {
        b.iter(|| {
            black_box(black_box(rot) * black_box(&point_matrix))
        });
    });

    group.finish();
}

criterion_group!(benches, bench_batch_transform);
criterion_main!(benches);

9. 实现一个微型光线追踪器：在一个场景中放置一个球体， 用射线扫描一个 40×20 的字符网格，命中球体输出 #，否则输出 .。 使用 nalgebra 的向量运算，实现射线-球体相交检测。

Rust 参考答案 — ASCII 光线追踪 
use nalgebra::{Point3, Vector3};

/// 射线-球体相交检测
/// 返回 Some(t) 为最近交点距离
fn ray_sphere_intersect(
    origin: &Point3<f64>,
    dir: &Vector3<f64>,
    center: &Point3<f64>,
    radius: f64,
) -> Option<f64> {
    let oc = origin - center;
    let a = dir.dot(dir);
    let b = 2.0 * oc.dot(dir);
    let c = oc.dot(&oc) - radius * radius;
    let discriminant = b * b - 4.0 * a * c;

    if discriminant < 0.0 {
        return None;
    }

    let t = (-b - discriminant.sqrt()) / (2.0 * a);
    if t > 0.0 { Some(t) } else { None }
}

fn main() {
    let width = 40;
    let height = 20;
    let sphere_center = Point3::new(0.0, 0.0, -3.0);
    let sphere_radius = 1.0;
    let camera = Point3::new(0.0, 0.0, 0.0);

    println!("=== ASCII 光线追踪器 ===");
    for y in 0..height {
        for x in 0..width {
            // 将像素映射到 [-1, 1] 范围
            let u = (x as f64 / width as f64) * 2.0 - 1.0;
            let v = 1.0 - (y as f64 / height as f64) * 2.0;
            let dir = Vector3::new(u, v * 0.5, -1.0).normalize();

            match ray_sphere_intersect(&camera, &dir, &sphere_center, sphere_radius) {
                Some(t) => {
                    // 简单着色：根据距离选择字符
                    let hit = camera + t * dir;
                    let normal = (hit - sphere_center).normalize();
                    let light_dir = Vector3::new(1.0, 1.0, -1.0).normalize();
                    let brightness = normal.dot(&light_dir).max(0.0);
                    let ch = if brightness > 0.8 { '@' }
                        else if brightness > 0.5 { '#' }
                        else if brightness > 0.2 { '+' }
                        else { '.' };
                    print!("{}", ch);
                }
                None => print!(" "),
            }
        }
        println!();
    }
}

10. 综合项目：SVG 变换渲染器：编写一个 Rust 程序， 读取一组 2D 多边形顶点，应用一系列变换（旋转、缩放、平移）， 然后输出 SVG 格式的文件，可以在浏览器中查看变换效果。 要求：(a) 变换链可配置；(b) 输出变换前后的两个多边形用不同颜色标注； (c) 在 SVG 中标注坐标值。

Rust 参考答案 — SVG 变换渲染器 
use nalgebra::{Matrix3, Vector3};
use std::f64::consts::PI;
use std::fs;

fn rotation(a: f64) -> Matrix3<f64> {
    let (s, c) = (a.sin(), a.cos());
    Matrix3::new(c, -s, 0.0, s, c, 0.0, 0.0, 0.0, 1.0)
}
fn scaling(sx: f64, sy: f64) -> Matrix3<f64> {
    Matrix3::new(sx, 0.0, 0.0, 0.0, sy, 0.0, 0.0, 0.0, 1.0)
}
fn translation(tx: f64, ty: f64) -> Matrix3<f64> {
    Matrix3::new(1.0, 0.0, tx, 0.0, 1.0, ty, 0.0, 0.0, 1.0)
}

fn polygon_to_svg_path(pts: &[(f64, f64)]) -> String {
    let mut d = String::new();
    for (i, (x, y)) in pts.iter().enumerate() {
        let cmd = if i == 0 { "M" } else { "L" };
        d.push_str(&format!("{}{:.1},{:.1} ", cmd, x, y));
    }
    d.push('Z');
    d
}

fn main() {
    let shape = vec![
        (50.0, 50.0), (150.0, 50.0),
        (150.0, 150.0), (50.0, 150.0),
    ];

    let transform = translation(200.0, 50.0)
        * scaling(0.8, 1.2)
        * rotation(PI / 6.0);

    let transformed: Vec<_> = shape.iter().map(|&(x, y)| {
        let p = transform * Vector3::new(x, y, 1.0);
        (p[0], p[1])
    }).collect();

    let mut svg = String::from(
        r#"<svg xmlns="http://www.w3.org/2000/svg" width="600" height="400">
  <rect width="600" height="400" fill="#1a1a2e"/>
"#
    );

    // 原始形状（蓝色）
    svg.push_str(&format!(
        r#"  <path d="{}" fill="none" stroke="#4fc3f7" stroke-width="2"/>
"#,
        polygon_to_svg_path(&shape)
    ));
    for (x, y) in &shape {
        svg.push_str(&format!(
            r#"  <text x="{:.0}" y="{:.0}" fill="#4fc3f7" font-size="10">({:.0},{:.0})</text>
"#,
            x + 3.0, y - 3.0, x, y
        ));
    }

    // 变换后（橙色）
    svg.push_str(&format!(
        r#"  <path d="{}" fill="none" stroke="#ffb74d" stroke-width="2"/>
"#,
        polygon_to_svg_path(&transformed)
    ));
    for (x, y) in &transformed {
        svg.push_str(&format!(
            r#"  <text x="{:.0}" y="{:.0}" fill="#ffb74d" font-size="10">({:.0},{:.0})</text>
"#,
            x + 3.0, y - 3.0, x, y
        ));
    }

    svg.push_str("</svg>\n");

    fs::write("output.svg", &svg).expect("写入 SVG 失败");
    println!("✓ 已生成 output.svg");
    println!("  原始形状（蓝色）和变换后（橙色）");
}
