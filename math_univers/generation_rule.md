# math_univers 标准化内容生成规则 (Standard Generation Rule)

> 本文件是 math_univers 模块的标准化内容生成规范。所有子 agent 必须在开始工作前完整阅读并理解本文件。

---

## 一、项目概述

**主题：** math_univers — 面向 Rust 开发者的数学基础学习平台

**目标受众：** 具备基础编程能力、正在学习或已掌握 Rust 的开发者，方向涵盖游戏开发、系统编程、安全攻防。

**核心理念：**
- 数学不是抽象符号堆砌，而是解决工程问题的工具
- 每个概念必须回答：为什么要学？怎么用？Rust 中如何实现？
- 交互式学习优先，拒绝填鸭式教学

## 三、工作步骤

每个子 agent 必须按以下顺序执行：

### 步骤 1：学习规范
- 完整阅读本 generation rule 文件
- 阅读示例页面，理解页面结构与风格

- 细致规划负责单元/章节的所有知识点

- 规划快速测试和课后练习题目
- 将规划写入 workflow.md

### 步骤 3：生成内容
- 按模板生成 HTML 页面
- 确保所有交互组件正常工作
- 确保代码示例可运行且正确

### 步骤 4：自我审阅
- 按评分规则自评
- 记录自评结果到 workflow.md
- 修复发现的问题

## 四、内容质量要求

### 4.1 知识讲解要求
1. **禁止**空泛的概念列举，必须用具体例子说明
2. 每个概念必须回答以下问题中的至少 2 个：
   - 为什么要研究这个概念？
   - 它解决了什么实际问题？
   - 在 Rust/工程中如何应用？
   - 常见误区是什么？
   - 与其他概念的关系是什么？
3. 数学公式必须使用 KaTeX 渲染
4. 每个概念配备至少一个**具体的、可运行的** Rust 代码示例

### 4.2 Rust 代码要求
1. 所有代码必须是合法的 Rust 代码
2. 代码必须有注释说明关键步骤
3. 优先使用标准库，必要时使用常见 crate（nalgebra, rand, num 等）
4. 代码风格遵循 Rust 社区惯例（rustfmt 格式化）

### 4.3 讲解风格要求
1. 使用 **引导式提问** 开始每个知识点
   - 例：「如果我们要在屏幕上旋转一个三角形，需要哪些数学工具？」
2. 使用 **类比** 解释抽象概念
   - 例：「向量空间就像一个工具箱——基向量是基础工具，所有其他向量都是这些工具的组合」
3. 使用 **对比** 区分易混淆概念
   - 例：「点积产生标量（数值），叉积产生向量（方向）——就像称重给你一个数字，而指南针给你一个方向」
4. 使用 **渐进式揭示** 构建理解
   - 先给出直觉 → 形式化定义 → 具体例子 → 代码实现

## 六、练习设计要求

### 6.1 快速知识点测试
- 每个知识点模块后 1-2 道题
- 题型：选择题 / 判断题
- 目的：检测核心概念是否掌握
- 聚焦易错点和难点
- 必须有即时反馈（答案解析）

### 6.2 课后练习
- 每章 8-15 道题
- 难度梯度：
  - ⭐ 基础（40%）：直接应用概念
  - ⭐⭐ 进阶（35%）：需要综合运用
  - ⭐⭐⭐ 挑战（15%）：复杂问题或证明
  - 🏆 附加题（10%）：结合工程实践或 Rust 实现
- 题型可包括：
  - 选择题 / 填空题
  - 计算题
  - 证明题
  - 代码编写题
  - 方案设计题
  - 调试分析题

### 6.3 附加题要求
- 必须结合实际工程场景
- 例如：「使用 Rust 实现一个函数，用矩阵乘法完成 2D 旋转变换」
- 提供部分代码框架，要求补全关键部分
